페이지 내용

    자주 하는 질문: 분석 도구 모듈

    이 FAQ에서는 비디오 클라우드 스튜디오 분석 모듈에 대한 몇 가지 일반적인 질문을 다룹니다. 분석 모듈에 대한 자세한 내용은 분석 모듈시작하기를참조하십시오.

    1. 분석 모듈이란 무엇입니까?

    Video Cloud Studio Analytics 모듈은 비디오 및 플레이어에 대한 실시간 분석 정보를 볼 때 사용할 수 있는 사용자 인터페이스 모듈입니다. Analytics 플랫폼은 빅데이터 기술의 최신 혁신을 사용하며 대규모로 설계되었습니다. 분석 플랫폼은 분석 데이터에 대한 빠르고 가까운 실시간 액세스를 제공하며 개발자가 Video Cloud Studio에 포함된 솔루션 외에 맞춤형 분석 솔루션 및 보고서를 생성할 수 있는 강력한 개방형 API를 제공합니다.

    2. 분석 모듈에서 보고서 데이터를 내보낼 수 있습니까?

    예. 분석 보고서에는 보고서 데이터를.csv 및.xlsx 파일로 다운로드하는 데 사용할 수 있는 링크가 있습니다.

    3. 분석 모듈에서 데이터가 얼마나 자주 업데이트됩니까?

    일반적으로 실시간에 가까운 데이터는 약 15분 지연됩니다. 이 시간은 데이터 수집 시스템에서 현재 수신 중인 데이터의 양에 따라 달라질 수 있으며 경우에 따라 최대 3시간까지 지연될 수 있습니다.

    Analytics 시스템은 여러 소스의 데이터를 컴파일하며 가까운 실시간 데이터는 지금까지 수집된 데이터를 기반으로 한 근사값입니다. 숫자는 보고서마다 다를 수 있으며 하드 수치가 아닌 추세로 간주해야합니다.

    4. 내부 동영상 ID를 보고 또는 API에 포함시킬 수 있는 방법이 있습니까?

    Video Cloud의 비디오에 내부 ID를 매핑하는 데 사용할 수 있는 비디오 속성으로 사용할 수 있는 참조 ID 필드가 있습니다. 사용자 정의 보고서를 작성할 때 참조 ID 필드를 표시할 수 있습니다. 사용자 지정 보고서에 대한 자세한 내용은 분석모듈에서 사용자 지정 분석 보고서 만들기를참조하십시오.

    5. 동영상 앞에 프리롤 광고가 있고 시청자가 몇 초 동안 광고를 본 다음 떠나는 경우 어떻게됩니까? 시청자가 주요 콘텐츠를 전혀 보지 않은 경우 동영상 보기로 계산됩니까?

    안 돼비디오 보기는 주 콘텐츠가 재생되기 시작한 후에만 계산됩니다.

    6. 분석에서 IP 주소 범위를 필터링할 수 있습니까?

    IP 주소 범위를 필터링하는 기능은 현재 사용할 수 없습니다.

    7. 분석은 실시간 이벤트 스트리밍을 통해 플레이어를 추적합니까?

    애널리틱스는 성과 보고서의 일부로 조회한 분을 추적합니다. 사용자 지정 보고서를 만들 때 또는 청구 모듈을 통해 본 라이브 초를 표시할 수 있습니다. 라이브 이벤트 대역폭은 표시되지 않습니다.

    8. 분석 모듈을 통해 특정 동영상에 대한 정보를 찾으려면 어떻게 해야 합니까?

    검색 기능은 특정 비디오를 검색하는 데 사용할 수 있습니다. 비디오 이름, 간단한 설명, 긴 설명, 태그, 비디오 ID, 참조 ID 및 사용자 정의 텍스트 필드가 검색됩니다. 결과를 클릭하면 해당 비디오의 실적이나 참여도가 표시됩니다. 사용자 지정 보고서를 만들어 지정된 동영상 ID에 대한 분석을 표시할 수도 있습니다.

    9. 분석 모듈에서 참조 ID를 사용하여 특정 동영상을 검색할 수 있습니까?

    예. 검색 기능은 참조 ID 필드를 기반으로 비디오를 검색하는 데 사용할 수 있습니다. 비디오 이름, 간단한 설명, 긴 설명, 태그, 비디오 ID 및 사용자 정의 텍스트 필드도 검색됩니다.

    10. 고객 참여에 대해 보고하고 특정 동영상을 중단할 수 있는 방법은 무엇입니까?

    서비스 보고서에는 이 정보가 표시됩니다. 참여도 보고서를 선택하고 비디오 측정기준을 선택하면 계정에 대한 전반적인 참여도를 보여주는 그래프가 표시됩니다. 비디오를 선택하면 시간에 따른 해당 비디오의 참여도를 보여주는 그래프가 표시됩니다.

    11. URL에 대해 보고할 수 있습니까? 예를 들어 페이지 A와 페이지 B에서 동일한 비디오를 호스팅하는 경우 두 게재 위치 간의 실적을 비교할 수 있습니까?

    예. 성능 보고서는 표시 도메인별로 비디오 분석을 볼 수 있는 기능을 제공합니다.

    12. 동영상이 반복되는 경우 동영상이 재생될 때마다 동영상 보기 1회 또는 여러 조회수로 계산되나요?

    비디오 보기는 스트림이 시작될 때 시청 세션당 한 번만 계산됩니다. 참여 보고서의x% 조회수는성과 보고서의비디오 보기와동일하지 않습니다. 비디오 시청수는 비디오가 처음 재생되기 시작할 때 올라갑니다. 1% 시점의 시청 횟수는 첫 1%를 시청했을 때 기록됩니다. 여기서 차이점은 첫 번째 퍼센트가 여러 번 시청되는 경우 (예: 비디오가 자동 재생으로 설정된 경우) 카운트가 증가하지만 비디오가 처음 재생되기 시작하지만 재생시 계산되지 않을 때 비디오 보기는 카운트된다는 것입니다 (사용자가 페이지를 다시 로드하거나재생 목록 등).

    13. 내 계정의 고유 시청자 수를 표시하는 보고서를 실행할 수 있습니까?

    고유 시청자 수는 성과 보고서에서 확인할 수 있습니다.

    14. Analytics 모듈에서 사용할 수 있는 표준 보고서를 사용자 지정할 수 있습니까?

    성과 및 참여 보고서에 표시된 데이터는 사용자 지정할 수 없습니다. 그러나 Analytics 모듈은 표시할 데이터를 선택할 수 있도록 사용자 지정 분석 보고서를 만들 수 있는 기능을 제공합니다. 프로그래밍 방식으로 분석 데이터에 액세스하는 데 사용할 수 있는 도 있습니다. Analytics API

    15. 일부 참여 점수가 100보다 높은 이유는 무엇입니까?

    인게이지먼트 점수는 100점을 초과할 수도 있습니다. 비디오를 스크러빙하고 다시 시청해도 새로운 비디오_시청 이벤트를 시작하지는 않지만, 비디오_인게이지먼트를 추적합니다. 예를 들어, 어떤 페이지 상 하나의 비디오 플레이어에 비디오가 있다고 가정합시다. 다음은 그런 경우에 비디오 인게이지먼트가 대단히 폭넓은 방식으로 달라질 수 있다는 것을 보여주는 사례입니다.

    1. 아직 시청 건수가 없습니다. 인게이지먼트 점수: 0.
    2. 뷰어는 페이지를로드하고 비디오를 끝까지 시청합니다. 인게이지먼트 점수: 100(100% 시청함/1회 시청).
    3. 같은 시청자가 비디오를 다시 스크러빙해 처음부터 끝까지 다시 시청합니다(브라우저를 다시 로드하지는 않았습니다). 인게이지먼트 점수: 200(200% 시청함/1회 시청).
    4. 또 다른 시청자가 해당 페이지를 로드하여 전체 비디오를 감상합니다. 인게이지먼트 점수: 150(300% 시청함/2회 시청).

    비디오 인게이지먼트 계산법에 대한 자세한 내용은 교육 비디오인 비디오 인게이지먼트 계산하기를 참고하십시오.

    16. 조회수에 대한 지표가 조회수보다 25% 높은 이유는 무엇입니까?

    두 숫자의 불일치를 일으킬 수있는 몇 가지 요인이 있습니다.

    • 1% 의 조회수와 x%의 조회수는 서로 다른 두 개의 지표입니다. 1% 의 뷰는 뷰 세션마다 고유하며 되감기 및 재생은 포함되지 않습니다. 시청자가 동영상의 이전 부분을 다시 재생하거나 되감으면 시청자 수는 증가하지만 동영상 조회수는 증가하지 않습니다.
    • % 및 참여 곡선의 뷰는 UTC 시간을 사용하여 일별 세분성으로 저장되므로 UTC 및 보고 표준 시간대 크로스오버인 경우 특정 날짜에 2일 분량의 데이터가 포함될 수 있습니다.

    17. 특정 날짜의 비디오 시청 횟수가 시간에 따라 달라지는 이유는 무엇입니까?

    분석 시스템은 매시간 수집된 비조정 최근 지표를 제공하고 일일 수준에서 보고된 실제 지표와 자동으로 조정합니다. 이 화해는 가능한 한 빨리 발생하며 대개 3 ~ 5 일 내에 발생합니다. 우리는 시간별 수준에서 수집한 데이터를 조정하지 않습니다. 게시자는 시간당 데이터를 중요하게 사용하므로 시간별 비조정 데이터를 32일 동안 사용할 수 있습니다. 조정에 대한 몇 가지 추가 세부 정보:

    • 플레이어는 데이터 수집 API로 데이터를 보냅니다.
    •  데이터 수집 API는 실시간 데이터베이스에 기록하기 위해 최선을 다하며 로그 저장소에 이벤트를 기록합니다.
    • CDN 및 기타 소스에서 추가 데이터를 수집하면 분석 데이터가 업데이트됩니다. 이는 조정 프로세스입니다.
    • 야간 배치 시스템은 원시 로그 이벤트를 확인하고 기록 데이터 가져오기 파일을 작성합니다.
    • 데이터 가져오기 파일이 기록 데이터베이스에 로드됩니다. API 호출을 수행할 때 조정 매개 변수를 사용하여 조정된 데이터만 표시할지 (가장 정확한) 조정된 데이터와 비조정 데이터를 모두 표시할지 여부를 선택할 수 있습니다. 자세한 내용은분석 API 개요를참조하십시오.
    • 모든 기록 데이터가 조정됩니다. 실시간 데이터 (32일 동안 사용 가능) 는 아직 조정되지 않았거나 조정되지 않았을 수 있습니다. 데이터를 가져 오는 기간이 32 일 이상 전에 끝나는 경우reconciled data, since the data will all be historical.

    18. 일부 지표가 null 값을 반환하거나 값이 없는 이유는 무엇입니까?

    이는 요청한 기간 동안 메트릭을 사용할 수 없는 경우에 발생합니다. 일부 지표에는 2 - 3일의 대기 시간이 있으므로 예를 들어 지난 30일 (오늘 포함) 동안 이 데이터를 요청하면 요청된 범위에 아직 이 데이터가 없는 일이 포함되므로 이러한 지표가 null일 수 있습니다. 데이터를 요청할 때 이를 방지하려면 요청된 범위가 대기 시간 기간을 벗어나야 합니다.

    또한 시간이 지남에 따라 측정항목 및 측정기준 조합도 추가되었습니다. 예를 들어 다년간의 보고서를 실행하는 경우 전체 보고 날짜 범위에 대해 해당 지표를 추적하지 않았기 때문에 일부 지표가 비어 있거나 부분적일 수 있습니다. 유효한 날짜 및 차원 조합에 대한 자세한 내용은Analytics API 개요문서를 참조하십시오.

    19. 2013의 대상 도메인 데이터가 다른 성능 데이터와 일치하지 않는 이유는 무엇입니까?

    대상 도메인 기능은 2014년 1월까지 설정되지 않았습니다. 2013년 3월부터 2014년 1월까지의 데이터는 부분적으로 채워졌지만 이 기간 동안 일부 대상 데이터만 사용할 수 있습니다. 2014년 1월 이후 데이터가 완료되었습니다.

    20. 삭제된 동영상에 대한 분석이 포함되어 있나요?

    애널리틱스 보고서에는 계정에서 삭제된 동영상에 대한 분석이 포함됩니다. 그러나 삭제된 특정 비디오에 대한 보고서를 보기 위해 애널리틱스 모듈에서 삭제된 비디오를 검색할 수는 없습니다. 이러한 보고서를 가져오려면 삭제된 동영상의 동영상 ID를 알고Analytics API를사용하여 보고서를 생성해야 합니다. 보고서 데이터를 내보내는 경우 삭제된 비디오의 데이터가 비디오 ID나 이름을 표시하지 않고 나타날 수 있습니다.

    21. 분석 보고서에서 직원 조회수를 제외할 수 있습니까?

    이를 쉽게 수행 할 수있는 방법은 없으며 트래픽이 많은 비디오의 경우 직원 조회수는 분석 결과에 미치는 영향은 무시할 정도로 적습니다. 그러나 이렇게 하는 것이 중요하다고 생각되는 트래픽이 낮은 비디오의 경우 가장 간단한 해결책은 프로덕션 플레이어를 복제하고 직원이 복사본에 대한 비디오를 테스트/보도록 하는 것입니다. 그런 다음 필터를 사용하여 애널리틱스 모듈의 사용자 지정 보고서 기능 또는 Analytics API를 사용하여 프로덕션 플레이어에 대해서만 보고서를 생성할 수 있습니다.

    22. 참여를 위해 항상 정수를 보지 않는 이유는 무엇입니까?

    결속 번호가 소수점으로 반환될 수 있습니다. 그 이유는 참여가 정규화되어 있기 때문입니다. 즉, “video_percent_video_engagement _sum (video_engagement _25/video_engagement _sum)" 의 비율이므로 쿼리에 대해 선택한 시간 범위에 따라 정확하게 나누지 않는 경우 부동 소수점 숫자가 표시 될 수 있습니다.

    23. 일부 보고서에서 동영상 ID만 표시하는 이유 또는 일부 동영상이 전혀 표시되지 않는 이유는 무엇입니까?

    일정 기간 동안 동영상에 대한 조회수가 없는 경우 보고서에 광범위한 데이터가 표시되지 않습니다 (예: 동영상 ID만 표시됨). 데이터가 전혀 다운로드되지 않으면 (즉, 이미지 데이터도 아님) 비디오가 보고서에 표시되지 않습니다.

    24. 분석 데이터를 얻을 수 있는 가장 빠른 날짜는 어떻게 되나요?

    분석 데이터의 가장 빠른 날짜는 2010년 12월 28일입니다. 일부 데이터 항목은 나중에 수집되지 않았으므로 데이터를 볼 수 있는 가장 빠른 날짜는 보고 있는 항목에 따라 달라집니다. 비디오 보기는 2010 년 12 월 28 일까지 사용할 수 있습니다.

    25. Analytics 모듈과 동일한 날짜 범위에 대해 동일한 데이터를Analytics API제공해야합니까?

    예. 하지만 비교적 최근의 데이터를 비교하고 Studio 및 API에서 다른 시간에 데이터를 얻는 경우 다를 수 있습니다.

    1. 데이터는 3 일 미만입니다. 3일 미만의 분석 데이터는 잠정적이며 언제든지 변경될 수 있습니다.
    2. 같은 기간의 데이터는 32 일 미만이었을 때, 그리고 32 일이 지난 후에 다시 살펴 봅니다. 32일이 지나면 데이터가 별도의 “기록” 저장소로 이동되고, 이 시점에서 일부 데이터, 특히 상세한 참여 데이터가 폐기됩니다. 즉, 덜 세분화된 데이터에 대한 계산이 이루어지기 때문에 참여 점수와 같은 계산된 필드가 약간씩 변경될 수 있습니다.
