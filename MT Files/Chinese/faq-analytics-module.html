    常問問題：分析模組

    此常見問題集解決了有關視訊雲端 Studio 分析模組的一些常見問題。有關Google Analytics（分析）模塊的更多信息，請參見Analytics模塊入門。

    1.什麼是分析模組？

    Video Cloud Studio 分析模組是一個使用者介面模組，您可以用來檢視視訊和播放器的接近即時分析資訊。Analytics 平台採用最新的大數據技術創新，並為大規模設計。該分析平台提供對分析數據的快速，接近實時的訪問，並包括一個強大的開放API，使開發人員可以創建自定義分析解決方案和報告，以及Video Cloud Studio中包含的那些解決方案和報告。

    2.我可以從分析模組匯出報告資料嗎？

    是的分析報表具有可用來將報告資料下載至 .csv 和 .xlsx 檔案的連結。

    3.分析模組中的資料更新頻率為何？

    通常情況下，接近實時的數據延遲約 15 分鐘。然而，該時間可能會因資料收集系統目前接收的資料量而有所不同，在某些情況下，延遲可能長達 3 小時。

    請注意，Analytics (分析) 系統會編譯來自多個來源的資料，而接近即時的資料則是以迄今為止收集到的資料為基礎的近似值。數字可能因一份報告而異，應視為趨勢，而非硬數字。

    4.有沒有辦法將我們的內部視頻 ID 包含在報告或您的 API 中？

    有一個參考 ID 欄位可用作視訊屬性，您可以用來將內部 ID 對應至 Video Cloud 中的影片。建立自訂報告時，可以顯示參考 ID 欄位。有關自定義報告的更多信息，請參見在分析模塊中創建自定義分析報告。

    5.如果影片前有一個預先播放廣告，而觀眾觀看廣告幾秒後再離開，會發生什麼情況。如果觀眾完全沒有觀看主要內容，這是否會計為影片觀看次數？

    沒有只有在主要內容開始播放後，才會計算影片檢視畫面。

    6.是否可以從我們的分析過濾 IP 地址範圍？

    目前無法使用篩選掉 IP 位址範圍的功能。

    7.分析是否會透過即時活動串流來追蹤玩家？

    「分析」會追蹤「績效」報表中已檢視的分鐘數。建立自訂報表或透過帳單模組時，可顯示即時檢視秒數。不會顯示即時事件頻寬。

    8.如何透過分析模組尋找特定影片的資訊？

    搜索功能可用於搜索特定視頻。系統會搜尋影片名稱、簡短說明、完整說明、標籤、影片 ID、參考 ID 以及您擁有的任何自訂文字欄位。按一下結果會顯示該影片的表現或參與度。您也可以建立自訂報表，以顯示指定視訊 ID 的分析。

    9.我可以使用分析模組中的參考 ID 來搜尋特定影片嗎？

    是的搜尋功能可用來根據參考 ID 欄位搜尋影片。您也會搜尋影片名稱、簡短說明、完整說明、標籤、影片 ID 以及您擁有的任何自訂文字欄位。

    十.如何報告特定影片的客戶參與度和送出？

    「業務開發報告」會顯示此資訊。當您選取參與報表並選取 [視訊] 維度時，您會看到顯示帳戶整體參與度的圖表。如果您選擇影片，您會看到一個圖表，顯示該視訊隨著時間的推移參與次數。

    十一我可以報告網址嗎？例如，如果我們在頁面 A 和頁面 B 上託管相同的視頻，我們可以比較兩個版本之間的性能嗎？

    是的效能報告提供依顯示網域檢視視訊分析的功能。

    十二如果影片正在循環播放，每次播放影片時，這是否會計為一個視訊檢視或多次觀看次數？

    串流開始時，每個觀看工作階段只會計算一次影片觀看次數。請注意觀看次數x％參與度報告中的內容與影片觀看在效果報告中。首次開始播放視頻時，將計算視頻觀看次數。觀看第一個百分率時，將記錄1％的觀看次數。此處的區別在於，如果多次觀看了前一個百分比（例如，如果將視頻設置為自動重播），則計數會增加，而當視頻首次開始播放但不計入視頻觀看次數時，重放（除非用戶重新加載頁面或觀看播放列表中的其他視頻，等等）。

    十三我可以執行報表來顯示我帳戶的獨特觀眾人數嗎？

    您可以在 [成效報告] 中查看獨特檢視者的數目。

    十四.可以自訂 Analytics (分析) 模組中提供的標準報表嗎？

    「績效」和「參與」報表中顯示的資料無法自訂。不過，Analytics (分析) 模組可讓您建立自訂分析報表，讓您選取要顯示的資料。還有一個Analytics API可用於以編程方式訪問分析數據。

    十五為什麼有些訂婚分數非常高，大於 100？

    參與度得分可能會大於100。擦洗和重新觀看視頻不會觸發新的video_view事件，但會跟踪video_engagement。例如，假設我們在某個地方的頁面上的單個視頻播放器中有一個視頻。以下事件說明了視頻互動的變化範圍：

    1. 還沒有手錶。參與度得分：0。
    2. 觀眾加載頁面並一直觀看視頻。參與度得分：100（已查看100％/ 1個視圖）。
    3. 相同的查看者會回滾並重新觀看整個過程（瀏覽器未重新加載）。參與度得分：200（查看200％的內容/ 1個視圖）。
    4. 另一位檢視器載入頁面並觀看整個視訊。參與度得分：150（已觀看300％/ 2次觀看）。

    有關我們如何計算視頻參與度的更多詳細信息，請觀看培訓視頻計算視頻參與度。

    十六。為什麼檢視表的度量比檢視次數高 25%，而且在 1% 的情況下？

    有幾個因素可能導致兩個數字的差異：

    • 在 1% 的視圖和 x％ 的視圖是兩個不同的度量。在每個檢視階段作業中，1% 的視圖是唯一的，不包括倒轉和重新顯示。觀眾重播或倒轉至影片的前一部分時，% 點數會增加觀眾人數，但不會增加影片觀看次數。
    • 百分比的視圖和參與曲線使用 UTC 時間以每日粒度存儲，因此如果 UTC 和報告時區交叉，特定日期有時可能包含 2 天的數據。

    十七為什麼特定日期的影片觀看次數會隨著時間而有所不同？

    分析系統提供每小時收集的未調和最近收集的量度，並自動與每日層級報告的實際指標進行調整。這種調解會盡快發生，通常會在 3 到 5 天內進行。我們從未調整每小時收集到的資料。由於發佈商重視每小時資料，因此我們將每小時未調節的資料提供 32 天。關於和解的一些進一步細節：

    • 玩家將資料傳送至資料收集 API
    •  數據收集API會盡力將其寫入實時數據庫 然後將事件寫入日誌存儲
    • 當我們從 CDN 和其他來源收集其他資料時，會更新分析資料；這是對帳程序
    • 每晚的批次系統查看原始日誌事件並構建歷史數據導入文件
    • 資料匯入檔案會載入到歷史資料庫中。當您進行 API 呼叫時，您可以選擇是否只查看已調解的資料 (最準確)，或是使用調解的參數來查看已調解和未調解的資料。見Analytics API概述欲獲得更多信息。
    • 所有歷史資料都會進行調解；即時資料 (可用 32 天) 可能尚未進行調解。如果您要獲取數據的時間段結束於32天前，則無需使用reconciled data, since the data will all be historical.

    十八歲為什麼某些度量返回空值或沒有值？

    如果您要求的期間內無法使用量度，就會發生這種情況。有些度量有 2 到 3 天的延遲，因此，例如，如果您在過去 30 天 (包括今天) 詢問此資料，這些度量可能為空，因為要求的範圍包含尚未擁有此資料的天數。若要避免在要求資料時發生這種情況，請確定要求的範圍超出延遲期間。

    隨著時間的推移，我們也新增了額外的量度和維度組合。因此，例如，如果您執行多年報表，某些量度可能是空白或部分的，因為我們沒有在整個報告日期範圍內追蹤該量度。有關有效日期和維度組合的信息，請參見Analytics API概述文件。

    十九為什麼 2013 的目的地網域資料與其他效能資料不符？

    目的地網域功能直到 2014 年 1 月才開啟。2013 年 3 月至 2014 年 1 月的數據部分回填，但這段期間只有部分目的地數據可用。2014 年 1 月之後的資料已完成。

    二十。是否包含已刪除影片的分析資料？

    Google Analytics（分析）報告確實包含對已從您的帳戶中刪除的視頻的分析。不過，您無法在 Analytics (分析) 模組中搜尋已刪除的影片，以查看特定已刪除影片的報表。要獲得此類報告，您需要知道已刪除視頻的視頻ID，並使用Analytics API生成報告。請注意，如果匯出報表資料，已刪除影片的資料可能不會顯示視訊 ID 或名稱。

    二十一歲。我可以從分析報表中排除員工檢視嗎？

    沒有簡單的方法可以做到這一點，對於高視頻量的視頻，員工觀看次數可能只佔很小的比例，因此其對分析的影響可以忽略不計。但是，對於您認為這樣做很重要的低流量視頻，最簡單的解決方案是複制您的製作播放器，並讓員工測試/觀看副本上的視頻。然後，您可以使用Analytics（分析）模塊中的“自定義報告”功能或Analytics API，使用過濾器僅在生產播放器上創建報告。

    二十二為什麼我不總是看到整數的參與？

    訂婚號可能會返回小數點。原因是參與度已標準化，這意味著它是“ video_percent_viewed *（video_engagement_25 / video_engagement_sum）”的比率，因此，根據為查詢選擇的時間範圍，在它們不完全相除的情況下，您可能會看到浮點數。

    二十三為什麼有些報表只會顯示視訊 ID，或為什麼某些影片完全不會顯示？

    如果一段時間內沒有影片的觀看次數，報表中將不會有大量資料 (例如，只有影片 ID，沒有其他內容)。如果完全沒有下載任何數據（即甚至沒有圖像數據），則該視頻將不會出現在報表中。

    二十四我可以取得分析資料的最早日期是什麼？

    任何分析資料的最早日期為 2010 年 12 月 28 日。有些資料項目在稍後才會收集，因此您可以看到資料的最早日期將取決於您正在查看的內容。影片觀看次數可於 2010 年 12 月 28 日發放。

    二十五歲Google Analytics（分析）模塊和Analytics API給我相同日期範圍內的相同數據？

    是的，但是如果您比較的是相對較新的數據，並且在不同的時間從Studio和API獲取數據，則它們可能會有所不同：

    1. 數據不到3天。少於 3 天的分析資料為暫時性資料，可能隨時變更。
    2. 在小於32天的時間裡查看同一時期的數據，在小於32天的時候再查看一次。32 天後，數據被移動到一個單獨的「歷史」存儲庫，並在這一點上丟棄一些數據，特別是詳細的參與數據。這意味著諸如參與度分數之類的計算字段可能會略有變化，因為該計算是針對粒度較小的數據進行的。
