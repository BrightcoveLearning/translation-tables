    概述：視訊雲端中的數位版權管理 (DRM)

    本主題提供視訊雲端中使用 DRM 的概觀。

    簡介

    數位版權管理 (DRM) 可防止您的影片播放，除非在獲得權限的用戶端中。DRM 的實現不同，但通常使用的機制是加密視頻並解密它，只有當客戶端有某種授權播放的密鑰時才會進行解密。

    另請參閱：

    • 使用 DRM 擷取內容
    • 為DRM配置Brightcove Player
    • 帶有Native Player SDK的內容安全性（DRM）
    • DRM 輸出保護設定

    如需可在播放程式組態或視訊屬性中設定的其他內容安全性機制，請參閱：

    • 設定播放程式內容限制
    • 設定視訊可用性
    • 使用 HLS 加密保護視訊
    注意事項：
    • 若要使用 DRM 或 HLS 加密，您的帳戶必須啟用這些功能。如需詳細資訊，請連絡您的客戶經理。
    • 啟用 DRM 的視訊時，MP4 來源/轉譯會在 Studio、CMS API 和播放 API 中自動隱藏 (表示它們存在，但不會以來源形式傳送給播放程式)。
    • 布萊特灣目前不支援僅限視訊或僅限音訊的 DRM 播放。

    支援的 DRM 類型

    視訊雲端為 DRM 提供廣泛的支援：

    • 全面支援封裝、授權服務和播放
    • 適當的 DRM 適應使用給定裝置：
      • 視頻雲有利於原生設備 DRM，最大限度地減少對插件的依賴
      • Video Cloud 運用標準來減少提供跨平台解決方案所需的轉譯次數

    布萊特灣支援三種最受歡迎的 HTML5 視訊 DRM 類型：

    • 蘋果FairPlay
    • Google Widevine模塊化
    • Microsoft PlayReady

    您可以註冊多種 DRM 類型，以保護您在所有裝置和平台上的內容。

    DRM 支援詳細資訊

    瀏覽器

    瀏覽器的 DRM 適用於桌面或移動網絡（Android 6 以上的設備上的 Chrome; iOS 11.2+ 設備上的 Safari）。布萊特灣使用 MPEG-DASH 和通用加密與威德維文模塊化，蘋果公平播放和微軟 PlayReady 在桌面上提供 DRM-使用哪種類型取決於瀏覽器。這提供了一組可以在不同瀏覽器上播放的資產，同時利用現代瀏覽器可用的原生 DRM。

    iOS 和安卓應用程式

    以下信息適用於使用Brightcove本機播放器SDK。請注意，除了常規播放外，SDK還支持使用DRM進行離線播放。

    iOS

    對於iOS設備，當前的Video Cloud DRM解決方案是帶FairPlay流的HLS。

    Apple 的 FairPlay 串流 (FPS) 技術提供安全的解決方案，可將 HTTP 直播串流 (HLS) 影片傳送至 Apple 裝置，包括 iPhone、iPad 和 Apple TV。它使用已建立的安全金鑰通訊協定，並將行動裝置的耗電量降至最低。

    Android

    對於 Android 設備，解決方案是 MPEG-破折號 + CENC（威德文模塊化）。

    Google Widevine 提供多平台、多格式 DRM 和影片優化解決方案，讓內容擁有者、網際網路影片供應商、零售商和影片營運商可以用來為網際網路連線的娛樂平台提供品牌高品質的娛樂產品。

    DRM 安全級別與安卓裝置

    Widevine 實施了一套業界標準，包括 CENC 加密、授權金鑰交換和適應性串流品質，以管理視訊保護。根據接收裝置的安全性功能，Widevine 支援多層級的串流品質。

    安全性等級被命名為 L1，L2 和 L3：

    • 1級 -最高的安全級別
      • 只有安全性處理器才能看到透明金鑰
      • 受保護的視訊路徑。清晰的視頻不會暴露。
    • 2級 -不太安全
      • 只有安全性處理器才能看到透明金鑰
      • 沒有受保護的視訊路徑。清晰的視訊曝光。
    • 3級 -最低安全等級
      • 主要 CPU 完成的關鍵處理
      • 沒有受保護的視訊路徑。清晰的視訊曝光。

    Android 設備支持 L1 或 L3 的安全級別，基於硬件和軟件配置。只有符合 L1 規格的裝置才能從受 WiDevine 保護的伺服器串流 HD 或更高品質的內容。如果您的裝置僅符合 L3 規範，您將無法串流受 DRM 保護的 HD 內容。

    如需詳細資訊，請參閱下列內容：

    • Widevine的DRM解決方案文檔
    • Android數字版權管理（DRM）文檔
    • 確定Android設備的安全級別

    使用 DRM 離線播放

    離線播放可讓使用者在連線時下載內容，然後在離線時觀看內容。為了保護您的內容，原生 SDK 會使用下列項目：

    • 安卓的原生 SDK 使用谷歌威德文
    • 適用於 iOS 的原生 SDK 使用

    有關詳細信息，請參見本機播放器SDK的脫機播放概述。

    其他裝置

    MPEG-DASH 受到許多設備的支持, 包括安卓電視和鉻.桌面瀏覽器包含 DRM 的影片也可以在這些裝置上播放。

    許多消費性電子產品裝置都支援舊版 DRM-透過流暢的串流播放就緒

    註冊 DRM

    DRM 需要額外費用。請聯絡您的客戶經理，瞭解如何搭配 DRM 保護功能，以便您在任何地方播放內容的安全。

    蘋果公平遊戲

    FairPlay 串流是蘋果公司為 HLS 格式的視頻內容開發的 DRM 系統。

    若要啟用您的帳戶使用 FairPlay，您必須：

    1. 從Apple獲得FairPlay許可證
    2. 聯絡支援部門以設定您的視訊雲端帳戶

    已知問題

    • 對於在移動設備上播放 Fairplay，並使用外部屏幕，音頻將播放，但視頻只是一個黑色圖像。

      有關更多信息，請參見帶有Native Player SDK的內容安全性（DRM）

    谷歌威德文模塊化

    威德文模塊化是由谷歌開發，以保護 MPEG 短跑格式的視頻內容的系統。

    視訊雲端預設會使用布萊特灣的 Widevine 授權伺服器。

    微軟播放就緒

    播放就緒是微軟開發的 DRM 系統。播放準備有兩種形式：

    • 暢玩 -為平滑的流媒體內容提供DRM保護
    • PlayReady -為MPEG-DASH內容提供DRM保護

    視訊雲端使用布萊特灣的 PlayReady 授權伺服器。

    請注意，您與 Microsoft 的服務合約中所定義的「PlayReady」有所定義的並行限制。如需管理並行的資訊，請參閱此微軟文件。如果超過影片的並行限制，觀眾將無法播放超過最大數目。

    設定您的視訊雲端帳戶

    DRM 必須由 Brightcove 支援和您的客戶經理為您的帳戶設定。如需詳細資訊，請聯絡您的客戶經理
