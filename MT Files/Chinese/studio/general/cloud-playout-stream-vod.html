    使用雲端播放串流影片

    在本主題中，您將了解如何將 VOD 視頻和現場活動的集合作為直播線性頻道進行串流。

    雲端播放模組可讓廣播公司和媒體發行商使用 VOD 視頻和直播集合，然後將它們作為線性頻道進行串流。該頻道可以串流到布萊特灣直播，亞馬遜 S3 或指定的 RTMP 網址。

    注意：此功能目前處於有限的可用性。如需詳細資訊，請聯絡您的 Brightcove 客戶經理。

    要訪問Cloud Playout模塊，請點擊雲播出在導航菜單中。會顯示所有已建立的雲端播放頻道清單 (如果有的話)。

    該頁面將顯示以下信息：

    • 頻道名稱 -頻道名稱
    • 開始時間 -頻道的開始時間
    • 時間結束 -頻道的結束時間
    • 循環播放 -指示程序是否將循環
    • 目的地 -頻道的目標位置
    • 創建日期 -頻道創建的日期
    • 最近更新時間 -頻道的最後更新日期
    • 狀態 -頻道狀態
      • 草案 -頻道已創建但尚未激活
      • 創作 -頻道正在激活中
      • 預定時間 -頻道已啟用，可以進行直播
      • 開始 -頻道啟動過程開始流式傳輸
      • 正在運行 -頻道正在積極播放
      • 停止 -頻道流媒體正在停止
      • 閒 -頻道成功運行並已停止
      • 刪除中 -頻道正在刪除
      • 錯誤 -創建或流式傳輸頻道時發生錯誤

    建立通道

    若要建立新的雲端播放頻道，請按照下列步驟操作。

    1. 請點擊雲播出在導航標題中。
    2. 請點擊添新：
    3. 進入頻道身份證明細節：
      • 名稱 -頻道名稱。名稱必須包含英文字符，並且允許使用以下特殊字符：   _。 ：/ = +-  
      • 描述 -頻道說明
      • 圖片網址 -頻道圖片文件的網址；意味著是可以作為通道元數據的一部分檢索的通道圖像
    4. 進入頻道時間表細節。
      • 循環播放內容 -如果上，程序將循環播放，並且結束日期/時間將被啟用。該程序仍然可以在任何時間進行修改，並且更改將反映在當前程序和循環遵循。見創建一個循環程序下面的部分。
      • 廣告插入 -如果啟用，則允許在頻道播放期間展示廣告。如果選擇此選項，則不允許實時輸入，並且您只能輸出到Brightcove直播活動或亞馬遜S3桶。見啟用服務器端廣告下面的部分。
      • 開始日期/時間 -頻道開始播放的時間。所有資產的開始時間都將其持續時間加入此開始時間。只有在通道不處於執行狀態時，才能更新此時間。
      • 結束日期/時間 -如果未輸入任何值，並且循環播放內容如果未選擇，則播放結束後，流將結束。如果循環播放內容被選中，沒有結束日期/時間，播放流將一直播放到頻道停止為止。這個時間可以隨時更新。
    5. 如果要使用實時輸入，請設置現場輸入設為開啟，然後輸入Slate URL，除非您要為平板使用默認的Brightcove MP4。每當由於編碼器停止或某些技術問題阻止即時串流而停止即時輸入時，石板 MP4 就會播放。注意Slate URL必須是始終可用的靜態URL（不允許使用短暫的CDN URL）。
    6. 進入頻道目的地細節。雲端播放同時支援 2 個輸出。例如，您可以將 HLS 輸出至 S3 儲存貯體，並將 RTMP 摘要輸出至 Brightcove 即時頻道或第三方擷取點。注意Brightcove直播活動和RTMP不支持。
      • 優化顯示 -控制輸出的比特率和尺寸
        • 移動與網絡 -如果選中，則高清輸出（RTMP或HLS單比特率）
        • 智能電視 -如果選中，則為全高清輸出（RTMP或HLS單比特率）
      • 輸出區域 -將在其中創建通道的區域。Brightcove 建議您選擇最接近大多數觀眾預期的地理區域。
      • 輸出到 -選擇通道的輸出方法
        • Brightcove直播活動 -創建或使用以前創建的Brightcove直播頻道。創建新頻道時，將根據輸出區域被選中。
        • RTMP -選擇以將流輸出到RTMP或RTP攝取點。僅當以下情況下，RTP不可用於第三方攝入點廣告插入和Brightcove直播活動已啟用。選中後， RTMP網址字段將出現-輸入實時流媒體服務的RTMP輸入URL。
        • 亞馬遜S3桶 -選擇將HLS輸出推送到Amazon S3存儲桶。您將需要輸入「時段名稱」、「區域」、「存取金鑰」和「秘密金鑰」。存儲桶應該是公共的和可寫的。
    7. 請點擊建立頻道。將創建狀態為的新頻道草案。

    將影片和即時活動新增至頻道

    建立頻道後，畫面會顯示頻道設定：

    您可以編輯詳細資料，但下一個主要步驟是將影片和/或即時活動 (如果您啟用即時輸入功能) 新增至頻道。

    新增視訊 (VOD)

    1. 請點擊+添加視頻。（如果您在頻道列表頁面上，請單擊頻道名稱以打開頻道設置頁面，然後單擊+添加視頻）。
    2. 按一下您要加入頻道的影片旁邊的核取方塊。
    3. 按一下「新增視訊」。
    4. 按一下「儲存」。
    5. 頻道的程式清單將會出現：

      請注意，開始和結束時間是根據影片持續時間自動計算的。您可以在清單中拖放影片以重新排序，開始/結束時間也會自動調整。

    新增即時事件

    1. 點擊+添加現場活動按鈕。
    2. 在“添加直播事件”對話框中，輸入名稱，持續時間和描述對於現場活動：

      請注意，即時事件的最短持續時間為 5 分鐘，持續時間只能以 5 分鐘的間隔遞增或減少-如果需要，可以在文本框中排列編輯持續時間以調整時間粒度。即使目前正在播放，也可以隨時修改直播活動的持續時間。

    3. 新增即時活動之後，它們會與 VOD 項目一起出現在程式清單中：
    4. 您可以拖曳即時事件來重新排列清單，就像影片一樣。您也可以使用快速編輯按鈕可修改直播活動的持續時間。

    將視頻和現場活動添加到頻道後，單擊啟用激活頻道。頻道必須先啟動才能開始串流。見使用頻道動作以下部分以獲取更多信息。

    注意：若要變更程式中視訊的順序，請按一下視訊並將其拖曳至新位置。

    將視訊新增至頻道時，系統會建立系統產生的播放清單，並在「媒體」模組中顯示。

    我們強烈建議您在Cloud Playout模塊中對頻道進行任何更改，並且請勿以任何方式編輯生成的播放列表。

    取得 RTMP 攝取點

    對於您的即時編碼器，您需要頻道的 RTMP 攝取點。您可以在：頻道設置：

    視您使用的編碼器而定，您可能需要同時輸入 RTMP 內嵌點和串流名稱。頻道的流名稱為頻道編號，這也是提取點地址的最後一部分。

    編輯和刪除通道

    要編輯雲播出頻道，請單擊頻道名稱，然後單擊編輯。進行適當的編輯，然後單擊更新頻道。請注意，一旦建立通道，就無法編輯「目標」屬性。

    要刪除雲播出頻道，請單擊頻道名稱旁邊的複選框，然後單擊刪除。

    建立迴圈程式

    要創建循環的程序，請在創建頻道時設置循環播放內容設置為上。隨著影片新增到節目中，節目清單將會更新，以顯示一系列影片與相關的開始/結束時間，以填滿整個頻道排定的時間。當您變更現場活動的持續時間時，已經通過的事件的持續時間將保持不變，但任何未來的活動播放都會使用新的持續時間。

    在下面的例子中，頻道包括在一個循環的 VOD 和現場事件。

    使用頻道動作

    視頻道的狀態而定，頻道設定頁面上會顯示各種頻道動作按鈕。按一下頻道名稱即可存取頻道動作按鈕，即可檢視頻道設定。以下是可用的頻道動作按鈕：

    • 啟用 -激活頻道後即可進行流式傳輸。頻道必須先擁有影片才能啟用。
    • 現在播放 -點擊現在播放開始流式傳輸頻道。此選項僅適用於有即將開始時間的頻道。
    • 立即停止 -點擊立即停止停止流媒體頻道。這將停止頻道並結束直播。

    啟用伺服器端廣告

    您可以設定頻道，以便使用 Brightcove 的伺服器端廣告插入技術顯示伺服器端廣告。

    注意事項：
    • 使用廣告插入時，頻道只能輸出至 Bright湾直播事件。
    • 如果您啟用廣告插入功能，頻道中可能不會使用「即時輸入」功能。

    設定廣告標籤和填色板

    請確定您已設定為帳戶管理的一部分的廣告標籤和填色板。有關配置廣告代碼的信息，請參見配置服務器端廣告設置。填色板是指即時串流廣告片段有空白時，可以提供的內容。有關創建填充板的信息，請參見管理實時設置。

    注意：Cloud Playout將使用配置為以下內容的默認廣告板實時設置。

    建立預留位置廣告片段影片

    您必須指出哪些影片應當做「預留位置」視訊使用。「預留位置」影片會插入計畫清單中，指出應該刊登廣告和廣告長度。必須這麼做，因為在 Brightcove Live 中使用伺服器端廣告插入功能時，「預留位置」影片會被廣告內容取代。如果您未插入廣告「預留位置」影片，實際影片的部分將會被廣告內容取代。如果廣告持續時間短於傳回的廣告，SSAI 會顯示廣告板。如果廣告持續時間超過傳回的廣告，則廣告後面會顯示廣告板。

    您可能需要建立/上傳影片來源檔案，其影片長度等於您想要的廣告片段時間 (即 2 分鐘、4 分鐘等)。這些視頻的內容並不重要，因為它們實際上並不會作為節目的一部分播放。

    要將視頻標記為“佔位符”視頻，請使用“媒體”模塊編輯這些視頻的視頻屬性，並標記為cp-ad-holder。

    啟用頻道的伺服器端廣告

    建立頻道時，請設定廣告插入選擇上。

    注意：對於現有的頻道，無法變更「廣告插入」設定。

    將廣告預留位置插入程式

    您需要將廣告佔位符插入到應該顯示廣告的程序列表中。請依照下列步驟插入廣告預留位置。

    1. 按一下頻道名稱即可進行編輯。
    2. 在節目列表中，點擊視頻旁邊的複選框後您要播放的廣告。
    3. 請點擊插入廣告然後選擇一個廣告時長。廣告時長列表是根據所有廣告中斷“佔位符”視頻的長度（標記為cp-ad-holder）。
    4. 請點擊好。所選長度的「預留位置」視訊會插入到程式中選取的視訊之後。
      注意：作為“佔位符”插入的視頻將不會作為程序的一部分播放。此影片僅用於指出要在何處提出廣告請求和廣告長度。
    5. 請點擊保存將更改保存到程序中。

    選取廣告組態

    程序開始播放後，單擊在Brightcove Live中查看在“實時”模塊中查看頻道。播放器下方的下拉式清單可用來選取要使用的廣告設定。

    有關服務器端廣告插入的完整信息，請參見Video Cloud SSAI概述話題。

    產生雲端播放頻道的內嵌程式碼

    若要產生雲端播放頻道的內嵌程式碼，您將使用即時模組。有關使用實時模塊的完整詳細信息，請參見使用實時模塊創建和管理實時事件。

    有兩種類型的內嵌程式碼可供選擇：

    • 標準 -iframe 嵌入代碼。建議在大多數情況下使用。這種類型將播放器放在 < iframe > 標籤內。
    • 進階 -頁內嵌程式碼。允許播放器直接存在於 HTML 頁面中，而不是在 iframe 中。這可讓開發人員輕鬆存取播放程式以及相關屬性和活動的優點。

    您也可以取得已發佈播放程式的 URL。這對於在瀏覽器中檢視事件非常有用。

    如果要產生即時事件的內嵌程式碼，請依照下列步驟執行：

    1. 開啟即時模組。
    2. 找出頻道要串流的即時事件。通常，該事件將位於支持標籤，並命名為渠道除非您選擇要在創建頻道時流式傳輸的特定事件。
    3. 點擊事件名稱以打開控制室頁。
    4. 請點擊發布和嵌入>播放器配置在左側導航欄中。
      注意：如果頻道設定為啟用伺服器端廣告 (SSAI)，則必須使用 Brightcove 播放程式 6.18.3 及以上版本。只有 6.18.3 版及以上的玩家才會出現在下拉列表中。
    5. 按一下播放程式 URL 可在瀏覽器中預覽事件，或複製要使用的適當內嵌程式碼。

    如果程序是使用廣告插入選項設置為上，發布選項將包括選擇廣告配置下拉菜單中可以選擇要使用的廣告配置。

    檢視頻道輸出和分析

    頻道處於正在運行狀態，則可以查看渠道輸出和分析。點擊頻道頻道頁面以打開頻道詳細信息。

    注意：預覽影像會每三秒更新一次。

    請點擊在Brightcove Live中查看使用當前正在運行的程序打開“實時”模塊“控制室”頁面。程序中當前播放的視頻將以突出顯示現在玩旗幟。另請注意狀態右上角的通知，以及如果有實時供稿，它將變為綠色。否則，如果現場活動正在播放，則會顯示紅色；如果播放 VOD 內容，則顯示灰色。

    將顯示下列分析值：

    • 網絡輸入 -最後一分鐘每秒的平均字節數
    • 網絡輸出 -最後一分鐘每秒的平均字節數
    • 丟幀 -最後一分鐘丟幀的百分比

    使用電子程式指南

    API 可用來產生雲端播放頻道的電子程式指南。有關更多信息，請參見概述：EPG API話題。

    限制

    下列假設與限制適用。

    • 您的影片必須使用動態傳遞設定檔擷取，而且必須有 MP4 轉譯
    • 雲端播放支援的最大輸入位元速率和解析度為 20MBPS/1080p。這同時適用於 VOD 內容和即時輸入。
    • Brightcove 建議使用持續時間至少 2 分鐘的影片，以確保影片與電子節目指南之間的正確轉換正確性
    • 即時活動的最小長度為 5 分鐘。
    • 雲端播放中的每個直播活動都與視訊雲端遠端資產分享一對一關係。此遠端資產無法在其他通道中重複使用。如果從頻道中移除即時事件，相關的遠端資產將會自動刪除。刪除頻道後，所有相關的遠端資產也會自動刪除。
    • 當現場活動目前正在播放，但沒有使用中的摘要可用時，會循環播放石板內容，直到即時摘要恢復或事件結束為止。在板板 MP4 開始播放之前，會有大約 30 秒的延遲。
    • 如果將視頻雲帳戶配置為地理限制，則該頻道輸出區域應在與“視頻雲”地理位置限制政策兼容的區域中創建。
    • 為了獲得最佳體驗，布萊特灣建議您使用谷歌瀏覽器瀏覽器。
