    クラウドプレイアウトを使用してビデオをストリーミングする

    このトピックでは、VOD ビデオとライブイベントのコレクションをライブリニアチャンネルとしてストリーミングする方法を学びます。

    Cloud Playoutモジュールを使用すると、放送局とメディアパブリッシャーは、VODビデオとライブストリームのコレクションを使用して、それらをリニアチャンネルとしてストリーミングできます。チャネルは、Brightcove Live、Amazon S3、または指定されたRTMPURLにストリーミングできます。

    注：この機能は現在、制限付きです。詳しい情報はブライトコーブのアカウントマネージャーにお問い合わせください。

    クラウドプレイアウトモジュールにアクセスするには、クラウドプレイアウトナビゲーションメニューで。作成されたすべてのクラウドプレイアウトチャネルのリストが表示されます（存在する場合）。

    このページには、次の情報が表示されます。

    • チャネル名 -チャンネル名
    • 始まる時間 -チャンネルの開始時間
    • 終了時間 -チャネルの終了時間
    • ループ -プログラムがループするかどうかを示します
    • 先 -チャネルのターゲット宛先
    • 作成日 -チャネルが作成された日付
    • 最終更新 -チャンネルが最後に更新された日付
    • 状態 -チャネルステータス
      • ドラフト -チャネルは作成されましたが、アクティブ化されていません
      • 作成 -チャンネルはアクティブ化中です
      • スケジュール -チャンネルはアクティブで、ストリーミングの準備ができています
      • 起動 -ストリーミングを開始するためのチャネル開始プロセス
      • ランニング -チャンネルは活発にストリーミングしています
      • 停止 -チャンネルストリーミングが停止しています
      • アイドル -チャネルは正常に実行され、停止されました
      • 削除 -チャンネルが削除されています
      • エラー -チャネルの作成またはストリーミング中にエラーが発生しました

    チャネルの作成

    新しいクラウドプレイアウトチャネルを作成するには、次の手順に従います。

    1. クリッククラウドプレイアウトナビゲーションヘッダー内。
    2. クリック新しく追加する：
    3. チャネルに入る識別詳細：
      • 名前 -チャネルの名前。名前には英語の文字が必要であり、次の特殊文字を使用できます。   _。 ：/ = +-  
      • 説明 -チャンネルの説明
      • 画像のURL -チャネルの画像ファイルへのURL。チャネルのメタデータの一部として取得できるチャネル画像であることを意味します
    4. チャネルに入るスケジュール詳細。
      • ループプレイアウトコンテンツ -もしオン、プログラムはループで再生され、終了日時有効になります。プログラムはいつでも変更でき、変更は現在のプログラムとそれに続くループに反映されます。を参照してくださいループプログラムの作成以下のセクション。
      • 広告の挿入 -オンの場合、チャンネル再生中に広告を表示できます。このオプションを選択した場合、ライブ入力は許可されず、出力できるのはブライトコーブライブイベントまたはAmazonS3バケット。を参照してくださいサーバーサイド広告の有効化以下のセクション。
      • 開始日時 -チャンネルの再生が開始される時間。すべてのアセットの開始時間は、その期間をこの開始時間に追加することによって計算されます。この時間は、チャネルが実行状態にない場合にのみ更新できます。
      • 終了日時 -値が入力されていない場合ループプレイアウトコンテンツが選択されていない場合、プレイアウトが完了するとストリームは終了します。場合ループプレイアウトコンテンツが選択されていない終了日時、チャンネルが停止するまでプレイアウトがストリーミングされます。この時間はいつでも更新できます。
    5. ライブ入力を使用する場合は、ライブ入力スレートにデフォルトのBrightcoveMP4を使用する場合を除いて、[オン]をクリックし、スレートURLを入力します。スレートMP4は、エンコーダーが停止しているか、技術的な問題によりライブストリームがブロックされているために、ライブ入力が停止するたびに再生されます。ご了承くださいスレートURLは、常に利用可能な静的URLである必要があります（短命のCDN URLは許可されていません）。
    6. チャネルに入る先詳細。Cloud Playoutは、同時に2つの出力をサポートします。たとえば、HLSをS3バケットに出力し、RTMPフィードをBrightcoveライブチャネルまたはサードパーティの取り込みポイントに出力できます。ご了承くださいブライトコーブライブイベントそしてRTMPサポートされていません。
      • ディスプレイの最適化 -出力のビットレートと寸法を制御します
        • モバイル＆ウェブ -チェックした場合、HD出力（RTMPまたはHLSシングルビットレート）
        • スマートテレビ -チェックした場合、フルHD出力（RTMPまたはHLSシングルビットレート）
      • 出力領域 -チャネルが作成される地域。Brightcoveは、大多数の視聴者が予想される場所に地理的に最も近い地域を選択することをお勧めします。
      • に出力 -チャネルの出力方法を選択します
        • ブライトコーブライブイベント -以前に作成したBrightcoveライブチャネルを作成または使用します。新しいチャネルを作成するとき、チャネルの領域はに基づいて最適化されます出力領域それが選択されました。
        • RTMP -ストリームをRTMPまたはRTP取り込みポイントに出力する場合に選択します。RTPは、サードパーティの取り込みポイントでは使用できません。広告の挿入そしてブライトコーブライブイベント有効になっています。選択すると、 RTMPURLフィールドが表示されます-ライブストリーミングサービスのRTMP入力URLを入力します。
        • AmazonS3バケット -HLS出力をAmazonS3バケットにプッシュする場合に選択します。バケット名、地域、アクセスキー、シークレットキーを入力する必要があります。バケットは公開され、書き込み可能である必要があります。
    7. クリックチャネルを作成する。新しいチャンネルは、次のステータスで作成されますドラフト。

    チャンネルへの動画やライブイベントの追加

    チャンネルを作成すると、画面にチャンネル設定が表示されます。

    詳細は編集できますが、次の主なステップは、動画やライブイベント（ライブ入力を有効にしている場合）をチャンネルに追加することです。

    ビデオを追加（VOD）

    1. クリック+ビデオを追加。（チャンネルリストページを表示している場合は、チャンネル名をクリックしてチャンネル設定ページを開き、[ +ビデオを追加）。
    2. チャンネルに追加する動画の横にあるチェックボックスをクリックします。
    3. [ ビデオを追加 ] をクリックします。
    4. [保存]をクリックします。
    5. チャネルのプログラムリストが表示されます。

      開始時間と終了時間は、ビデオの長さから自動的に計算されることに注意してください。リスト内のビデオをドラッグアンドドロップして並べ替えることができ、開始/終了時間が自動的に調整されます。

    ライブイベントを追加する

    1. クリック+ライブイベントを追加ボタン。
    2. [ライブイベントの追加]ダイアログで、名前、デュレーション、および説明ライブイベントの場合：

      ライブイベントの最小期間は5分であり、期間は5分間隔でのみインクリメントまたはデクリメントできることに注意してください。必要に応じて、テキストボックスで期間をインラインで編集して、時間の粒度を調整できます。ライブイベントの期間は、現在再生中であってもいつでも変更できます。

    3. ライブイベントを追加すると、VODアイテムとともにプログラムリストに表示されます。
    4. ビデオと同じように、ライブイベントをドラッグしてリストを並べ替えることができます。また、使用することができますクイック編集ライブイベントの期間を変更するボタン。

    動画とライブイベントがチャンネルに追加されたら、をクリックしますアクティベートチャネルをアクティブにします。ストリーミングを開始する前に、チャネルをアクティブ化する必要があります。を参照してくださいチャネルアクションの使用詳細については、以下のセクションを参照してください。

    注：プログラム内のビデオの順序を変更するには、ビデオをクリックして新しい位置にドラッグします。

    動画がチャンネルに追加されると、システムで生成された再生リストが作成され、メディアモジュールに表示されます。

    クラウドプレイアウトモジュールでチャンネルに変更を加えることを強くお勧めします。生成されたプレイリストは一切編集しないでください。

    RTMP取り込みポイントの取得

    ライブエンコーダーの場合、チャネルのRTMP取り込みポイントが必要になります。あなたはそれを見つけることができます：チャンネル設定：

    使用しているエンコーダーによっては、RTMP取り込みポイントとストリーム名の両方を入力する必要がある場合があります。チャネルのストリーム名はチャネルID、これは取り込みポイントアドレスの最後の部分でもあります。

    OBSソフトウェアエンコーダーを使用している場合は、を指定する必要があることに注意してくださいカスタム...サービス、および入力する必要がありますサーバそしてストリームキー。ザ・サーバの最初の部分になります取り込みポイント、 そしてそのスチームキー以下の例に示すように、残りの部分になります。

    チャンネルの編集と削除

    Cloud Playoutチャネルを編集するには、チャネル名をクリックしてから、編集。適切な編集を行ってから、チャネルの更新。チャネルが作成されると、Destinationプロパティは編集できないことに注意してください。

    Cloud Playoutチャネルを削除するには、チャネル名の横にあるチェックボックスをクリックしてから、削除。

    ループプログラムの作成

    ループするプログラムを作成するには、チャンネルを作成するときに、ループプレイアウトコンテンツに設定オン。ビデオがプログラムに追加されると、プログラムリストが更新され、チャンネルのスケジュールされた時間全体を満たすために、開始/終了時間が関連付けられた一連のビデオが表示されます。ライブイベントの期間を変更すると、すでに経過したイベントの期間は変更されませんが、イベントの今後の再生では新しい期間が使用されます。

    チャネルアクションの使用

    チャネルのステータスに応じて、チャネル設定ページでさまざまなチャネルアクションボタンを使用できます。チャネル名をクリックしてチャネル設定を表示することにより、チャネルアクションボタンにアクセスします。次のチャネルアクションボタンを使用できます。

    • アクティベート -チャネルをアクティブ化すると、ストリーミングに使用できるようになります。チャンネルをアクティブにするには、動画が必要です。
    • 今すぐプレイ -クリック今すぐプレイチャンネルのストリーミングを開始します。このオプションは、開始時刻が近づいているチャンネルでのみ使用できます。
    • 今すぐ停止 -クリック今すぐ停止ストリーミングチャネルを停止します。これにより、チャンネルが停止し、ライブストリームが終了します。

    サーバーサイド広告の有効化

    プレースホルダー広告休憩動画の作成

    どの動画を「プレースホルダー」動画として使用するかを指定する必要があります。「プレースホルダー」動画がプログラムリストに挿入され、広告を配信する必要があることと、広告の長さが示されます。Brightcove Liveでサーバー側の広告挿入を使用すると、「プレースホルダー」ビデオが広告コンテンツに置き換えられるため、これを行う必要があります。広告の「プレースホルダー」動画を挿入しない場合、実際の動画の一部が広告コンテンツに置き換えられます。広告の継続時間が返された広告よりも短い場合、SSAIは広告のスレートを表示します。広告の継続時間が返された広告よりも長い場合、広告が表示され、その後に広告スレートが表示されます。

    必要な広告休憩時間（2分、4分など）に等しいビデオ長のビデオソースファイルを作成/アップロードする必要がある場合があります。これらのビデオの内容は、実際にはプログラムの一部として再生されないため、重要ではありません。

    ビデオを「プレースホルダー」ビデオとしてマークするには、Mediaモジュールを使用してこれらのビデオのビデオプロパティを編集し、タグを付けますcp-ad-holder。

    チャネルのサーバー側広告の有効化

    チャネルを作成するときは、広告の挿入オプションオン。

    注：既存のチャネルの場合、広告挿入設定は変更できません。

    プログラムへの広告プレースホルダーの挿入

    広告を表示するプログラムリストに広告プレースホルダーを挿入する必要があります。次の手順に従って、広告プレースホルダーを挿入します。

    1. チャンネル名をクリックして編集します。
    2. プログラムリストで、ビデオの横にあるチェックボックスをクリックします後広告を再生したいもの。
    3. クリック広告を挿入を選択します広告期間。広告期間のリストは、すべての広告休憩の「プレースホルダー」動画の長さに基づいて生成されます（タグ付き）cp-ad-holder）。
    4. クリックOK。選択した長さの「プレースホルダー」ビデオは、選択したビデオの後にプログラムに挿入されます。
      注：「プレースホルダー」として挿入されるビデオしないプログラムの一部として再生されます。この動画は、広告リクエストが行われる場所と広告の長さを示すためにのみ使用されます。
    5. クリック保存するプログラムへの変更を保存します。

    CloudPlayoutチャネルの埋め込みコードを生成する

    Cloud Playoutチャネルの埋め込みコードを生成するには、Liveモジュールを使用します。Liveモジュールの使用の詳細については、を参照してください。ライブモジュールを使用したライブイベントの作成と管理。

    次の 2 種類の埋め込みコードを使用できます。

    • 標準 -iframe埋め込みコード。ほとんど場合に推奨されます。このタイプは、プレーヤーを < iframe > タグ内に配置します。
    • 詳細 -ページ内埋め込みコード。プレーヤーは、iframeではなく、HTMLページに直接存在することを許可します。これによって、開発者はプレーヤーおよび関連するプロパティやイベントにアクセスしやすくなります。

    また、公開されたプレーヤーへの URL を取得する機能もあります。これは、ブラウザーでイベントを表示する場合に便利です。

    ライブイベントの埋め込みコードを生成するには、次の手順に従います。

    1. ライブモジュールを開きます。
    2. チャンネルがストリーミングされるライブイベントを見つけます。通常、イベントは次の場所にあります。待機するタブと名前を持っているチャネルチャネルの作成時にストリーミングする特定のイベントを選択した場合を除きます。
    3. イベント名をクリックして、制御室ページ。
    4. 左側のナビゲーションで [ > プレーヤー設定の公開と埋め込み ] をクリックします。
      注：チャネルがサーバー側広告（SSAI）を有効にするように構成されている場合は、BrightcovePlayerバージョン6.18.3以降を使用する必要があります。ドロップダウンリストには、バージョン 6.18.3 以上のプレイヤーのみが表示されます。
    5. プレーヤーの URL をクリックして、ブラウザーでイベントをプレビューするか、使用する適切な埋め込みコードをコピーします。

    プログラムがで作成された場合広告の挿入オプションをに設定オン、公開オプションには、広告設定を選択使用する広告構成を選択できるドロップダウン。

    制約事項

    以下の仮定と制限が適用されます。

    • 動画は動的配信プロファイルを使用して取り込む必要があり、MP4レンディションが必要です
    • CloudPlayoutでサポートされる最大入力ビットレートと解像度は20MBPS / 1080pです。これは、VODコンテンツとライブ入力の両方に適用されます。
    • Brightcoveは、ビデオ間の正しい移行と電子番組ガイドの精度を確保するために、少なくとも2分間のビデオを使用することをお勧めします。
    • ライブイベントの最小時間は5分です。
    • Cloud Playoutのすべてのライブイベントは、Video Cloudリモートアセットと1：1の関係を共有します。このリモートアセットを他のチャネルで再利用することはできません。ライブイベントがチャンネルから削除されると、関連するリモートアセットは自動的に削除されます。チャネルが削除されると、関連するリモートアセットも自動的に削除されます。
    • ライブイベントが現在再生されているが、アクティブなフィードが利用できない場合、ライブフィードが再開されるか、イベントが終了するまで、スレートコンテンツがループで再生されます。スレートMP4の再生が開始されるまでに約30秒の遅延があります。
    • Video Cloudアカウントが地理的制限用に構成されている場合、チャネル出力領域 VideoCloudの地理的制限ポリシーと互換性のあるリージョンで作成する必要があります。
    • 最高のエクスペリエンスを得るには、BrightcoveはGoogleChromeブラウザの使用をお勧めします。
