    Paramétrage du mode de mise à jour du lecteur et de sa version

    Cette rubrique présente les paramètres Player Update Mode et Version, configurables dans le module Players.
     

    Un avantage majeur du Brightcove Player est sa capacité à être personnalisé. La personnalisation permet des éléments et des couleurs de l'interface utilisateur spécifiques à la marque, des éléments interactifs, l'intégration avec n'importe quel système de publicité ou d'analyse tiers, et bien plus encore. Mais la personnalisation peut également rendre le joueur différent des tests de version Brightcove dans le cadre du processus régulier d'assurance qualité et de publication. Pour les clients qui ont personnalisé leurs joueurs et qui veulent s'assurer que les nouvelles versions ne brisent pas le comportement du joueur ou du site, Brightcove offre aux clients la possibilité de se désinscrire des mises à jour automatiques régulières des versions.

    Dans le cadre des paramètres du lecteur, le Mode de mise à jour et Version du joueur peut être configuré. Le paramètre Update Mode contrôle la gestion des mises à jour du lecteur. Le paramètre Player Version permet de définir la version du lecteur. Être capable de contrôler le Mode de mise à jour et Version du joueur offre ces avantages:

    • Les éditeurs peuvent tester des versions préliminaires de lecteurs sans utiliser l'API
    • Les éditeurs peuvent contrôler les mises à jour des lecteurs

    Procédure de mise à jour

    Brightcove s'engage à fournir les dernières fonctionnalités et corrections de bogues aux clients le plus rapidement possible. Dans le cadre de la discipline de livraison agile, Brightcove s'efforce de diviser les améliorations des fonctionnalités et les corrections de bogues en mises à jour incrémentielles qui peuvent être publiées toutes les deux semaines. Le processus de publication comprend des tests unitaires, des tests automatisés d'intégration et de régression, ainsi que des tests manuels sur un échantillon représentatif de sites clients - où la page est modifiée localement dans l'environnement de test. Une fois qu'une nouvelle version a passé tous les tests, Brightcove la publie en pré-version. Cela permet aux clients de choisir explicitement cette version à l'aide de l'API de gestion du lecteur ou de la liste déroulante du sélecteur de version du lecteur dans le module Lecteur. À ce stade, aucun joueur n'est mis à jour sauf ceux qui ont explicitement choisi la nouvelle version. Si vous utilisez le mode de mise à jour automatique par défaut et que vous souhaitez vous assurer que votre site fonctionnera correctement avec la prochaine mise à jour globale, vous pouvez créer un lecteur de test spécial avec la nouvelle version et l'essayer sur une version de test ou de mise en scène de votre site pendant la période de pré-sortie.

    Mises à jour majeures

    Les mises à jour de version majeures sont indiquées par une modification du premier numéro de l'identifiant de version, par exemple la version majeure de 6.16.3 est 6. Une modification de la version principale indique un changement significatif dans le fonctionnement d'une API, ce qui peut rompre une intégration existante. Pour cette raison, et afin d'éviter des problèmes avec votre site, les mises à jour automatiques ne seront pas effectuées pour les mises à jour de version majeures. Cela signifie que votre joueur sera effectivement verrouillé à la version finale d'une version majeure, même si vous avez sélectionné le mode de mise à jour automatique.

    Version gérée

    Brightcove permet généralement aux clients d'essayer une version préliminaire pendant environ une semaine avant le début d'une version gérée. La version gérée est le processus d'exécution d'un test A/B contrôlé sur les sites Web de production. Le système de publication gérée permet de charger la nouvelle version du lecteur sur un petit pourcentage de chargement de pages pendant une courte période - généralement quelques heures. Pendant ce temps, Brightcove collecte des données afin de comparer les performances de l'ancienne et de la nouvelle version. La répartition du nombre d'erreurs, du nombre de minutes consultées, du nombre de démarrages d'annonces et de plusieurs autres mesures sont tous examinés. Brightcove examine les dénombrements de distribution et les résumés des données, mais aussi sur les sites aberrantes, qui présentent des différences supérieures à la moyenne dans les mesures suivies. Après un examen attentif de toutes les données, la mise à jour est validée ou annulée. S'il est décidé de l'annuler, de petites mises à jour peuvent être effectuées pour résoudre un problème identifié, puis la version gérée est réexécutée.

    Définition du mode de mise à jour du lecteur

    Pour modifier le lecteur Mode de mise à jour , Suivez ces étapes.

    1. Ouvrez le module Players.
    2. Cliquez sur un lecteur pour accéder aux propriétés.
    3. Cliquez sur Informations sur le lecteur dans le menu de navigation de gauche.
    4. Sélectionnez un mode de mise à jour. Tous les deux Automatique et Manuel les modes sont disponibles.

      Mises à jour automatiques

      Automatique est le mode par défaut. La sélection du mode Automatic permet la mise à jour du lecteur chaque fois que Brightcove définit une version de lecteur comme « version actuelle par défaut ». Quand Automatique est sélectionné, le Version du joueur les options sont Toujours la dernière version et Toujours la dernière version préliminaire.

      Remarque : Pour plus d'informations sur la dernière version préliminaire du lecteur, consultez le Pré-version du lecteur Brightcove document.

      Mises à jour manuelles

      Dans Manuel mode, les joueurs ne seront pas automatiquement mis à jour. Ce procédé est également appelé « verrouillage » du lecteur. Lorsque cette option est choisie pour un joueur, ce joueur ne fera pas partie du processus de sortie géré et restera à la version choisie. Vous devriez réfléchir attentivement avant de choisir ce mode car cela signifie que vous n'obtiendrez plus de corrections automatiques de bogues ou d'améliorations de fonctionnalités. À un moment donné, les anciennes versions des joueurs ne seront plus prises en charge. Nous vous recommandons ainsi d'utiliser le mode Manual uniquement en tant que solution temporaire en attendant de résoudre les problèmes entre la version actuelle du lecteur et la dernière version. Quand Manuel est sélectionné, une version spécifique du lecteur peut être sélectionnée à l'aide du Version du joueur la liste déroulante.

      Raisons de choisir les mises à jour manu

      • Configuration publicitaire avancée - Si vous utilisez les plugins IMA ou FreeWheel et que vous avez rendu impossible toute configuration ou extension de comportement publicitaire via l'interface utilisateur de Studio, y compris la définition de paramètres de configuration personnalisés ou l'interaction avec le plugin à l'aide de JavaScript dans un plugin personnalisé ou une logique de page, cela peut être une raison de verrouiller votre lecteur.
      • Plugins tiers - Si vous utilisez une analyse ou un autre plugin fourni par un tiers, reconnaissez que ces plugins ne sont pas maintenus par Brightcove et peuvent être sensibles à des changements mineurs dans le comportement des événements JavaScript du lecteur. Brightcove recommande aux utilisateurs de ces plugins de verrouiller leurs joueurs.
      • Changements CSS importants - Des exemples de changements CSS importants incluent: la modification de la mise en page, le dimensionnement, le positionnement des éléments ou les changements qui ne subiraient pas de changements de mise en page, de dimensionnement ou de positionnement après une mise à jour. Exemples de modifications CSS sécurisées : modification du texte, des couleurs d'arrière-plan ou modification des polices sans affecter la mise en page.
      Remarque : Lorsqu'un joueur est verrouillé sur une version, les plugins associés à ce joueur ne seront PAS automatiquement mis à jour, et également verrouillés sur la version actuelle pour chaque plugin.
    5. Cliquez sur Save.

    Assurez-vous de publier les modifications apportées au lecteur.

    Définition de la version du lecteur

    Le paramètre Player Version permet de définir la version du lecteur. Si le mode de mise à jour est Manuel et qu'une nouvelle version du lecteur est disponible, un message s'affiche.

    Pour modifier le Version du joueur , Suivez ces étapes.

    1. Ouvrez le module Players.
    2. Cliquez sur un lecteur pour accéder aux propriétés.
    3. Cliquez sur Informations sur le lecteur dans le menu de navigation de gauche.
    4. Si la Mode de mise à jour est réglé sur Automatique , la Version du joueur peut être:
      • Toujours la dernière version - Cette option entraînera la mise à jour automatique de la version du lecteur vers la dernière version à mesure que de nouvelles versions seront disponibles
      • Toujours la dernière version préliminaire - Cette option entraînera la mise à jour automatique de la version du lecteur vers la dernière version préliminaire (ou la version finale s'il n'y a pas de version préliminaire disponible) lorsque de nouvelles versions préliminaires sont disponibles.
    5. Si la Mode de mise à jour est réglé sur Manuel , sélectionnez un Version du joueur.
    6. Cliquez sur Save.

    Assurez-vous de publier les modifications apportées au lecteur.

    Support technique

    Le support technique de Brightcove fournira une assistance et des conseils à tout client ayant un contrat d'assistance en cours utilisant n'importe quelle version du Brightcove Player. Cependant, les corrections de bogues ne seront appliquées qu'à la dernière révision d'une version majeure. Après la publication d'une version majeure, certaines corrections de bogues seront appliquées à la dernière révision de la version majeure précédente pour une durée maximale de 6 mois.

    Brightcove espère que ces options vous fourniront les outils dont vous avez besoin pour assurer la meilleure expérience de joueur possible sur votre site tout en offrant le maximum d'options de personnalisation.
