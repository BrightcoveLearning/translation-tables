    Référence pour les rapports de performances

    Cette rubrique vous expose les détails du rapport des performances du compte qui est fourni dans le module de statistiques.
     

    Les données d'analyse sont envoyées automatiquement par les lecteurs Brightcove, y compris celles fournies par les SDK Native Player. Si vous n'utilisez pas Brightcove Player pour diffuser des vidéos Video Cloud, vous devez instrumenter le lecteur que vous utilisez pour envoyer les données au collecteur de données. Voir le Aperçu: API de collecte de données sujet pour plus de détails.

    Pour accéder au rapport des performances, ouvrez le module de statistiques et sélectionnez Performance dans le volet de gauche. Le rapport présente un graphique qui affiche le nombre total de visionnages par jour pour la période choisie. Vous pouvez choisir une autre mesure qui vous intéresse dans la liste déroulante :

    • Vues vidéo - Le nombre de fois où une vidéo a commencé à jouer, enregistré au démarrage du flux (n'inclut pas le rembobinage ou la relecture); ce n'est pas une mesure des téléspectateurs individuels; pour les vidéos avec des publicités pré-roll, une vue n'est pas enregistrée tant que la vidéo ne démarre pas
    • Video Impressions : le nombre de fois où une vidéo a été chargée et prête pour interaction.
    • Play Rate : le pourcentage de vidéos qui ont été chargées et vraiment lues (le nombre de visionnages divisé par le nombre d'affichages).
    • Viewed Minutes : le nombre de minutes de visionnage de la vidéo.
    • Engagement Score : le pourcentage moyen de la vidéo vue par visionnage (calculé en additionnant tous les pourcentages regardés et en divisant le résultat par le nombre de visionnages de la vidéo).
    • Daily Unique Viewers : calculé en fonction de l'adresse IP et de l'utilisateur-agent des visiteurs. Brightcove a choisi de ne pas utiliser de cookies tiers car ils sont de plus en plus fréquemment bloqués par les navigateurs. Daily Unique Viewers (Nombre de visiteurs uniques quotidiens) n'est disponible qu'à l'échelle du compte ; ce paramètre n'est pas présent lors d'un filtrage par vidéo (ou par quoi que ce soit d'autre). Brightcove permet également de déterminer les utilisateurs uniques par rapport à une valeur de votre choix au lieu de leur adresse IP. Pour plus d'informations, consultez le document développeur Suivi des événements statistiques.
    • Bande passante vidéo - Inclut la bande passante pour les vidéos uniquement. Les données sur la bande passante ne sont disponibles qu'au bout de trois à cinq jours.
    • Bande passante totale - Inclut la bande passante pour les vidéos, les images et les charges de lecteur. Les données sur la bande passante ne sont disponibles qu'au bout de trois à cinq jours.

    Si votre compte est activé pour Brightcove Live, vous avez la possibilité d'afficher les données de performances de toutes les vidéos, ressources VOD ou actifs en direct.

    Remarques :
    • Les rapports de statistiques ne comprennent pas les statistiques des vidéos supprimées de votre compte. Vous ne pouvez pas rechercher des vidéos supprimées dans le module de statistiques pour consulter un rapport sur une vidéo supprimée particulière. Pour obtenir un tel rapport, vous devrez connaître l'ID vidéo de la vidéo supprimée et utiliser le API Analytics pour générer le rapport.
    • Il faut jusqu'à 3 jours pour que les données sur les visionneuses uniques quotidiennes et les données de bande passante vidéo soient disponibles.
    • Une vue vidéo n'est pas la même que les vues à x% dans le rapport d'engagement. Un visionnage vidéo est compté lors de la première lecture de la vidéo. Les visionnages à 1 % sont enregistrés lorsque le premier % est regardé. La différence ici est que si le premier pourcentage est regardé plusieurs fois (si la vidéo est réglée sur la relecture automatique, par exemple), le nombre augmente, alors qu'une vue vidéo est comptée lorsque la vidéo commence à lire mais n'est pas comptée sur une relecture (sauf si l'utilisateur recharge la page ou regarde une autre vidéo dans la liste de lecture, etc.).
    • Les demandes de rapport simultanées dans le même compte s'exécuteront en série. Si les rapports ont de grandes plages de dates et que de nombreux champs sont sélectionnés, il peut y avoir un délai avant que tous les résultats ne soient renvoyés. Une bonne alternative consiste à planifier des rapports volumineux.
    • Avec Expériences sur la page , même si les vidéos semblent être chargées dans un lecteur, les vidéos ne sont pas chargées tant que le spectateur n'a pas cliqué sur la superposition du bouton de lecture. Par conséquent, les impressions vidéo ne seront pas enregistrées par le module Analytics tant qu'un spectateur n'a pas lu une vidéo.

    Le commutateur de rapports situé sous le graphique peut être utilisé pour passer rapidement à un autre rapport.

    La liste déroulante à droite du commutateur de rapports peut être utilisée pour sélectionner une dimension de rapport. Les dimensions suivantes sont disponibles avec le rapport sur les performances :

    • Vidéo
    • Joueur
    • Source de trafic
    • Afficher le domaine
    • Pays
    • Appareil
    • Type d'appareil
    • Système d'exploitation de l'appareil
    • Fabricant de l'appareil
    • Type de navigateur
    • Jour

    Notez que les dimensions disponibles changeront en fonction du rapport sélectionné. Au bas du rapport, vous trouverez des contrôles qui vous permettent de sélectionner le nombre de lignes à afficher et de naviguer vers d'autres pages.

    Compare to previous period

    Cocher la case Compare to previous period ajoute un graphique à lignes segmentées superposé qui affiche les données de la période précédente.

    La période précédente est fonction de la période de dates sélectionnée dans le rapport. Par exemple, si vous sélectionnez une plage de dates qui correspond aux sept derniers jours, la période précédente couvrira les 14 jours qui précèdent ceux-ci. Il est conseillé de sélectionner des plages de dates qui correspondent à des semaines entières si vous désirez comparer des tendances sur plusieurs semaines. Par exemple, si vous sélectionnez une plage qui commence un lundi, la date de fin devrait être un dimanche. Si la plage commence un vendredi, faites-la conclure un jeudi. Vous pouvez également survoler les barres et la ligne segmentée avec votre curseur pour afficher la date et les valeurs correspondantes.

    Détails des rapports de performances

    La première colonne qui s'affiche dans le corps du rapport varie selon l'échelle sélectionnée (nom de la vidéo, nom du lecteur, pays, type de dispositif, etc.). Tous les rapports affichent les données suivantes :

    • Video Views : le nombre de fois où la lecture de la vidéo a été lancée. Cette statistique est recueillie au lancement du flux et ne comprend pas les retours en arrière ou les relectures. Il ne s'agit pas d'une mesure par visiteur individuel.
    • Video Impressions : le nombre de fois où une vidéo a été chargée dans un lecteur et a été prête pour interaction.
    • Play Rate : le pourcentage d'utilisateurs qui ont lu la vidéo une fois celle-ci chargée (il s'agit du nombre de visionnages de la vidéo divisé par les affichages vidéo).
    • Viewed Minutes : le nombre total de minutes de visionnage de la vidéo.
    • Engagement Score : le pourcentage moyen de la vidéo vue par visionnage (calculé en additionnant tous les pourcentages regardés et en divisant le résultat par le nombre de visionnages de la vidéo).

    Traffic Sources

    Lorsque vous consultez les performances par sources du trafic, les sources possibles sont les suivantes :

    • Referral : l'utilisateur a cliqué sur un lien vers la vidéo.
    • Direct : l'utilisateur a saisi le lien à la main ou le référent n'a pas pu être identifié.
    • Secure search : l'utilisateur s'est connecté à un compte Google, a lancé une recherche et a cliqué sur un résultat (standard ou parrainé ; les informations de recherche ne sont pas précisées dans ce cas).
    • Organic search : l'utilisateur a lancé une recherche et a cliqué sur un résultat pour afficher la vidéo.
    • Paid search : l'utilisateur a lancé une recherche et a cliqué sur un résultat parrainé pour afficher la vidéo.

    Device Types

    Plusieurs rapports de performances permettent d'afficher les données par type de dispositif. Voici la liste des dispositifs pris en charge :

    • Desktop : comprend les ordinateurs de bureau et les ordinateurs portables
    • Tablet : comprend les iPad, les tablettes sous Android, etc.
    • Mobile : téléphones portables
    • Connected TV/Game Console/Companion Device : comprend les TV connectées, les box de salon et les consoles de jeu
    • Other : pour les ressources qui ne rentrent dans aucune catégorie ci-dessus

    Exportation des données de rapports

    Les données de rapports peuvent être téléchargées en tant que valeurs séparées par des virgules et des formats Excel. Notez que la fonction d'exportation enregistre les données directement à partir d'un Analytics API appel. Les données exportées ne sont pas modifiées comme les données de l'interface utilisateur. Elles contiennent actuellement toutes les données inhérentes au rapport et pas seulement le sous-ensemble de champs de l'interface utilisateur. Voici quelques-unes des différences entre ce qui s'affiche dans l'interface utilisateur et dans les données exportées :

    • Des unités différentes (minutes contre secondes)
    • Des nombres arrondis
    • L'exportation de données contient les mesures d'engagement et de performances, peu importe où vous êtes dans l'interface utilisateur.
    • L'entrée Other/Third Party sera exportée sans ID de vidéo ou nom de vidéo associés.

    Obtenir toutes les données au format de base depuis l'API vous permet de manipuler les données via d'autres méthodes que celles de l'interface utilisateur.

    Les pages des rapports sont dans un format d'impression. Imprimer un rapport ou l'enregistrer en PDF donnera donc des pages avec une disposition parfaite.

    Affichage des statistiques vidéo

    Lorsque l'échelle Video est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport présente le nom de la vidéo, l'ID de vidéo, la durée et les détails des performances pour chaque vidéo.

    Cliquez sur le lien d'une vidéo pour obtenir les performances spécifiques de cette vidéo.

    Remarque : Si une vidéo qui ne se trouve pas dans votre bibliothèque Video Cloud est lue dans un lecteur Video Cloud (par exemple une ressource distante), Autre / Tiers sera affiché sans ID vidéo associé.

    Affichage des statistiques des lecteurs

    Lorsque l'échelle Player est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport présente les données de performances pour chaque lecteur.

    Cliquez sur le lien d'un lecteur pour obtenir ses performances spécifiques.

    Remarque : avant février 2016, les lecteurs utilisés dans les applications et conçus avec le SDK de lecteur Brightcove pour iOS ou Android portaient le nom de lecteur Custom ou Native Playback. À partir de février 2016, ces lecteurs sont désormais identifiés grâce aux valeurs de source et de destination précisées dans l'application. Pour plus d’informations, consultez la documentation du SDK iOS ou Android.

    Affichage des statistiques de source de trafic

    Lorsque l'échelle Source of Traffic est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport affichera les données de performance pour chaque source de trafic. Seules les catégories ayant des données pour la période sélectionnée seront affichées.

    Cliquez sur le lien d'une des sources pour afficher les performances de celle-ci.

    Affichage des statistiques des domaines d'affichage

    Lorsque l'échelle Display Domain est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport affichera les statistiques des domaines d'affichage. Le domaine d'affichage est le domaine sur lequel se trouvait le visiteur lorsqu'il a regardé la vidéo.

    En cliquant sur le lien de n'importe quel domaine d'affichage, vous verrez les URL d'affichage de ces domaines.

    Remarque : si le lecteur se trouve dans une iframe, le domaine d'affichage consigné sera l'URL de la page dans l'iframe.

    Affichage des statistiques par pays

    Lorsque l'échelle Country est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport affichera les statistiques pour chaque pays où la vidéo a été vue. Seuls les pays ayant des données pour la période sélectionnée seront affichés.

    Remarque : Brightcove utilise un service tiers pour nous fournir l'adresse IP et les informations de localisation. S'il n'y a pas d'entrée pour l'adresse IP en question, nous ne serons pas en mesure d'obtenir les informations sur le pays et Inconnue apparaîtra.

    Cliquez sur le lien de l'un des pays pour afficher les performances par région ou ville pour le pays sélectionné, si vous consultez les analyses pour le Compte , pas une vidéo individuelle.

    Affichage des statistiques par dispositif

    Lorsque l'échelle Device est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport affichera les données de performances pour chaque combinaison type de dispositif - système d'exploitation qui a été utilisée pour afficher une vidéo. Seuls les dispositifs ayant des données pour la période sélectionnée seront affichés.

    Cliquez sur un type de dispositif pour obtenir les performances par système d'exploitation.

    Remarque : le module de statistiques détermine le dispositif utilisé selon le header utilisateur-agent. Si quelqu'un utilise un dispositif avec des headers usurpés, les données seront collectées selon le header déclaré. Le module de statistiques n'offre aucune mesure de protection contre l'usurpation d'utilisateur-agent.

    Affichage des statistiques par type de dispositif

    Lorsque l'échelle Device Type est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport affichera les détails des performances par type d'appareil. Seules les catégories ayant des données pour la période sélectionnée seront affichées.

    Cliquez sur un type de dispositif pour obtenir les performances par système d'exploitation.

    Affichage des statistiques par système d'exploitation

    Lorsque l'échelle Device OS est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport affichera les données de performance pour chaque système d'exploitation de l'appareil. Seules les catégories ayant des données pour la période sélectionnée seront affichées.

    Affichage des analyses du fabricant de périphériques

    Quand le Fabricant de l'appareil est sélectionnée, un graphique affichera le nombre de vues vidéo par jour pour la période sélectionnée. Le corps du rapport affichera les données de performance pour chaque fabricant de l'appareil. Seuls les dispositifs ayant des données pour la période sélectionnée seront affichés.

    Affichage de l'analyse des types de navigateur

    Quand le Type de navigateur est sélectionnée, un graphique affichera le nombre de vues vidéo par jour pour la période sélectionnée. Le corps du rapport affichera les données de performance pour chaque type de navigateur. Seuls les navigateurs qui ont des données dans la période sélectionnée seront affichés.

    Affichage des statistiques des jours

    Lorsque l'échelle Day est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport affichera les statistiques pour chaque jour de trafic. Seuls les jours ayant des données pour la période sélectionnée seront affichés.
