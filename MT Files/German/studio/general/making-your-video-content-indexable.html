    Ihre Videoinhalte indexierbar machen (SEO)

    In diesem Thema erfahren Sie, wie Sie sicherstellen können, dass Ihre Videoinhalte von Suchmaschinen ordnungsgemäß indiziert werden.

    Endbenutzer verwenden Internet-Suchmaschinen für den schnellen und leichten Zugriff auf Informationen im Internet. Da Multimedia-Content im Internet ständig zunimmt, müssen Sie sicherstellen, dass Ihr Content von den Suchmaschinen richtig indexiert wird, damit er von Benutzern gefunden werden kann. Dieser Artikel vermittelt einen Überblick darüber, wie Sie Ihren Videocontent so optimieren, dass er von den verschiedenen Suchmaschinen (Google, Yahoo, Microsoft) gefunden wird.

    Wenn Suchmaschinen-Indexierer Ihre Website untersuchen, suchen Sie nach einem Standarddokument, nämlich nach einer Sitemap. Dieses Dokument informiert die Suchmaschinen über den Content Ihrer Website, die Metadaten für diesen Content, und wo sich der Content auf Ihrer Website befindet. Eine Sitemap ist eine nach Standardrichtlinien erstellte XML-Datei.

    Sie können (und sollten) zwei verschiedene Arten von Sitemaps erstellen:

    • Seitenverzeichnis - Eine Sitemap, die Ihren Inhalt in den standardmäßigen textbasierten Suchmaschinen wie www.google.com oder search.yahoo.com indiziert
    • Video-Sitemap - Eine Sitemap, die Ihre Inhalte in medienzentrierten Suchmaschinen wie video.google.com indiziert

    Beide Sitemaps indexieren die Metadaten über Ihren Videocontent und bieten Links für Endbenutzer. Der Unterschied liegt darin, woher die indexierten Metadaten stammen und wie der Content in den Suchergebnissen auftaucht.

    Eine richtige SEO-Strategie für Ihre Videoinhalte umfasst das Erstellen beide eine Standard-Sitemap sowie eine Video-Sitemap. Aus einer Prioritätsperspektive möchten Sie erstellen eine Standard-Sitemap zuerst und dann a Video-Sitemap. Verwenden der Video Cloud CMS API Sie schreiben ein Skript, um Ihre Video-Sitemap automatisch zu generieren, anstatt sie von Hand zu kompilieren.

    Beachten Sie, dass Brightcove Learning Services über eine verfügt Beispiel-App zum Generieren von Video-Sitemaps Verwenden der CMS-API.

    Weitere Informationen zum Erstellen des Einreichens einer Sitemap finden Sie in der Google-Dokumentation.

    Standard-Sitemaps

    Sitemaps folgen der hier definierten Sitemap-Spezifikation: www.sitemaps.org. Der Zweck der Sitemap ist die Bereitstellung einer Liste mit URLs auf Ihrer Website für Suchmaschinen. Die einzige andere auf eine URL bezogene Information ist das Datum der letzten Seitenänderung und die Häufigkeit der Seitenänderungen. Beachten Sie, dass dieser Index keine Metadaten zu Ihrem Inhalt enthält. Da die Sitemap seitenspezifisch ist, müssen wir ein Modell erstellen, bei dem jedem Video in Ihrer Bibliothek eine bestimmte Seite oder eine bestimmte URL zugewiesen ist. Dies ist mit einer einzelnen Seite möglich, deren Verhalten und Content durch Einbindung verschiedener Abfrageparameter auf der Seite dynamisch geändert werden kann. Zum Beispiel, wenn Sie eine URL wie haben http://www.example.com/video.html?videoId=123 , du hättest das video.html Seite suchen für die videoId Abfrageparameter (videoId=123 ) und ändern Sie den Inhalt der an den Browser zurückgegebenen Seite, um Informationen zum Video mit der ID 123 zu enthalten. Dies erfolgt auf der Serverseite Ihrer Anwendung, wo die Seite nach der ID sucht und dann die Video Cloud verwendet CMS API um Metadaten über das Video abzurufen und in die Seite zu schreiben. Alternativ können Sie die URL, unter der das Video veröffentlicht wird, in einem benutzerdefinierten Feld speichern, sodass die CMS API könnte es abrufen.

    Da ist auch ein changefreq Tag in der Sitemap, das Suchmaschinen mitteilt, wie oft sich der Inhalt ändert. Wenn Sie Videos häufig aktualisieren, müssen Sie diese Informationen auch in einem Tag oder einem benutzerdefinierten Feld aufzeichnen, damit die CMS API könnte es zurückgeben. Viele Videos werden selten aktualisiert. Wenn dies bei Ihnen der Fall ist, können Sie einfach den Wert des Tags auf festlegen yearly , oder Sie können dieses optionale Tag ganz weglassen. Es gibt keine eindeutigen Beweise dafür, dass Site-Crawler dem sowieso viel Aufmerksamkeit schenken.

    Im Rahmen dieses Artikels gehen wir davon aus, dass auf Ihrer Website eine einzelne Startseite vorhanden ist, über die der gesamte Videocontent Ihrer Website abgespielt werden kann. An die Seite werden verschiedene Abfrageparameter gesendet, um der Seite mitzuteilen, welches Video abgespielt werden soll und für welches Video Metadaten angezeigt werden sollen. Angenommen, auf einer Seite wird der Inhalt einer gesamten Wiedergabeliste angezeigt, und ein bestimmtes Video erscheint abspielbereit in einem Player. Sie teilen der Seite über a mit, für welche Wiedergabeliste Metadaten angezeigt werden sollen bcpid Abfrageparameter und welches Video in einem Player über die angezeigt werden soll bcvid Parameter. Wir wollen also eine URL für jede eindeutige Kombination aus Wiedergabeliste und Video-ID erstellen.

    Hier ist ein Beispiel für eine Sitemap, die erstellt werden würde:

    
<?xml version="1.0" encoding="UTF-8" ?>
 <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
   <url>
     <loc>http://www.example.com/video?bclid=345&bctid=123</loc>
     <lastmod>2015-01-01</lastmod>
     <changefreq>weekly</changefreq>
   </url>
   <url>
     <loc>http://www.example.com/video?bclid=12&bctid=544</loc>
     <lastmod>2015-03-01</lastmod>
   </url>
 </urlset>

    Um eine Sitemap-Datei zu erstellen, können Sie ein Skript erstellen, das verwendet CMS API fordert Sie auf, die Wiedergabelisten im Video Cloud-Konto durchzugehen und für jede Wiedergabeliste die darin enthaltenen Videos durchzugehen. Das Script erstellt für jede Kombination aus Wiedergabeliste und Video einen Eintrag in der Sitemap und eine eindeutige URL für die Kombination aus Wiedergabeliste und Video. Nachdem Sie eine Sitemap-Datei für Ihre Site erstellt haben, platzieren Sie die resultierende sitemap.xml Datei im Stammverzeichnis Ihres Webservers. Zum Beispiel, wenn Ihre Domain ist www.example.com wäre die Sitemap-Datei unter verfügbar http://www.example.com/sitemap.xml. Noch besser wäre es, diese Datei automatisch mit einer serverseitigen Anwendung zu generieren und die Datei auf Ihrer Live-Site zu aktualisieren.

    Video-Sitemaps

    Eine Video-Sitemap ähnelt im Konzept der Standard-Sitemap-Datei. Die Video-Sitemap-Datei enthält einen Eintrag pro Video in Ihrem Konto. Eine Video-Sitemap verwendet das Sitemap-Schema als Basis und wird durch weitere videospezifische Tags erweitert.

    In der folgenden Tabelle sind die Tags aufgeführt, die Google für Video-Sitemaps benötigt.

    Etikett Erforderlich Beschreibung
    <loc> J Gibt die Zielseite (auch als Wiedergabeseite, Referenzseite) für das Video an.
    <video:video> J Schließt alle Informationen zum Video ein.
    <video:thumbnail_loc> J Eine URL, die auf die Video-Miniaturbilddatei verweist.
    <video:title> J Der Titel des Videos.
    <video:description> J Die Beschreibung des Videos.
    <video:content_loc> U. u. möglich Eine URL, die auf die eigentliche Videomediendatei verweist. Hinweis: Sie müssen mindestens einen von <video:player_loc> oder angeben <video:content_loc>.
    <video:player_loc> U. u. möglich Eine URL, die auf einen Player für ein bestimmtes Video verweist. Normalerweise sind dies die Informationen im src-Element eines <embed> -Tags und sollten nicht mit dem Inhalt des <loc> -Tags übereinstimmen. Hinweis: Sie müssen mindestens einen von <video:player_loc> oder angeben <video:content_loc>.

    Eine Liste aller unterstützten Tags und eine ausführlichere Erläuterung der Tags finden Sie in der Google-Dokumentation.

    Unten finden Sie eine Beispiel-Video-Sitemap.

    <urlset xmlns = "http://www.sitemaps.org/schemas/sitemap/0.9" xmlns: video = "http://www.google.com/schemas/sitemap-video/1.1">
   <url>
     <loc> http://www.example.com/videos/video_landing_page.html </ loc>
     <video: video>
       <video: thumbnail_loc> http://www.example.com/thumbs/123.jpg </ video: thumbnail_loc>
       <video: title> Grillsteaks für den Sommer </ video: title>
       <video: description> Machen Sie jedes Mal perfekt gemachte Steaks </ video: description>
       <video: content_loc> http://brightcove04.brightcove.com/22/20318290001/201510/2249/20318290001_4582301804001_4582289844001.mp4?pubId=20318290001&videoId=4582289844001 </ video: content_l>
       <video: player_loc> http://players.brightcove.net/20318290001/default_default/index.html?videoId=4582289844001 </ video: player_loc>
       <Video: Dauer> 600 </ Video: Dauer>
       <video: Publikationsdatum> 2014-11-05T19: 20: 30 + 08: 00. </ video: Publikationsdatum>
       <video: tag> Steak </ video: tag>
       <video: tag> Fleisch </ video: tag>
       <video: tag> Sommer </ video: tag>
       <video: familienfreundlich> ja </ video: familienfreundlich>
     </ video: video>
   </ url>
 </ url>

    Nachdem Sie Ihre Video-Sitemap erstellt haben, übermitteln Sie sie entsprechend den hier aufgeführten Anweisungen an Google.

    Hinweis: Je nach Wunsch können Sie entweder separate Dateien für Standard- und Video-Sitemaps erstellen oder einem vorhandenen Sitemap-Dokument Informationen zu Ihrem Videoinhalt hinzufügen. Beachten Sie jedoch, dass Sitemap-Dateien nicht mehr als 50.000 URLs enthalten und nicht größer als 50 MB sein dürfen. Weitere Informationen zu Sitemap-Richtlinien finden Sie in den Google-Richtlinien Sitemap-Formate und Richtlinien Support-Artikel.

    SEO Schema Generator Plugin

    Ein weiterer guter Weg, um die SEO für Ihre Videoinhalte zu verbessern, ist die Verwendung von SEO Schema Generator Plugin. Das Plugin generiert automatisch ein Video-Schema Mikroformat Hier finden Suchmaschinen Informationen zu Videoinhalten, die in Ihren HTML-Code eingebettet sind.

    Weitere Informationen zu Mikroformaten und Möglichkeiten zum Erstellen von Videoschemata, wenn Sie das Plugin aus irgendeinem Grund nicht verwenden können, finden Sie im folgenden Abschnitt.

    Hinzufügen von Kontext mithilfe von Mikroformaten

    Zusätzlich zu Video-Sitemaps können Sie die Sichtbarkeit von Videoinhalten auf Ihrer Website mithilfe von Mikroformaten verbessern.

    HTML-Tags informieren den Browser darüber, wie Content anzuzeigen ist. Sie enthalten keine Informationen über die Bedeutung von Inhalten. Ihre Website wird womöglich anhand strukturierter Daten erstellt, die in Datenbanken gespeichert sind. Die Struktur geht jedoch verloren, wenn sie in HTML formatiert ist. Suchmaschinen können davon profitieren, wenn sie über die Datenstruktur Ihrer Website Bescheid wissen.

    Um die Art des Contents auf Ihren Webseiten zu identifizieren, wurden verschiedene Schemas (d. h. HTML-Tags) erstellt. Suchmaschinen wie Bing, Google und Yahoo verlassen sich auf dieses Markup, um die Suchergebnisse zu verbessern.

    Diese Schemata oder Mikroformate bieten Ihnen eine Sammlung gemeinsamer Vokabulare, die Sie Ihrem Markup hinzufügen können, um den Inhalt Ihrer Webseite zu definieren. Details zu diesen Tags finden Sie unter schema.org.

    Für Ihren Medienplayer empfiehlt Brightcove die Verwendung des VideoObject-Schemas.

    Brightcove hat ein Tool erstellt, mit dem Sie Ihren Player-Veröffentlichungscode mit den enthaltenen Daten im Mikroformat generieren können. Probier das aus Video Schema Builder.

    Fazit

    Eine vollständige Strategie zur Suchmaschinenoptimierung für Ihren Videocontent umfasst sowohl eine Standard-Sitemap als auch eine Video-Sitemap. Somit stellen Sie sicher, dass Ihr Content für die größtmögliche Vielfalt von Suchmaschinen indexiert wird. Außerdem wird Ihr Content bei der Suche so ansprechend wie möglich aufgelistet. Der vorliegende Artikel beschreibt eine Methode zur Bereitstellung einer eindeutigen URL pro Video in Ihrem Konto und bietet Beispielcode zum Generieren der Sitemap-Dateien, die von den Suchmaschinen indexiert werden.
