    Standard-Ingest-Profile für dynamische Bereitstellung und kontextbezogenes Codieren

    In diesem Thema erfahren Sie mehr über die Eigenschaften der standardmäßigen Aufgaufgabeprofile, die bei der Verwendung von Dynamic Delivery und Context Aware Encoding bereitgestellt werden.

    Einführung

    Wenn Sie eine Dynamic Ingest-Anfrage mit dem erstellen Dynamic Ingest API oder Videos mit dem Upload-Modul hochladen, müssen Sie ein Profil angeben, das die Anzahl und Art der Wiedergaben bestimmt, die durch den Video Cloud-Aufnahmeprozess erstellt wurden. Dieses Thema enthält detaillierte Informationen zu jedem der Profile Dynamic Delivery and Context Aware Encoding (CAE) , einschließlich HEVC (h.265) -Profilen.

    Hinweise:
    • Context Aware Encoding und HEVC-Codierung sind kostenpflichtige Add-Ons zu Video Cloud. Wenn Sie daran interessiert sind, diese Funktionen Ihrem Konto hinzuzufügen, wenden Sie sich an Ihren Account Manager.

    Einen Überblick über Dynamic Delivery finden Sie unter Überblick über Dynamic Delivery.

    Einen Überblick über Context Aware Encoding finden Sie unter Überblick über Context Aware Encoding.

    Einen Überblick über HEVC finden Sie unter HEVC-Video

    Hinweise:

    • Dieses Dokument zeigt Profile für die reguläre dynamische Bereitstellung und kontextbezogene Codierung, einschließlich HEVC-Profile
    • Aufgrund der verschiedenen Möglichkeiten, wie statische Dynamic Delivery und Context Aware Encoding Profile funktionieren, werden im folgenden Profildetails verschiedene Informationen angezeigt:
      • Für statische Dynamic Delivery Profile wird eine Liste der enthaltenen Formatvarianten angezeigt
      • Für kontextbezoge Codierungsprofile wird eine Tabelle mit dynamischen Profileinstellungen angezeigt, die verwendet werden, um zu bestimmen, welche Formatvarianten erzeugt werden sollen

    Es ist auch möglich, eigene benutzerspezifische Ingest-Profile zu erstellen. Weitere Informationen finden Sie im entsprechenden Thema:

    • Erstellen von Aufgabe-Profilen für die dynamische Bereitstellung
    • Erstellen von Aufgaufriefiles für kontextbezogenes

    Wiedergabeformen

    Einzelheiten zu allen verfügbaren Standard-Formatvarianten, die Sie in Ihren Aufrufprofilen verwenden können, finden Sie in den folgenden Tabellen.

    Hinweise:
    1. Dies sind alle Formatvarianten, die zur Verwendung in Dynamic Delivery Profilen verfügbar sind. Einige von ihnen werden derzeit möglicherweise nicht in den Standardprofilen verwendet.
    2. Die Fragmentdauer und die Segmentlänge, die Sie in der Tabelle sehen, spiegeln nicht die tatsächlich gelieferten Segmentlängen wider. Dynamic Delivery kann Segmente unterschiedlicher Länge liefern, abhängig von den Bedingungen für eine bestimmte Wiedergabesitzung. Standardmäßig werden 10-Sekunden-Segmente für HLS und 6-Sekunden-Segmente für DASH ausgeliefert. In diesem Hinweis unten finden Sie eine umfassendere Erklärung von Fragmenten und Segmenten.

    Audio-Wiedergaben

    Segmentierte Audio-Wiedergaben

    Segmentierte Audio-Neuformate
    Vorname[1-1] Audiobitrate (Kbit/s) Audiokanäle Audiocodec Erzwungene Keyframe- Fragment-Dauer (ms) Zeitskala für Fragment-Spuren Segmentlänge (Sek)
    Hinweise
    • [ 1-1] Wenn Sie auf eine Formatvariante in einem Ingest-Profil verweisen, stellen Sie sicher, dass Sie dem Namen voranstellen default/- zum Beispiel: default/audio128
    • [ 1-2] Ein Segment korreliert mit einer herunterladbaren Datei. Zum Beispiel könnten Sie mit HLS die Datei alle 6 Sekunden segmentieren. Jedes Segment enthält genügend Informationen, damit es unabhängig von anderen Segmenten dekodiert und gespielt werden kann. Ein HLS-Manifest listet jeden Segmentpfad und seine Dauer auf.

      Ein Fragment bezieht sich auf Fragmente einer MP4-Datei. Ein nicht fragmentiertes MP4 enthält Metadaten am Anfang oder Ende der Datei und erfordert, dass die Daten (FTYP, MOOV Atome) vor dem Medium (MDAT Atom) dekodiert werden. Um das Streaming zu vereinfachen, kann eine MP4-Datei fragmentiert werden, sodass die für die Wiedergabe erforderlichen Metadaten sowie die Medien in Fragmente in der Datei aufgeteilt werden. Dies ermöglicht es den Spielern, leicht nach bestimmten Medienbrüchen zu suchen und fließende Übergänge zwischen verschiedenen Darstellungen zu ermöglichen. Fragmentieren wurde ursprünglich für DASH erstellt, wird jetzt aber auch von HLS unterstützt.

      Sie können auch Medien fragmentieren und strichen, sodass ein Segment (eine Datei) mehrere Fragmente enthalten kann. Es ist üblich, die Segment- und Fragmentdauer gleich zu gestalten. Beachten Sie dies duration und length beziehen Sie sich auf dasselbe, wenn Sie Fragmente und Segmente besprechen.
