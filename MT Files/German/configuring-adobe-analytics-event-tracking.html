    Konfigurieren von Ereignisverfolgung durch A

    In diesem Thema erfahren Sie, wie Sie die Adobe Analytics-Ereignisverfolgung im Players-Modul konfigurieren.

    Das Players-Modul bietet die Möglichkeit, Brightcove-Player einfach in Adobe Analytics zu integrieren. Dadurch können Ereignisverfolgungsdaten zu Berichtszwecken an ein Adobe Analytics-Konto gesendet werden.

    Hinweis: Jedes Analysesystem sammelt Daten und berechnet Metriken unterschiedlich. Video Cloud-Analysedaten sind nicht identisch mit denen in Adobe Analytics.
    Hinweis: Das Adobe Analytics-Plugin wird bei Verwendung der Brightcove Player-SDKs nicht unterstützt.

    Um die Integration zu konfigurieren, müssen die folgenden Aufgaben durchgeführt werden:

    • Überprüfen Sie die verfügbaren Ereignisse zur Nachverfolgung
    • Erstellen Sie benutzerdefinierte Erfolgsereignisse in Ado
    • Überprüfen Sie die verfügbaren Parameter zur Erfassung
    • Erstellen von Traffic-Variablen in Adobe
    • Aktivieren der Integration in das Players-Modul
    • Konfigurieren Sie die Seite zum Sammeln von Daten

    Beachten Sie, dass bei der Konfiguration der Integration nur Analysen für Videoereignisse an Adobe Analytics weitergegeben werden.

    Überprüfen der verfügbaren Ereignisse, die verfolgt werden sollen

    Jedes Brightcove-Ereignis, das verfolgt wird, muss als benutzerdefiniertes Erfolgsereignis in Adobe Analytics erstellt werden. Die Ereignis-IDs von Adobe werden dann im Players-Modul verwendet. Dadurch können Brightcove-Ereignisdaten an Adobe Success-Ereignisse weitergegeben werden. Sehen Sie sich die Liste der Ereignisse unten an, um zu bestimmen, welche Ereignisse Sie verfolgen möchten.

    • Player Load (bcvideo_player_load) - Wird gesendet, wenn ein Player geladen wurde und für die Wiedergabe bereit ist
    • Video Impression (bcvideo_video_impression) - Wird gesendet, wenn ein Video in einen Player geladen wurde und zur Wiedergabe bereit ist
    • Videoansicht (bcvideo_video_view) - Wird gesendet, wenn ein Video mit der Wiedergabe eines Videos entweder durch automatische Wiedergabe oder aufgrund von Benutzerinteraktion begonnen hat
    • Video Interaktion (bcvideo_video_engagement) - Ein Benutzer hat eine Reihe von Sekunden der Timeline eines Videos gesehen. Dieses Ereignis ist ein Herzschlag für die Verfolgung der Videointeraktion und wird wahrscheinlich mehrmals während der Wiedergabe gesendet, abhängig von der Benutzerinteraktion und der Länge des Videos.
    • Wiedergabeanfrage (bcvideo_play_request) - Wird gesendet, wenn die Wiedergabe entweder durch ausdrücklichen Klick des Benutzers auf die Wiedergabetaste oder automatisch initiiert wird, wenn die Plattform die Wiedergabe in einem Auto-Play-Szenario auslöst
    • Ad Start (bcvideo_ad_start) - Wird gesendet, wenn die Kontrolle von der Playback-Plattform an einen Werbeagenten übergeben wird
    • Ad End (bcvideo_ad_end) - Wird gesendet, wenn die Steuerung vom Werbeagenten an die Wiedergabe-Plattform zurückgegeben wird
    • Error (bcvideo_error)- Wird gesendet, wenn schwerwiegende Fehler auftreten, die das Wiedergabeerlebnis beeinträchtigen
    • Video abgeschlossen (bcvideo_video_complete) - Wird gesendet, wenn die Videowiedergabe beendet ist

    Hinzufügen von benutzerdefinierten Erfolgsereignissen zu Ado

    Nachdem Sie festgelegt haben, welche Ereignisse Sie nachverfolgen möchten, müssen benutzerdefinierte Erfolgsereignisse in Adobe Analytics erstellt werden. Gehen Sie folgendermaßen vor, um benutzerdefinierte Erfolgsereignisse in Adobe Analytics

    1. Melden Sie sich bei Ihrem Adobe Analytics-Konto
    2. Wählen Sie eine Report Suite aus, mit der Sie arbeiten oder eine neue erstellen möchten (Admin > Report Suites).
    3. Klicken Sie auf Einstellungen bearbeiten > > Konversionsereignisse.
    4. Klicken Sie auf „ Neu hinzufügen“
    5. Geben Sie die Veranstaltungsinformationen ein:
      • Name - Sie müssen den im vorherigen Abschnitt aufgeführten bcvideo-Namen verwenden
      • Typ - Auf Zählersetzen
      • Beschreibung - Fügen Sie eine aussagekräftige Beschreibung
    6. Erstellen Sie ein neues Erfolgsereignis für jedes Event, das Sie verfolgen möchten. Klicken Sie auf Speichern , um die Ereignisse zu speichern.

    Das Bild unten zeigt benutzerdefinierte Erfolgsereignisse, die für jedes Brightcove-Event erstellt wurden (event7 - event15).

    Überprüfen der verfügbaren Parameter für die Erfassung

    Parameter enthalten zusätzliche Daten und werden mit Ereignissen übergeben. Jeder Brightcove-Parameter, der aufgezeichnet wird, muss in Adobe Analytics als Verkehrsvariable (eVarXX, wobei XX eine Zahl wie 88 ist) erstellt werden. Der genannte Parameter wird dann im Players-Modul verwendet. Auf diese Weise können Brightcove-Parameter an Adobe Traffic Variables übergeben werden. Sehen Sie sich die folgende Liste der Parameter an, um zu bestimmen, welche Sie erfassen möchten.

    • Konto-ID (bcvideo_account) - Brightcove Kontoname
    • Spieler ID (bcvideo_player) - Spieler-ID
    • Spielername (bcvideo_player_name) - Spielername
    • Spielerversion (bcvideo_platform_version) - Player-Versionsnummer
    • Session (bcvideo_session) - Sitzungskennung
    • Video ID (bcvideo_video) - Video ID des Videos
    • Videoname (bcvideo_video_name) - Name des Videos
    • Angebot (bcvideo_range) - Der Sekundenbereich, der im Format StartSecond..EndSecond angezeigt wird
    • Angesehene Videosekunden (bcvideo_video_seconds_viewed) - Anzahl der beobachteten Sekunden
    • Videodauer (bcvideo_video_duration) - Die Dauer des Videos in Sekunden
    • Video Prozent angesehen (bcvideo_video_percent_viewed) - Prozentsatz des angezeigten Videos
    • Fehlercode (bcvideo_error_code) - Plattformspezifischer Fehlercode für das Ereignis

    Beispiel

    1. Im Player-Plugin werden Werte in der Video Cloud festgelegt bcvideo_player_name = eVar88
    2. Und in der Adobe-Konsole sollten Sie festlegen eVar88 zu bcvideo_player_name

    Erstellen von Verkehrsvariablen in Adobe Analy

    Nachdem Sie festgelegt haben, welche Parameter Sie verfolgen möchten, müssen Traffic-Variablen in Adobe Analytics erstellt werden. Um Verkehrsvariablen in Adobe Analytics zu erstellen, führen Sie diese Schritte aus.

    1. Melden Sie sich bei Ihrem Adobe Analytics-Konto
    2. Wählen Sie eine Report Suite aus, mit der Sie arbeiten oder eine neue erstellen möchten (Admin > Report Suites).
    3. Klicken Sie auf Einstellungen bearbeiten > > Traffic-Variablen.
    4. Geben Sie die Parameterinformationen ein:
      • Name - Sie müssen den im vorherigen Abschnitt aufgeführten bcvideo-Namen verwenden
      • Beschreibung - Fügen Sie eine aussagekräftige Beschreibung
    5. Erstellen Sie eine neue Verkehrsvariable für jeden Parameter, den Sie verfolgen möchten. Klicken Sie auf Speichern , um die Variablen zu speichern.

    Das Bild unten zeigt Verkehrsvariablen, die für jeden der Brightcove-Parameter erstellt wurden.

    Ermöglichung der Integration

    Möglicherweise möchten Sie sich bei Ihrem Adobe Analytics-Konto anmelden, um die erforderlichen Informationen von dort in das Players-Modul zu kopieren. Gehen Sie folgendermaßen vor, um die Adobe Analytics-Ereignisverfolgung für einen Player zu konfigurieren:

    1. Klicken Sie im Navigationsheader auf Players, um das Modul Players zu öffnen.
    2. Klicken Sie auf den Link für den zu konfigurierenden Player.
    3. Klicken Analytics von Drittanbietern> Adobe Analytics im linken Navigationsmenü.
    4. Aktivieren Sie Adobe Analyticsaktivieren
    5. Geben Sie Ihre Adobe-Kontoinformationen ein.
      • Suite-ID melden
      • Marketing-Cloud-Organisations-ID
      • Tracking-Server
      • CNAME aktiviert
      • Besucher-Namespace
    6. Geben Sie für jedes Ereignis, das Sie verfolgen möchten, die Ereignis-ID von Adobe Analytics in das entsprechende Feld für das Brightcove-Ereignis ein. In diesem Beispiel, wenn die bcvideo_player_load Das Ereignis wurde in Adobe Analytics als benutzerdefiniertes Erfolgsereignis erstellt. Die zugewiesene Ereignis-ID lautete event7.

      Dies ist der Wert, der in das Feld für das Event Player Load (bcvideo_player_load) im Players-Modul eingegeben werden sollte.

    7. Geben Sie bei Bedarf alle Ereignis-IDs von Adobe Analytics in das Player-Modul ein.
    8. Klicken Sie auf Speichern , um die Ereigniszuordnungen zu speichern.
    9. Geben Sie für jeden Parameter, den Sie erfassen möchten, die Parameternummer von Adobe Analytics in das entsprechende Feld für den Brightcove-Parameter im Format ein propX wo X. entspricht dem Eigentum # in Adobe Analytics. In diesem Beispiel wurde in Adobe Analytics ein bcvideo-Konto Traffic Variable erstellt, das die Brightcove-Konto-ID enthält. Es wurde Eigentum Nr. 1 zugewiesen.

      Daher sollte prop1 auch in das entsprechende Feld für den Parameter im Players-Modul eingegeben werden.

    10. Geben Sie bei Bedarf alle Parameternamen von Adobe Analytics in das Players-Modul ein.
    11. Klicken Sie auf Speichern , um die Parameterzuordnungen zu speichern.

    Stellen Sie sicher, dass Sie die Änderungen für den Player veröffentlichen.

    Verwenden der Integration mit einer Galerie-In-Page-Erfahrung

    Wenn der Player in einer Galerie-In-Page-Erfahrung verwendet wird, ist eine Konfiguration der In-Page-Erfahrung erforderlich:

    1. Öffnen Sie das Gallery-Modul.
    2. Bearbeiten Sie die Erfahrung mit dem Player mit Adobe Analytics-Integration.
    3. Klicken EINSTELLUNGEN> Erweitert in der linken Navigation.
    4. Fügen Sie dem JavaScript Folgendes hinzu Code Sektion: 
      window.s_gi = window.top.s_gi
    5. Klicken Sie auf Save.

    Denken Sie daran, die Erfahrung erneut zu veröffentlichen.

    Konfigurieren einer Seite zum Sammeln von Daten

    Um mit dem Senden von Daten an Adobe Analytics zu beginnen, müssen Daten an eine Report Suite gesendet werden, um sie in Berichten anzuzeigen. Um eine Seite mit Code zum Sammeln von Daten erfolgreich zu implementieren, müssen Sie Zugriff auf Ihre Hosting-Server haben, um neue Inhalte auf Ihre Website hochzuladen. Es ist auch nützlich, eine vorhandene Site zu haben, um Code zu implementieren.

    Hinweis: Verwenden Sie den erweiterten Einbettungscode (In-Page) bei der Veröffentlichung Ihres Players. Der Standard-Einbettungscode (iframe) funktioniert nicht, da das Plugin innerhalb eines iFrames ausgeführt wird und daher nicht mit dem benötigten JavaScript interagieren kann.

    Die folgenden Schritte durchlaufen eine grundlegende Implementierung von Adobe Analytics.

    1. Laden Sie AppMeasurement für JavaScript und den Besucher-ID-Service herunter Der Download ist im Code Manager (Analytics > Admin > Code Manager) verfügbar. Die Download-Zip enthält mehrere Dateien. AppMeasurement.js und VisitorAPI.js sind die relevanten Dateien bei der Implementierung von Analytics.
    2. Host AppMeasurement.js und VisitorAPI.js. Diese zentralen JavaScript-Dateien müssen auf einem Webserver gehostet werden, der für alle Seiten Ihrer Website zugänglich ist. Sie benötigen den Pfad zu diesen Dateien, wenn Sie sie Ihren Seiten hinzufügen.
    3. Referenz AppMeasurement.js und VisitorAPI.js auf allen Seiten einer Website mit einem Brightcove Player, indem Sie die folgenden Codezeilen in der <head> oder <body> tagge auf jeder Seite. VisitorAPI.js muss vorher einbezogen werden AppMeasurement.js. 
      <script language="JavaScript“ type="text/javascript“  src=“ https://Insert Domain und Pfad zur Datei here/VisitorAPI.js "> < /script >
<script language="JavaScript“ type="text/javascript“  src=“ https://Insert Domain und Pfad zur Datei here/AppMeasurement.js "> < /script >

    Bestätigung des Setups

    Gehen Sie folgendermaßen vor, um zu bestätigen, dass Brightcove-Ereignisdaten in Adobe Analytics erfasst werden.

    1. Melden Sie sich bei Ihrem Adobe Analytics-Konto
    2. Klicken Sie auf Berichte.
    3. Klicken Sie auf Site-Metriken > in Echtzeit
    4. Bestätigen Sie, dass die richtige Report Suite ausgewählt ist.
    5. Klicken Sie auf Konfigurieren.
    6. Geben Sie einen Berichtsnamenein.
    7. Wählen Sie für „ Metrik“ , „ Primäre Dimension“ und „Sekundäre Dimension“ die Brightcove-Ereignisse aus, die Sie in Adobe Analytics als Erfol Die Ereignisse sollten in der Liste erscheinen.
    8. Wählen Sie andere Ereignisse aus, um die Dimensionen auszufüllen.
    9. Klicken Sie auf „ Speichern und Bericht anzeigen“.
    10. Bestätigen Sie, dass die erwarteten Daten im Bericht angezeigt werden.

    Wenn keine Daten angezeigt werden, überprüfen Sie, ob die URLs zu  AppMeasurement.js und VisitorAPI.js Dateien sind gültig. Überprüfen Sie die Browserkonsole, um sicherzustellen, dass keine 404-Fehler auftreten. Außerdem können Werbeblocker verhindern, dass Daten an Adobe Analytics gesendet werden.
