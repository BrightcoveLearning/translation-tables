    Erstellen von benutzerdefinierten Analytics-Ber

    In diesem Abschnitt erfahren Sie, wie Sie mit dem Analytics-Modul benutzerdefinierte Berichte erstellen.
    Hinweis: Die Möglichkeit, benutzerdefinierte Berichte im Analytics-Modul zu erstellen, steht nur Enterprise- und Professional-Publishern zur Verfügung.
     

    Sie können mit dem Analytics-Modul benutzerdefinierte Ad-hoc-Analytics-Berichte erstellen, die im CSV- oder XLSX-Format herunterladbar sind. Berichte können auch täglich, wöchentlich oder monatlich erstellt werden.

    Das Analytics API wird verwendet, um Daten aus dem Analysesystem abzurufen. Das Analytics API kann auch verwendet werden, um eigene benutzerdefinierte Berichte zu erstellen. Weitere Informationen über die Analytics-API finden Sie in der Dokumentation Video Cloud Analytics API.

    Um benutzerdefinierte Berichte zu erstellen, öffnen Sie das Analytics-Modul und klicken Sie auf Benutzerdefiniert Link in der linken Navigation.

    Hinweis: Möglicherweise stellen Sie fest, dass die Anzahl der Videoaufrufe für einen bestimmten Tag im Laufe der Zeit variiert. Eine Erklärung, warum dies passieren kann, finden Sie in der Analytics FAQ.
    Hinweis: Berichte, die mehr als eine Million Zeilen zurückgeben, werden nicht geladen.
    Hinweis: Gleichzeitige benutzerdefinierte Berichtsanforderungen innerhalb desselben Kontos werden in Reihe ausgeführt. Wenn die Berichte große Datumsbereiche haben und viele Felder ausgewählt sind, kann es zu einer Verzögerung kommen, bevor alle Ergebnisse zurückgegeben werden. Eine gute Alternative ist das Planen großer Berichte.
    Hinweis: Die Videodauer wird in Sekunden angezeigt.

    Führen Sie die folgenden Schritte aus, um einen benutzerdefinierten Bericht zu erstellen:

    1. Konten auswählen
    2. Wählen Sie einen Berichtstyp aus
    3. Datengruppen auswählen
    4. Fügen Sie einen Berichtsfilter hinzu
    5. Datenfelder auswählen
    6. Vorschau des Berichts
    7. Konfigurieren Sie die Lieferoptionen
    8. Fordern Sie den Bericht an

    Zeitzone für die Berichterstellung

    Die in benutzerdefinierten Berichten enthaltenen Daten entsprechen der angepassten Zeitzoneneinstellung für das jeweilige Konto. Die Berichtszeitzone kann auf der Video Cloud Studio-Startseite unter konfiguriert werden ADMINISTRATOR > Kontoeinstellungen> Standard-Berichtszeitzone. Nur Kontoadministratoren haben Zugriff auf die Kontoeinstellungen.

    Die Zeitzone für die Berichterstellung wird für Berichte zur Berechnung von Tagesgrenzen verwendet. Wird die Zeitzone für die Berichterstellung geändert, gilt die Änderung nicht rückwirkend. Wenn Sie diese Einstellung ändern, bemerken Sie an Tagesgrenzen, an denen die Änderung vorgenommen wurde, möglicherweise ein extrem niedriges oder hohes Datenaufkommen. Eine Aktualisierung dieser Einstellung tritt möglicherweise nicht sofort in Kraft, und Analytics-Daten beziehen sich u. U. weiterhin solange auf die alte Einstellung, bis die Änderungen vom System verarbeitet wurden.

    Beachten Sie, dass die Standard-Berichtszeitzone gilt nur zum Analytics-Modul und nicht zu Live Analytics, das technisch gesehen ein getrennter Dienst ist

    Konten auswählen

    Wenn Sie Zugriff auf mehrere Video Cloud-Konten haben, können Sie Daten aus dem aktuellen Video Cloud-Konto einschließen oder mehrere Konten auswählen.

    Wenn Sie einen Bericht erstellen, der alle Konten umfasst, und diese Konten verschiedene Zeitzonen umfassen, werden die Daten mit der kontospezifischen Zeitzone aufgelistet.

    Berichttyp auswählen

    Wählen Sie den Berichttyp:

    • One-time – der Bericht wird sofort, aber nur einmal erstellt.
    • Recurring – der Bericht kann täglich, wöchentlich oder monatlich erstellt werden.
      Hinweis: Wiederkehrende Berichte werden ausgeführt, nachdem die Analyseabstimmung stattgefunden hat, normalerweise 3 bis 5 Tage nach dem Ende eines Berichtszeitraums.

    Einmaliger Bericht

    Mit der Option One-time wird der Bericht nur einmal erstellt. Sie legen mit einem Kalender-Steuerelement den Zeitraum fest, für den Sie die Analytics-Daten einsehen möchten.

    Tipp: Wählen Sie zunächst einen kleineren Datumsbereich, um zu bestätigen, dass der Bericht die gewünschten Daten enthält. Erweitern Sie den Datumsbereich nach Bedarf, nachdem Sie die Berichtdaten bestätigt haben.

    Regelmäßig erstellter Bericht

    Mit der Option Recurring wird der Bericht regelmäßig erstellt. Wählen Sie eine Häufigkeit (Frequency) für den Bericht:

    • Daily – der Bericht deckt einen gesamten Tag ab.
    • Weekly – der Bericht deckt eine gesamte Woche von Montag bis Sonntag ab.
    • Monthly – der Bericht deckt einen gesamten Monat ab.
    • Unternehmenskonten sind auf 20 wiederkehrende Berichte beschränkt, professionelle Konten auf 5
    • Regelmäßig erstellte Berichte sind einem Benutzer zugewiesen und über alle Konten verfügbar, auf die der Benutzer Zugriff hat.
    • Diese Berichte werden nach Abschluss des Analytics-Abgleichs erstellt – normalerweise 3 bis 5 Tage nach einem Berichtzeitraum.

    Datengruppen auswählen

    Mit Datengruppen bestimmen Sie, welche Datenfelder dem Bericht hinzugefügt werden können. Die Datenfelder im Abschnitt Data Fields basieren auf den ausgewählten Datengruppen. Sie können mehrere Datengruppen wählen, allerdings sind bestimmte Kombinationen ungültig. Während der Auswahl von Datengruppen wird die Liste verfügbarer Datengruppen aktualisiert, und sie enthält nur die gültigen Gruppen, die auch ausgewählt werden können.

    Hinweis: Das Datum Datengruppe kann nicht mit anderen Dimensionen kombiniert werden.
    Hinweis: Eine Liste der Datenfelder, die mit jeder Dimension zurückgegeben werden, finden Sie unter dieses Dokument.

    Datengruppen und Felder

    Wenn Sie a auswählen Datengruppe können die Datenfelder ausgewählt werden, die zurückgegeben werden können. Abgeblendete Felder können für diese Datengruppe nicht zurückgegeben werden.

    Klicken Sie auf die Schaltfläche , um eine Datengruppe hinzuzufügen. Klicken Sie auf , um eine Gruppe zu entfernen.

    Berichtfilter hinzufügen

    Fügen Sie Filter hinzu, um die zurückgegebene Datenmenge zu begrenzen. Wenn Sie mehrere Filter nutzen, wird ein UND-Vorgang verwendet. Im folgenden Beispiel werden Videos mit Tags versehen Video Cloud Essentials UND haben das benutzerdefinierte Feld Produkt = Video Cloud Wird zurückgegeben.

    Hinweis: Ein Filter wird in Ihrem Bericht nur angezeigt, wenn Sie auch die entsprechende Datengruppe ausgewählt haben. Wenn Sie beispielsweise nach Video-Tags filtern möchten, müssen Sie die Videodimension auswählen.

    Klicken Sie auf die Schaltfläche , um einen Filter hinzuzufügen. Klicken Sie auf , um einen Filter zu entfernen. Die folgenden Werte sind als Filter verwendbar.

    • Video – mindestens eine aktive Video-ID im Konto
    • Video-Tag - Ein oder mehrere Video-Tags
      • Eintreten "Finanzdienstleistungen" (mit den Anführungszeichen) gibt Videos zurück, die mit Tags versehen sind Finanzdienstleistungen.
      • Bei Verwendung mehrerer Tags werden Videos mit ALLEN Tags zurückgegeben. Eintreten "Finanzdienstleistungen", Buchhaltung gibt alle getaggten Videos zurück Finanzdienstleistungen und Buchhaltung.
      • Hinweis: Der Mechanismus zum Zurückgeben von Videos nach Tags verfügt über keine Sortier- oder Paging-Optionen.
    • Benutzerdefinierte Felder - Ein oder mehrere benutzerdefinierte Felder im Konto
      • Beim Filtern nach benutzerdefinierten Feldern und Wählen Sie Alle Wenn diese Option ausgewählt ist, enthält die Liste der benutzerdefinierten Felder die benutzerdefinierten Felder aller Ihrer Konten. Bei der Auswahl von benutzerdefinierten Feldern wird die Liste so gefiltert, dass sie nur die benutzerdefinierten Felder enthält, die allen Konten gemeinsam sind.
    • Spieler - Der Filterwert hängt vom gesuchten Kontext ab:
      • Brightcove-Spieler - Geben Sie die Player-URL in diesem Format ein: player.brightcove.com/account_id/.playerID_Standard
      • Brightcove-Player, die in SDK-Apps verwendet werden - Verwenden Sie die Zieleigenschaft aus der App. Der Standardwert für die Zieleigenschaft ist bcsdk: // gefolgt von der Bundle-ID der App, die in der Datei info.plist der App aufgeführt ist.
    • Quelle Typ - Einer der folgenden Werte:
      • Direkt (Lesezeichen, eingegebene URL oder Link aus E-Mail)
      • Empfehlung (direkter Link von einer anderen Seite)
      • Organische Suche (gewöhnliche Suche mit einer Suchmaschine)
      • Bezahlte Suche (Klicken auf einen gesponserten Link in den Suchergebnissen)
      • Sichere Suche (Suche mit einem angemeldeten Viewer)
    • Device Type – einer der folgenden Werte:
      • Mobilgerät
      • Tablet
      • Angeschlossenes Fernsehgerät / Spielekonsole / Begleitgerät
      • Desktop
      • Sonstiges
    • Device OS – einer der folgenden Werte:
      • Android
      • bada
      • iOS
      • BlackBerry
      • Symbian
      • Web OS
      • Windows
      • OS X
      • Mac OS
      • Linux
      • Sonstiges
    • Gerätehersteller - Einer der folgenden Werte:
      • Apple

      • HTC

    • Browsertyp - Einer der folgenden Werte:
      • Chrome

      • Alle anderen
    • Country – ISO-3611-1-Ländercode
    • Destination Domain – die Domäne, in der eine Videoimpression (video_impression) vorkam

    Datenfelder auswählen

    Verwenden Sie den Abschnitt Data Fields zur Auswahl der Datenwerte, die im Bericht enthalten sind. Die Liste der verfügbaren Datenfelder basiert auf den ausgewählten Dimensionen.

    Wenn das ausgewählte Konto mit benutzerdefinierten Feldern konfiguriert ist, können diese auch ausgewählt werden. Wenn der Bericht alle Konten enthält, werden alle benutzerdefinierten Felder dieser Konten angezeigt.

    Hinweis: Für die Nutzungsdaten geben wir nur Live-Sekunden-Streams an, keine Live-Bandbreite.

    Bericht in Vorschau darstellen

    Das Vorschau des Berichts Abschnitt zeigt die ausgewählten Datenspalten, die mit gefüllt sind Beispieldaten. Klicken Sie auf ein Papierkorbsymbol, um dieses Datenelement aus dem Bericht zu entfernen.

    Hinweis: Die in der Berichtsvorschau angezeigten Daten sind Beispieldaten , keine tatsächlichen Daten aus Ihrem Video Cloud-Konto. Die Sortierreihenfolge der Vorschau hat auch keinen Einfluss darauf, wie die Daten in dem von Ihnen heruntergeladenen Bericht sortiert werden. Die Daten im heruntergeladenen Bericht können mithilfe einer Tabellenkalkulationsanwendung eines Drittanbieters sortiert werden.

    Bereitstellungsoptionen konfigurieren

    Beginnen Sie, indem Sie den Bericht aussagekräftig benennen. Ein aussagekräftiger Name ist insbesondere wichtig, wenn ein Bericht regelmäßig erstellt wird.

    Wählen Sie im Abschnitt Format das Format, in dem die Berichtdaten heruntergeladen werden.

    • CSV – durch Komma getrennte Werte
    • XLSX – Microsoft Excel

    Lassen Sie sich selbst und andere Benutzer im Unternehmen per E-Mail benachrichtigen, sobald der Bericht fertig ist. Aktivieren Sie E-mail me when my report is ready, damit Sie per E-Mail über die Fertigstellung des Berichts informiert werden. Fügen Sie dem Textfeld Notify others die E-Mail-Adressen weiterer Personen hinzu, um diese Personen zu informieren. Die Personen erhalten eine E-Mail mit einem Link zum Bericht, sobald der Bericht fertig ist.

    Der Link zum Herunterladen des Berichts ist 90 Tage gültig.

    Bericht anfordern

    Drücke den Bericht anfordern (für einmalige Berichte) oder Planen Sie den Bericht Schaltfläche (für wiederkehrende Berichte), um den Bericht auszuführen. Handelt es sich um einen einmaligen Bericht, wird dieser nach Fertigstellung heruntergeladen. Regelmäßig erstellte Berichte werden in eine Warteschlange eingereiht und generiert, wenn die Daten bereitstehen.

    Geplante Berichte anzeigen

    Klicken Sie in der linken Navigationsleiste auf Scheduled Reports, um alle regelmäßig erstellten Berichte anzuzeigen. Weitere Informationen über die Verwendung von geplanten Berichten finden Sie unter Verwalten geplanter Berichte.
