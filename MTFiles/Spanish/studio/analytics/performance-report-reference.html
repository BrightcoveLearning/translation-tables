    Referencia de los informes de rendimiento

    En este apartado analizaremos el informe de rendimiento, que forma parte del módulo Analytics.
     

    Los reproductores de Brightcove envían automáticamente los datos de análisis, incluidos los proporcionados por los SDK de Native Player. Si no utiliza un reproductor de Brightcove para entregar vídeos de Video Cloud, debe instrumentar el reproductor que está utilizando para enviar los datos al recopilador de datos. Consulte la descripción general: Tema Data Collection API para obtener más detalles.

    Para ver el informe de rendimiento, abra el módulo Analytics y seleccione Rendimiento en la barra de navegación izquierda. El informe mostrará un gráfico con el número total de visionados diarios de sus vídeos durante el período de tiempo seleccionado. La lista desplegable permite seleccionar otro parámetro:

    • Visualizaciones de vídeo : el número de veces que un vídeo comenzó a reproducirse, grabado cuando se inicia la transmisión (no incluye rebobinado o repetición); no es una medida de espectadores individuales; para vídeos con anuncios pre-roll, una vista no se graba hasta que el vídeo comience a reproducirse.
    • Impresiones de vídeo - Número de veces que se cargó un vídeo y listo para la interacción
    • Tasa de reproducción: porcentaje de los vídeos que realmente se reprodujeron una vez cargados (visionados del vídeo / impresiones del vídeo).
    • Minutos vistos - Número de minutos que se vio el vídeo
    • Índice de interés despertado: promedio porcentual de tiempos de visionado (se calcula sumando todos los porcentajes de visionado, dividido por el número de visionados del vídeo)
    • Visores únicos diarios : se calcula utilizando la dirección IP y el agente de usuario de los espectadores. Brightcove decidió no utilizar cookies ajenos porque los navegadores cada vez los suelen bloquear más. El dato Espectadores únicos diarios solo está disponible a nivel de cuenta. Este parámetro deja de estar disponible cuando se filtra por un vídeo (o por cualquier otro concepto). Brightcove también permite calcular el número de espectadores únicos a partir del valor que usted indique en lugar de su dirección IP. Si desea más información, consulte el documento para desarrolladores Seguimiento de eventos con el módulo Analytics.
    • Ancho de banda de vídeo : incluye ancho de banda sólo para vídeos. Tenga en cuenta que los datos del ancho de banda se muestran con un retraso de 3 a 5 días.
    • Ancho de banda total : incluye ancho de banda para vídeos, imágenes y cargas de reproductores. Tenga en cuenta que los datos del ancho de banda se muestran con un retraso de 3 a 5 días.

    Si su cuenta está habilitada para Brightcove Live, tendrá la opción de ver los datos de rendimiento de todos los vídeos, activos de VOD o activos en directo.

    Notas:
    • Los informes del módulo Analytics también reflejan los análisis de los vídeos eliminados de su cuenta. En el módulo Analytics ya no se pueden buscar vídeos eliminados para ver los informes de vídeos eliminados concretos. Para obtener un informe de este tipo, es necesario conocer el identificador del vídeo eliminado y generar el informe con la Analytics API.
    • Los datos de los espectadores únicos diarios y los datos de ancho de banda de vídeo tardan hasta 3 días en estar disponibles.
    • Una vista de vídeo no es la misma que las vistas en x% en el informe de participación. Se contabiliza como "visionado de un vídeo" la primera vez que se comienza a reproducir un vídeo. Se contabiliza como "visionados al 1%" cuando el espectador ha visto el primer 1% del vídeo. La diferencia aquí es que si el primer porcentaje se ve varias veces (si el vídeo está configurado para reproducción automática, por ejemplo), el recuento aumenta, mientras que una vista de vídeo se cuenta cuando el vídeo comienza a reproducirse por primera vez, pero no se cuenta con una reproducción (a menos que el usuario vuelva a cargar la página o vea otro vídeo en la lista de reproducción, etc.).
    • Las solicitudes de informes simultáneas dentro de la misma cuenta se ejecutarán en serie. Si los informes tienen rangos de fechas grandes y se seleccionan muchos campos, puede haber un retraso antes de que se devuelvan todos los resultados. Una buena alternativa es programar informes grandes.
    • Con las experiencias In-Page, aunque los vídeos parezcan estar cargados en un reproductor, los vídeos no se cargan hasta que el espectador haga clic en la superposición del botón de reproducción. Por lo tanto, las impresiones de vídeo no serán grabadas por el módulo Analytics hasta que el espectador reproduzca un vídeo.

    El selector de informes debajo del gráfico se puede utilizar para cambiar rápidamente a otro informe.

    La lista desplegable a la derecha del selector de informes se puede utilizar para seleccionar una dimensión de informe. Las siguientes dimensiones están disponibles con el informe de rendimiento:

    • Vídeo
    • Jugador
    • Origen del tráfico
    • Mostrar dominio
    • País
    • Dispositivo
    • Tipo de dispositivo
    • SO del dispositivo
    • Fabricante del dispositivo
    • Tipo de navegador
    • Día

    Tenga en cuenta que las dimensiones disponibles cambiarán según el informe seleccionado. La parte inferior del informe contiene unos controles que le permiten seleccionar cuántas filas desea mostrar, así como navegar a otras páginas.

    Comparar con periodo anterior

    Si se activa la casilla de verificación Comparar con periodo anterior, se sobreimpresionará un gráfico de líneas de trazos para mostrar los datos correspondientes al periodo anterior.

    El periodo anterior queda definido por la selección actual del intervalo de fechas del informe. Por ejemplo, si se selecciona el rango de fechas correspondiente a los últimos 7 días, el periodo anterior será 2 semanas atrás. La mejor forma de comparar las tendencias de varias semanas consiste en seleccionar rangos de fechas que abarquen semanas completas. Por ejemplo, si selecciona un lunes como fecha de comienzo, ajuste un domingo como fecha de finalización. Si la fecha de comienzo es un viernes, ajuste un jueves como fecha de finalización. También puede desplazar el ratón sobre las barras y la línea de trazos para ver la fecha y los valores correspondientes.

    Detalles del informe de rendimiento

    La primera columna que aparece en el cuerpo del informe variará en función de la dimensión del informe seleccionado (nombre del vídeo, nombre del reproductor, país, tipo de dispositivo, etc.). Todos los informes mostrarán los datos siguientes:

    • Visionados del vídeo: número de veces que se ha comenzado a reproducir un vídeo. Este contador se incrementa cada vez que se comienza a retransmitir la secuencia (las operaciones de rebobinado y de repetición no cuentan). No mide el número de espectadores individuales.
    • Número de impresiones del vídeo: número de veces que un vídeo fue cargado en un reproductor y quedó listo para la interacción con el usuario.
    • Tasa de reproducciones: porcentaje de usuarios que realmente reprodujeron el vídeo después de haberlo cargado (los visionados del vídeo divididos por las impresiones del vídeo)
    • Minutos vistos: número total de minutos de visionado del vídeo
    • Índice de interés despertado: promedio porcentual de tiempos de visionado (se calcula añadiendo todos los porcentajes de visionado, divididos por el número de visionados del vídeo)

    Orígenes de tráfico

    Al ver el gráfico Rendimiento por origen de tráfico, las posibles fuentes de tráfico son:

    • Referencia: el usuario ha seguido un enlace para llegar al vídeo
    • Directo: el usuario escribió el enlace directamente o el referenciador no ha podido ser determinado
    • Búsqueda segura: el usuario inició una sesión con una cuenta de Google, ejecutó una búsqueda e hizo clic en un resultado (de pago u orgánico; en este caso no se incluyen los datos de la búsqueda)
    • Búsqueda orgánica: el usuario ejecutó una búsqueda e hizo clic en un resultado para acceder al vídeo
    • Búsqueda de pago: el usuario ejecutó una búsqueda e hizo clic en un resultado de pago para acceder al vídeo

    Exportación de datos de informes

    Los datos del informe se pueden descargar en dos formatos: archivo con valores separados por comas y Excel. Tenga en cuenta que la función de exportación guarda los datos directamente de una llamada.Analytics API Los datos exportados no se procesan del mismo modo que los datos de la interfaz de usuario, por lo que actualmente contienen todos los datos subyacentes al informe, es decir, no solo el subconjunto de campos visibles en la interfaz de usuario. Algunas diferencias entre lo que se muestra en la interfaz de usuario frente a los datos que se exportan:

    • Unidades diferentes (minutos en lugar de segundos)
    • Redondeo de números
    • Los datos exportados contienen datos tanto de rendimiento como de interés despertado, independientemente de dónde se encuentre usted en la interfaz de usuario
    • La entrada Otro/Tercero se exportará sin asociarla a ningún identificador o nombre de vídeo

    Disponer de todos los datos en el formato original de la API permite procesarlos de maneras distintas a las que se ofrecen en la interfaz de usuario.

    Los informes disponen de plantillas de impresión que permiten enviarlos en un formato imprimible a archivos PDF o impresoras.

    Visualización de los análisis de vídeos

    Al seleccionar la dimensión Vídeo, aparecerá un gráfico que muestra el número de visionados diarios de un vídeo correspondiente al periodo seleccionado. En el cuerpo del informe se indican el nombre, el identificador, la duración y los detalles de rendimiento de cada vídeo.

    Al hacer clic en el enlace de cualquiera de los vídeos, aparecerá el interés despertado por el mismo.

    Nota: Si un vídeo que no está en la biblioteca de Video Cloud se reproduce en un reproductor de Video Cloud (por ejemplo, un recurso remoto), se mostrará Otros/Terceros sin ningún ID de vídeo asociado.

    Visualización de análisis de reproductores

    Al seleccionar la dimensión Reproductor, aparecerá un gráfico que muestra el número de visionados diarios de un vídeo correspondientes al periodo seleccionado. En el cuerpo del informe aparecen los detalles de rendimiento de cada reproductor.

    Al hacer clic en el enlace de cualquiera de los reproductores, aparecerá el interés despertado por el mismo.

    Nota: Hasta febrero de 2016, los reproductores utilizados en aplicaciones creadas con el SDK de reproductores de Brightcove para iOS o para Android se mostraban con el nombre de reproductor Reproducción personalizada o nativa. Desde febrero de 2016, estos reproductores se pueden identificar mediante los valores de origen y de destino especificados en la aplicación. Para obtener más información, consulte la documentación del SDK para iOS o para Android.

    Visualización de análisis de orígenes del tráfico

    Al seleccionar la dimensión Origen del tráfico, aparecerá un gráfico que muestra el número de visionados diarios de un vídeo correspondientes al periodo seleccionado. El cuerpo del informe mostrará los datos de rendimiento de cada fuente de tráfico. Solo se mostrarán las categorías que contengan datos correspondientes al período de tiempo seleccionado.

    Al hacer clic en el enlace de cualquiera de los orígenes del tráfico aparecerá el rendimiento correspondiente al origen de tráfico seleccionado.

    Visualización de análisis de dominios de visualización

    Al seleccionar la dimensión Dominio de visualización, aparecerá un gráfico que muestra el número de visionados diarios de un vídeo correspondientes al periodo seleccionado. El cuerpo del informe mostrará los datos de rendimiento de los dominios de visualización. El dominio de visualización es el dominio desde el que el espectador visionó el vídeo.

    Al hacer clic en el enlace de cualquiera de los dominios de visualización aparecerán las URL correspondientes al dominio de visualización seleccionado.

    Nota: Si el reproductor se encuentra en un iframe, se registrará como dominio de visualización la URL de la página en dicho iframe.

    Visualización de análisis por países

    Al seleccionar la dimensión País, aparecerá un gráfico que muestra el número de visionados diarios de un vídeo correspondientes al periodo seleccionado. El cuerpo del informe mostrará los datos de rendimiento de cada país en el que se visionaron los vídeos. Solo se mostrarán los países que contengan datos correspondientes al período de tiempo seleccionado.

    Nota: Brightcove utiliza un servicio de terceros para proporcionarnos la dirección IP y la información de ubicación. Si no hay ninguna entrada para la dirección IP en cuestión, no podremos obtener la información del país y aparecerá Desconocido .

    Al hacer clic en el enlace de cualquiera de los países se mostrará el rendimiento por región o ciudad para el país seleccionado, si está viendo análisis de la cuenta, no un video individual.

    Visualización de análisis por dispositivos

    Al seleccionar la dimensión Dispositivo, aparecerá un gráfico que muestra el número de visionados diarios de un vídeo correspondientes al periodo seleccionado. El cuerpo del informe mostrará los datos de rendimiento de las diferentes combinaciones de tipo y sistema operativo del dispositivo utilizadas para ver un vídeo. Solo se mostrarán los dispositivos que contengan datos correspondientes al período de tiempo seleccionado.

    Al hacer clic en el enlace de cualquiera de los tipos de dispositivos, aparecerá el rendimiento correspondiente al sistema operativo del mismo.

    Nota: El módulo Analytics determina el dispositivo a partir del encabezado de agente de usuario (user-agent). Si alguien utiliza un dispositivo con encabezados falsos, los datos se contabilizarán con respecto al encabezado declarado. El módulo Analytics no incorpora ninguna función para impedir la manipulación del encabezado de agente del usuario.

    Visualización de análisis por tipos de dispositivos

    Al seleccionar la dimensión Tipo de dispositivo, aparecerá un gráfico que muestra el número de visionados diarios de un vídeo correspondientes al periodo seleccionado. El cuerpo del informe mostrará los detalles de rendimiento por tipo de dispositivo. Solo se mostrarán las categorías que contengan datos correspondientes al período de tiempo seleccionado.

    Al hacer clic en el enlace de cualquiera de los tipos de dispositivos, aparecerá el rendimiento correspondiente al sistema operativo del mismo.

    Visualización de análisis por sistemas operativos de los dispositivos

    Al seleccionar la dimensión Sistema operativo del dispositivo, aparecerá un gráfico que muestra el número de visionados diarios de un vídeo correspondientes al periodo seleccionado. El cuerpo del informe mostrará los datos de rendimiento de cada sistema operativo de dispositivo. Solo se mostrarán las categorías que contengan datos correspondientes al período de tiempo seleccionado.

    Visualización de análisis del fabricante de dispositivos
