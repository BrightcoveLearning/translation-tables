    內容傳遞選項

    在本主題中，您將了解使用 Video Cloud 時可用的各種內容傳遞選項。

    簡介

    建立 Video Cloud 帳戶後，首先要決定的事情之一，就是您計劃如何傳遞影片檔案。Video Cloud 提供多種優點和限制的選擇。您選擇的傳送方式取決於您希望影片檔案受到的保護程度、接觸觀眾的方式、您偏好的上傳方式、您的視訊資產是否已儲存在另一個 CDN (內容傳遞網路或內容發佈網路) 等等。

    本主題中所述的內容傳遞選項僅適用於 Video Cloud 企業發行者。Video Cloud Enterprise和Express發布者必須使用Brightcove作為其CDN。

    在本主題中，您將學習：

    • 什麼是CDN？
    • 我可以在沒有CDN的情況下交付內容嗎？
    • 我的媒體內容如何傳遞？（漸進式下載與流式傳輸）
    • 比較內容交付：Brightcove託管的視頻資產vs. BYO與遠程資產
    • 配置內容交付以使用BYO CDN或遠程資產
    • Brightcove對BYO CDN支持哪些CDN？
    • 視頻雲報告和BYO CDN或遠程資產

    什麼是 CDN？

    出版商使用 CDN 將其媒體廣泛地分發給線上觀眾。維基百科將CDN定義為：

    內容交付網絡或內容分發網絡（CDN）是跨Internet聯網在一起的計算機系統，它們透明地協作以最頻繁地交付內容，以提高最終用戶的性能，可伸縮性和成本效率。

    有許多市售的 CDN。如果您是Video Cloud Enterprise或Express發布商，Brightcove是您的CDN解決方案。如果您是Video Cloud Enterprise發布商，則可以選擇使用Brightcove CDN或更高版本交付內容Video Cloud支持的任何CDN。

    我可以在沒有 CDN 的情況下傳遞我的內容嗎？

    如果您預期流量不足，或完全控制影片流量，您可能不需要 CDN 即可為您的影片提供服務。例如，有些發佈商想要在內部 Wiki 或私人網路上散佈其內容。您的系統可能可以處理媒體的傳遞，而不需要外部 CDN。如果您想要更多有關 Brightcove 如何支援此設定的資訊，請聯絡 Brightcove 客戶支援或您的客戶經理。

    我的媒體內容是如何傳遞的？

    內容傳遞機制有數種類型：

    • 流媒體
    • 漸進式下載

    串流 (也稱為 HLS 或短跑)

    流視頻是通過流服務器傳遞的，而文件從未下載到查看器的計算機/設備。一旦觀眾按下播放，視頻將開始播放。如果用戶決定轉發或跳到視頻的其他部分，他/她可以立即這樣做，視頻將繼續從該點開始播放。串流媒體的其中一個優點是，頻寬僅用於觀眾觀看過的影片，因為只有影片的觀看部分已傳遞。沒有任何東西保留在客戶端; 一切都在服務器端。

    請注意，在HTML5視頻播放器時代，所有現代設備和瀏覽器都支持流傳輸，並且會在可用時使用流傳輸（而不是逐步下載）。這是一件好事，因為如果您不觀看整個視頻，它可以更好地管理緩衝並節省帶寬。

    在您想要或需要執行以下操作的情況下，串流非常有用：

    • 傳送長檔案 (10 分鐘以上) 或高位元速率檔案 (高解析度)
    • 利用 Video Cloud 多位元速率串流功能，可執行頻寬偵測並為可用硬體提供最佳品質；您可以透過漸進式下載使用多位元速率串流，但頻寬偵測只會在播放開始時發生，播放程式不會」不適應觀眾頻寬的變化
    • 以較少的頻寬提供更多串流
    請注意，您可能會遇到 RTMP 串流的參照。這是較舊的 Flash 串流格式，現在很少使用。

    串流至 iOS 裝置

    蘋果 iOS 裝置只支援蘋果 HTTP 直播串流 (HLS) 和 HTTP (漸進式下載) 來傳送視訊檔案。您必須確保您的視訊有一或多個可用的 HLS 轉譯。應用程式商店規則要求使用 Apple HTTP 即時串流來處理長形式視訊內容 (大於 5 MB 或 10 分鐘)。

    串流關鍵點：

    • 通過臨時視頻鏈接保護視頻內容
    • 推進視頻是可能的
    • 有效利用視訊雲端的動態傳遞功能
    • 讓您以更少的頻寬提供更多蒸汽
    • 使用中的 Apple HTTP 即時串流 (HLS) 或即時訊息通訊協定 (RTMP) 通訊協定

    實際上，您很少會意識到內容是否已經串流或漸進式下載，除非您尋找上述的某些辨別功能。

    漸進式下載 (也稱為 PD)

    僅舊版本的瀏覽器或設備仍需要逐步下載。現在，它的主要用途是提供可由用戶下載和保存的演繹版本。

    當您使用漸進式下載傳送內容時，檔案會透過 HTTP 要求從標準 Web 伺服器提供，就像一般網頁或任何其他可下載的文件一樣。播放視訊時，視訊檔案會下載至檢視器的電腦或裝置，並儲存在暫存目錄中。當檔案已下載足夠的電腦/裝置時，影片將開始播放。如果觀眾想要快轉或跳到視頻的其他部分，他/她只能這樣做，如果該部分的視頻已經下載並存儲。與串流視訊相比，漸進式下載實際上只有一個一致的好處 — 您不需要串流伺服器即可傳遞視訊。漸進式下載視頻可以從任何一般的 Web 服務器提供。

    漸進式下載對於業餘愛好者或流量要求低的網站來說很好，不要介意他們的內容是否緩存在觀眾的計算機/設備上，如果您只需要傳遞較短的長度視頻（10 分鐘以下）。需要進階功能和控制視訊傳遞的發佈商，以及/或需要向較大觀眾顯示視訊的人 (數百位或以上同步觀眾)、需要追蹤和報告使用情形或觀看影片的統計資料，或是想要提供最佳互動式播放體驗的發佈商，將需要串流他們的視頻。串流傳遞的頻寬也比漸進式傳遞更少，因為實際上只有觀看的視訊部分才會傳遞。

    漸進式下載要點：

    • 搜尋與導覽功能有限
    • 觀眾可以存取並重新使用內容
    • 無法有效利用視訊雲端的動態傳遞功能
    • 安全性較低，內容可以很容易地複製
    • 下載影片的所有部分，而不僅僅是觀看的部分，所以會使用更多頻寬

    如何傳送影像資產？

    無論您是否使用串流或漸進式下載的視訊內容，傳遞給 Video Cloud 播放器中使用者的影像 (例如縮圖和視訊靜止影像) 都會透過 HTTP 下載傳送給使用者。

    比較內容傳遞：光線託管的視訊資產與 BYO 與遠端資產

    視訊雲端支援一些基本的 CDN 設定，視您的 CDN 供應商和設定而定。選擇是：

    • Brightcove CDN
    • 自帶CDN
    • 遠程資產
    • 多項選擇

    布萊特灣 CDN

    您可以選擇使用Brightcove CDN。Video Cloud 運用 Akamai、FastY 和其他第 1 層 CDN，為您的視訊內容提供串流和漸進式下載傳遞。通常會使用 HLS 或 DASH 串流，漸進式下載僅在特殊情況下使用，或提供視訊的可下載轉譯。

    由 CDN

    您也可以選擇使用自己選擇的CDN（自帶CDN或“自帶帶寬”）。在這種情況下，視您與 CDN 供應商的協議而定，您的視頻將通過視頻雲端播放器使用漸進式下載（PD）或串流機制無縫傳遞您的視頻。如果您選擇 BYO CDN，您可以使用視訊雲端提取式擷取內容，將內容嵌入到您的 Video Cloud 帳戶中，然後移至您的 CDN 供應商，透過視訊雲端播放器傳送給最終使用者。

    遠端資產

    最終的選擇是遠程資產。在這種情況下，您可能已經有您的視頻文件存儲在 CDN 中，並且您不希望使用視頻雲上傳工具在所有添加新的內容。您可以使用Video Cloud Media模塊，Dynamic Ingest API或CMS API在您的Video Cloud帳戶中創建指向由CDN提供商遠程存儲的基礎視頻資產的視頻。

    限制

    當您建立具有遠端資產的視訊時，您無法使用影像擷取功能建立遠端資產視訊的靜態影像和縮圖影像。可上傳影片靜態影像和縮圖影像，以供遠端資產影片使用。

    有關使用遠程資產的信息，請參閱使用遠程資產視頻。

    多重選擇

    您可以將視訊雲端帳戶設定為同時使用 BYO CDN 和遠端資產。舉例來說，如果您的 CDN 中已有部分視訊內容，您可能會想要為現有內容使用遠端資產，以避免再次上傳，並使用 BYO CDN 來進行新上傳。如果您需要此設定，請與我們聯絡。您也可以建立多個 Video Cloud 帳戶，每個帳戶都有不同的內容傳遞策略。

    設定內容傳遞以使用 BYO CDN 或遠端資產

    以下是使用 BYO CDN 或遠程資產設置的主要步驟：

    1. 設置一個 CDN 的帳戶（請參閱以下章節以查看當前支持的帳戶）。
    2. 如果您有需要遷移的現有視頻內容（從 Brightcove 或外部 CDN），請聯繫我們的服務團隊。
    3. 請連絡您的視訊雲端客戶經理以完成設定。

    視訊雲端支援什麼 CDN？

    視訊雲端支援下列主要 CDN 用於 BYOB CDN 內容傳遞：

    • 阿卡
    • 雲端
    • 快
    • JOCDN
    • 聚光燈

    可能支援其他 BYO CDN，但功能支援可能會有所不同。聯繫銷售或您的客戶經理以獲取更多信息。

    BYO CDN 的支援功能

    以下是 BYO CDN 支援的功能：

    • HTTP/
    • 僅限 HTTPS
    • 簽署的資訊清單 URL
    • 已簽署的內容 URL (僅適用於上面列出的 CDN)
    • CDN 地理限制 (如果 CDN 支援)
    • CDN IP 限制（如果 CDN 支持）
    • 自訂 TTL 設定 (如果 CDN 支援)

    視訊雲端報告與 BYO CDN 或遠端資產
