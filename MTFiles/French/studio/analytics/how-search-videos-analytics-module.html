    Comment rechercher des vidéos dans le module Analytics

    Dans cette rubrique, vous apprendrez à rechercher des vidéos précises avec la fonctionnalité Quick Find du module de statistiques.

    Fonctionnement de la recherche rapide

    La fonctionnalité Quick Find du module de statistiques permet de trouver une vidéo spécifique. Après avoir entré des critères de recherche, Recherche rapide recherche le nom de la vidéo, la description courte, la description longue, les balises, l'ID vidéo et tous les champs de texte personnalisés que vous pourriez avoir. Une liste de correspondances s'affiche sous le champ de recherche.

    Notez que supprimé Les vidéos ne sont pas consultables via la fonctionnalité de recherche, mais nous conservons des analyses pour elles. Vous pouvez les trouver en développant/naviguant dans le rapport de performances et en utilisant la fonction de recherche du navigateur ou en téléchargeant le rapport et en effectuant une recherche sur une application tableur.

    Cliquez sur le lien d'une correspondance pour ouvrir le rapport « en contexte ». Par exemple, si vous consultez les statistiques d'engagement de votre compte et que vous vous servez de la recherche rapide pour localiser une vidéo précise, cliquer sur le lien de la vidéo ouvrira le rapport sur l'engagement. Si vous examinez les statistiques des performances, faire une recherche et cliquer sur un lien ouvrira le rapport des performances.

    Recherche basique

    Si vous fournissez une valeur de recherche sans qualificatifs, cela exécutera une recherche de texte dans les champs suivants :

    • Prénom
    • Short description
    • Long description
    • Balises
    • ID de la vidéo
    • ID de référence
    • Tous les champs personnalisés

    Si vous voulez faire correspondre plusieurs mots, il suffit d'envelopper le terme entre guillemets.

    Recherche avancée

    Le module de statistiques permet également de rechercher dans des champs spécifiques, rechercher par date et inclure/exclure des termes précis.

    Recherche dans des champs spécifiques

    Pour rechercher une valeur donnée dans un champ spécifique, servez-vous de cette syntaxe :

    • field:value to search

    Les champs suivants sont pris en charge :

    Champ Valeurs acceptées
    name chaînes de caractères entre guillemets ou non
    text chaînes ou chaînes entre guillemets (valeurs dans name , short description , long description)
    tags chaînes de caractères entre guillemets ou non
    reference_id chaîne de caractères ou chaînes de caractères entre guillemets
    state ACTIVE, INACTIVE , DELETED , PENDING
    updated_at consultez la section Recherche par date pour plus de détails sur le format des dates
    created_at consultez la section Recherche par date pour plus de détails sur le format des dates
    published_at consultez la section Recherche par date pour plus de détails sur le format des dates
    complete true ou false

    Si vous voulez trouver une correspondance exacte ou une correspondance de plusieurs mots, saisissez le ou les termes entre guillemets :

    • "live"
    • "live streaming"

    Cela fonctionne également lors de la recherche dans un champ spécifique :

    • name:"live"
    • name:"live streaming"

    Recherche par date

    Les champs de recherche par date acceptent des valeurs de plages de date. Différents formats de date sont pris en charge :

    Format C'est-à-dire
    2013-08-01 Date au format aaaa-mm-jj
    2013-08-01T00:00:00Z L'heure au format UTC
    -1d L'heure actuelle moins un jour (voir les dates relatives)
    +1d L'heure actuelle plus un jour (voir les dates relatives)
     

    Dates relatives

    Dans le cas des dates relatives, nous acceptons une direction temporelle (+ ou -) suivie d'un nombre et d'une durée (ou de son abréviation). Les durées légales sont les suivantes : minutes (m), heures h, jours d

    Par exemple :

    • -1day
    • -2days
    • +4hours
    • -60minutes
    • -1d
    • -2d
    • +4h
    • -60m

    Plages de dates

    Si le champ concerné par la recherche est une date, vous pouvez spécifier une plage de dates de recherche :

    • updated_at:2013-08-01T00:00:00Z..2013-10-08T00:00:00Z

    Cela recherchera toutes les vidéos avec un updated_at valeur entre le 1er août 2013 et le 8 octobre 2013.

    Plages ouvertes

    Si vous voulez inclure toutes les dates jusqu'à un moment donné, ou inclure toutes les dates à partir d'un moment donné, effacez la partie de la plage concernée. Par exemple, dans le cas d'une recherche de toutes les vidéos modifiées durant les deux derniers jours :

    • q=updated_at:-2days..

    Recherche de toutes les vidéos modifiées avant le 11 août 2013 :

    • q=updated_at:..2013-08-11T00:00:00Z

     

    Inclusion/exclusion de termes de recherche spécifiques

    Vous pouvez exiger qu'un terme soit obligatoire (les vidéos trouvées DOIVENT le mentionner) ou interdit (les vidéos trouvées ne doivent PAS le mentionner). Ceci est contrôlé avec un + ou - signe précédant immédiatement le terme:

    Exemple C'est-à-dire
    +animals vidéos DOIT inclure le terme `` animaux '' dans name , short description , ou long description
    -animals les vidéos doivent NE PAS inclure le terme `` animaux '' dans name , short description , ou long description
    +name:animals les vidéos DOIVENT inclure le terme « animaux » dans leur nom.
    -name:animals les vidéos NE DOIVENT PAS inclure le terme « animaux » dans leur nom.
