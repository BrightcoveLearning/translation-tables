페이지 내용

    API 인증 자격 증명 관리

    이 항목에서는 API 인증 자격 증명 만들기 및 관리에 대한 개요를 제공합니다.
     

    Video Cloud API를 사용하는 경우 플레이어 서비스 설정에서 기본 인증을 활성화하지 않는 한 메서드 호출에 액세스 토큰이 필요합니다. 기본 인증 활성화에 대한 자세한 내용은계정 설정 관리를참조하십시오.

    참고: 이 기능은 프로페셔널 및 엔터프라이즈 계정에만 사용할 수 있습니다.

    클라이언트 등록 생성

    API 인증 페이지는 새 클라이언트 등록을 만드는 데 사용됩니다. 등록을 생성할 때 등록으로 활성화할 서비스 및 역할을 선택할 수 있습니다.

    계정에 대한 새 클라이언트 등록을 생성하려면 다음과 같이 하십시오.

    1. Studio 헤더에서ADMIN링크를 클릭합니다.
    2. API 인증링크를 클릭합니다. API 인증 페이지가 열리고 현재 클라이언트 등록이 표시됩니다.
      참고: 로그인한 계정에 대한 클라이언트 등록만 표시됩니다.
    3. 새 응용 프로그램 등록을클릭합니다.
    4. 클라이언트 등록에 대한이름및설명을입력합니다.
    5. 승인을 위해 계정을 하나 이상 선택합니다.
    6. 이 등록에 사용할 API 및 권한을 선택합니다.
      참고: 많은 수의 계정 및 API를 선택하면 HTTP 헤더 크기가 크고 유효하지 않게 될 수 있습니다. 서버에서 허용하는 최대 헤더 크기는 8K입니다. 즉, 자격 증명을 적용하려면 30 개 이하의 계정을 선택해야 합니다. 보안상의 이유로 자격 증명 집합에 대한 사용 권한 범위를 필요한 최소 계정 및 작업 집합으로 제한하는 것이 좋습니다.
    7. 저장을 클릭합니다.
    8. 클라이언트 ID및클라이언트 암호가반환됩니다.
      참고: 클라이언트 암호가 다시 표시되지않으므로클라이언트 암호를 복사하여 안전한 위치에 저장하십시오.

    클라이언트 등록 관리

    API 인증 페이지를 사용하여 클라이언트 등록을 보고 관리할 수 있습니다. Studio 헤더에서ADMIN링크를 클릭한 다음API 인증을클릭합니다. 현재 등록 목록이 표시됩니다.

    등록 세부 정보를 보려면 이름을 클릭하십시오. 여기에서 다음을 수행할 수 있습니다.

    • 이름 수정
    • 클라이언트 ID 복사
    • 권한 부여를 위한 계정 추가
    • 활성화할 추가 API 선택
    • 자격 증명 삭제

    저장을클릭하여 변경 사항을 저장합니다.

    여러 계정에 대해 자격 증명이 생성되면 모든 계정에 액세스할 수 있는 경우에만 해당 자격 증명을 볼 수 있습니다. 예를 들어 사용자 1이 계정 A 및 B의 자격 증명을 만듭니다. 사용자 2가 로그인하고 계정 A에 대한 액세스 권한만 있습니다. 사용자 2는 계정 B에 대한 액세스 권한이 없으므로 자격 증명을 볼 수 없습니다.

    클라이언트 등록 삭제

    클라이언트 등록을 삭제하려면 등록 옆에 있는 삭제 아이콘 ( ) 을 클릭하거나 등록을 편집하고 등록삭제를클릭합니다. 등록을 삭제하면 영구적으로 비활성화되며 취소할 수 없습니다.

    DFP 작업

    현재 여러 Video Cloud 계정이 DFP에서 콘텐츠 소스로 설정되어 있는 경우 다음 두 가지 옵션이 있습니다.

    1. DFP에서 콘텐츠 원본으로 설정된 각 Video Cloud 계정에 대해 고유한 클라이언트 ID 및 클라이언트 암호를 만들 수 있습니다. 예를 들어 비디오 클라우드 계정이 10개인 경우 Video Cloud의 API 인증 설정에 DFP라는 애플리케이션이 10개 있을 수 있습니다.
    2. DFP에서 콘텐츠 소스로 설정된 모든 Video Cloud 계정에 대해 단일 클라이언트 ID 및 클라이언트 암호를 생성할 수 있습니다. 예를 들어 비디오 클라우드 계정이 10개인 경우 Video Cloud의 API 인증 설정에 DFP라는 애플리케이션 1개가 있을 수 있습니다.

    #1 또는 #2 옵션을 선택했는지 여부에 관계없이 DFP에서 설정한 각 콘텐츠 원본에 대해 고유한 계정 ID를 제공해야 합니다. 따라서 DFP에 현재 설정되어 있는 콘텐츠 원본의 수는 줄어들지 않습니다. 읽기 토큰을 클라이언트 ID 및 클라이언트 암호로 바꾸기만 하면 됩니다. 이 암호는 콘텐츠 원본에서 동일하거나 각각에 대해 고유할 수 있습니다.

    #1 옵션은 더 많은 시간이 소요됩니다. 그러나 클라이언트 ID와 클라이언트 암호가 모든 콘텐츠 원본에서 동일한 경우 (옵션 #2) 새 클라이언트 ID와 클라이언트 암호를 생성하고 DFP의 모든 콘텐츠 원본을 업데이트해야 하므로 새 콘텐츠 원본을 추가하는 부담이 됩니다. 따라서 브라이트코브는 #1 옵션을 권장합니다.
