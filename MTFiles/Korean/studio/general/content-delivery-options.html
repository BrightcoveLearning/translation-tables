페이지 내용

    콘텐츠 전송 옵션

    이 항목에서는 Video Cloud를 사용할 때 사용할 수 있는 다양한 콘텐츠 전송 옵션에 대해 알아봅니다.

    서문

    Video Cloud 계정을 만든 후 가장 먼저 결정해야 할 사항 중 하나는 비디오 파일을 전송할 계획입니다. Video Cloud는 다양한 장점과 한계가 있는 다양한 옵션을 제공합니다. 선택하는 전송 방법은 동영상 파일의 보호 방법, 시청자에게 도달하는 방법, 선호하는 업로드 방법, 동영상 자산이 이미 다른 CDN (콘텐츠 전송 네트워크 또는 콘텐츠 배포 네트워크) 에 저장되어 있는지 여부에 따라 달라집니다.

    이 항목에 설명된 콘텐츠 전송 옵션은 Video Cloud Enterprise 게시자만 사용할 수 있습니다. Video Cloud Enterprise 및 Express 게시자는 Brightcove를 CDN으로 사용해야합니다.

    이 항목에서는 다음 내용을 배우게 됩니다.

    • CDN이란 무엇입니까?
    • CDN 없이 콘텐츠를 전송할 수 있습니까?
    • 미디어 콘텐츠는 어떻게 전달됩니까? (점진적 다운로드 대 스트리밍)
    • 콘텐츠 전송 비교: 브라이트코브 호스팅 비디오 자산과 BYO 대 원격 자산
    • BYO CDN 또는 원격 자산을 사용하도록 콘텐츠 전송 구성
    • 브라이트코브가 BYO CDN을 지원하는 CDN은 무엇입니까?
    • 비디오 클라우드 보고 및 BYO CDN 또는 원격 자산

    CDN이란 무엇입니까?

    게시자는 CDN을 사용하여 온라인 시청자에게 미디어를 널리 배포합니다. 위키 백과는 CDN을 다음과 같이 정의합니다.

    콘텐츠 전송 네트워크 또는 CDN (콘텐츠 배포 네트워크) 은 인터넷을 통해 네트워크로 연결된 컴퓨터 시스템으로, 최종 사용자에게 성능, 확장성 및 비용 효율성을 향상시키기 위해 가장 자주 콘텐츠를 제공하기 위해 투명하게 협력합니다.

    상업적으로 이용 가능한 많은 CDN이 있습니다. Video Cloud Enterprise 또는 Express 게시자 인 경우 Brightcove가 CDN 솔루션입니다. 비디오 클라우드 엔터프라이즈 퍼블리셔인 경우 브라이트코브 CDN 또는비디오 클라우드가 지원하는 CDN을 통해 콘텐츠를 전송하도록 선택할 수 있습니다.

    CDN 없이 콘텐츠를 전송할 수 있습니까?

    트래픽이 적거나 동영상의 트래픽을 완벽하게 제어할 수 있는 경우 동영상을 제공하는 데 CDN이 필요하지 않을 수 있습니다. 예를 들어 일부 게시자는 내부 Wiki 또는 개인 네트워크에 콘텐츠를 배포하려고 합니다. 외부 CDN을 사용하지 않고도 시스템에서 미디어 배달을 처리할 수 있습니다. 브라이트코브가 이 설정을 지원하는 방법에 대한 자세한 내용은 브라이트코브 고객 지원부 또는 계정 관리자에게 문의하십시오.

    미디어 콘텐츠는 어떻게 전달됩니까?

    콘텐츠 전송 메커니즘에는 다음과 같은 몇 가지 유형이 있습니다.

    • 스트리밍
    • 프로그레시브 다운로드

    스트리밍 (HLS 또는 DASH라고도 함)

    스트리밍 비디오는 뷰어의 컴퓨터 / 장치에 파일을 다운로드하지 않고 스트리밍 서버를 통해 전달됩니다. 시청자가 재생을 누르는 즉시 비디오가 재생되기 시작합니다. 사용자가 비디오의 다른 부분으로 전달하거나 건너뛰기로 결정하면 즉시 할 수 있으며 동영상은 해당 시점부터 계속 재생됩니다. 스트리밍 미디어의 장점 중 하나는 시청자가 시청한 비디오에만 대역폭이 사용된다는 것입니다. 비디오의 시청한 부분만 전송되었기 때문입니다. 클라이언트 측에는 아무 것도 보관되지 않습니다. 모든 것이 서버 측에 있습니다.

    HTML5 비디오 플레이어 시대에는 모든 최신 장치와 브라우저가 스트리밍을 지원하며 가능한 경우이를 사용합니다 (점진적 다운로드가 아님). 시청자가 전체 비디오를 보지 않으면 버퍼링을 더 잘 관리하고 대역폭을 절약 할 수 있기 때문에 이것은 좋은 것입니다.

    스트리밍은 다음을 수행하거나 수행해야 하는 경우에 유용합니다.

    • 긴 파일 (10분 이상) 또는 높은 비트레이트 파일 (고화질) 전송
    • 대역폭 감지를 수행하고 사용 가능한 하드웨어에 대해 최상의 품질을 제공하는 Video Cloud 다중 비트 전송률 스트리밍 기능을 활용하십시오. 점진적 다운로드로 멀티 비트 전송률 스트리밍을 사용할 수 있지만 대역폭 감지는 재생 시작 시에만 발생하며 플레이어는시청자의 대역폭 변화에 적응할 수 있음
    • 더 적은 대역폭으로 더 많은 스트림 제공
    RTMP 스트리밍에 대한 참조가 발생할 수 있습니다. 이것은 오래된 플래시 스트리밍 형식이며 지금은 거의 사용되지 않습니다.

    iOS 디바이스로 스트리밍

    애플 iOS 장치는 비디오 파일의 배달을 위해 애플 HTTP 라이브 스트리밍 (HLS) 및 HTTP (점진적 다운로드) 만 지원합니다. 동영상에 사용 가능한 HLS 변환이 하나 이상 있는지 확인해야 합니다. App Store 규칙은 긴 형태의 비디오 콘텐츠 (5Mb 또는 10분 이상) 에 대해 Apple HTTP 라이브 스트리밍을 사용하도록 요구합니다.

    스트리밍 핵심 포인트:

    • 비디오 콘텐츠는 임시 비디오 링크를 통해 보호됩니다.
    • 전진 비디오 가능
    • Video Cloud의 동적 전송 기능을 효과적으로 활용
    • 더 적은 대역폭으로 더 많은 증기를 제공 할 수 있습니다.
    • 사용 중인 Apple HTTP 라이브 스트리밍 (HLS) 또는 실시간 메시징 프로토콜 (RTMP) 프로토콜

    실제로 위에서 설명한 몇 가지 고유 기능을 찾지 않는 한 콘텐츠가 스트리밍되었는지 또는 점진적으로 다운로드되었는지 여부는 거의 인식하지 못합니다.

    프로그레시브 다운로드 (PD라고도 함)

    아주 오래된 브라우저 나 장치에만 점진적 다운로드가 필요합니다. 이제 주요 용도는 사용자가 다운로드하고 저장할 수있는 변환을 제공하는 것입니다.

    점진적 다운로드를 사용하여 콘텐츠를 전송하면 일반 웹 페이지나 기타 다운로드 가능한 문서와 마찬가지로 HTTP 요청을 통해 파일이 표준 웹 서버에서 제공됩니다. 비디오가 재생되면 비디오 파일이 뷰어의 컴퓨터 또는 장치로 다운로드되고 임시 디렉토리에 저장됩니다. 충분한 파일이 컴퓨터/장치에 다운로드되면 비디오가 재생되기 시작합니다. 시청자가 동영상의 다른 부분으로 빨리 감기 또는 건너뛰기를 원할 경우 동영상의 해당 부분이 이미 다운로드되어 저장된 경우에만 시청자가 그렇게 할 수 있습니다. 스트리밍 비디오와 비교할 때 점진적 다운로드에는 단 하나의 일관된 이점이 있습니다. 비디오를 전송하기 위해 스트리밍 서버가 필요하지 않습니다. 프로그레시브 다운로드 비디오는 일반 웹 서버에서 제공 될 수 있습니다.

    점진적 다운로드는 트래픽 요구 사항이 낮은 취미 또는 웹 사이트에 적합합니다. 시청자의 컴퓨터/장치에 콘텐츠가 캐시되는지, 짧은 길이의 동영상 (10분 미만) 만 제공하면 됩니다. 동영상 전송에 대한 고급 기능 및 제어 기능이 필요한 게시자 및/또는 더 많은 시청자에게 비디오를 표시해야 하는 퍼블리셔 (수백 명 이상의 동시 시청자), 비디오 사용 현황 또는 시청 통계를 추적 및 보고해야 하거나 최상의 대화형 재생 환경을 제공하려는 퍼블리셔는 비디오를 스트리밍해야합니다. 또한 스트리밍 전송은 시청한 비디오 부분만 실제로 전송되기 때문에 점진적 전송보다 대역폭을 적게 소비합니다.

    점진적 다운로드 핵심 사항:

    • 제한된 탐색 및 탐색 기능
    • 시청자가 콘텐츠에 액세스하고 용도를 변경할 수 있음
    • Video Cloud의 동적 전송 기능을 효과적으로 활용하지 못함
    • 보안 수준이 낮고 컨텐츠를 쉽게 복제할 수 있음
    • 시청한 비디오뿐만 아니라 비디오의 모든 부분이 다운로드될 때 더 많은 대역폭이 사용됩니다.

    이미지 자산은 어떻게 제공됩니까?

    비디오 콘텐츠에 스트리밍을 사용하든 점진적 다운로드를 사용하든 관계없이 Video Cloud 플레이어에서 최종 사용자에게 전송된 이미지 (예: 미리보기 이미지 및 비디오 스틸) 는 HTTP 다운로드를 통해 전송됩니다.

    콘텐츠 전송 비교: 브라이트코브 호스팅 비디오 자산과 BYO 대 원격 자산

    Video Cloud는 CDN 제공업체 및 설정에 따라 몇 가지 기본 CDN 구성을 지원합니다. 선택 사항은 다음과 같습니다.

    • 브라이트코브 CDN
    • 분류: 바이오 CDN
    • 원격 자산
    • 다중 선택

    브라이트코브 CDN

    브라이트코브CDN을사용하도록 선택할 수 있습니다. Video Cloud는 Akamai, Faststly 및 기타 계층 1 CDN을 활용하여 비디오 콘텐츠에 대한 스트리밍 및 점진적 다운로드 전송을 모두 제공합니다. 일반적으로 HLS 또는 DASH 스트리밍이 사용되며 점진적 다운로드는 특수한 경우에만 사용하거나 비디오의 다운로드 가능한 변환을 제공하기 위해서만 사용됩니다.

    바이오 CDN

    또한 원하는 CDN ( BYO CDN또는 “자신의 대역폭 가져 오기”) 을 사용하도록 선택할 수 있습니다. 이 경우 CDN 공급자와의 계약에 따라 점진적 다운로드 (PD) 또는 스트리밍 메커니즘을 사용하여 Video Cloud 플레이어를 통해 동영상이 원활하게 전송됩니다. BYO CDN을 선택하면 Video Cloud 풀 기반 수집을 사용하여 Video Cloud 계정으로 콘텐츠를 인제스트할 수 있습니다. 그런 다음 CDN 공급자로 이동하여 Video Cloud 플레이어를 통해 최종 사용자에게 전달할 수 있습니다.

    원격 자원

    최종 선택은원격 자산입니다. 이 경우 이미 CDN에 비디오 파일이 저장되어 있을 수 있으며 Video Cloud 업로드 도구를 사용하여 새 콘텐츠를 추가하지 않으려는 경우가 있습니다. 비디오 클라우드 미디어 모듈, 동적 인제스트 API 또는CMS API를 사용하여 CDN 공급자가 원격으로 저장한 기본 비디오 자산을 가리키는 비디오를 Video Cloud 계정으로 만들 수 있습니다.

    제한 사항

    원격 에셋으로 비디오를 만들 때는 이미지 캡처 기능을 사용하여 원격 에셋 비디오에 대한 비디오 스틸 및 축소판 이미지를 만들 수 없습니다. 동영상 정지 및 축소판 이미지를 원격 자산 비디오에 업로드할 수 있습니다.

    원격 자산 사용에 대한 자세한 내용은원격 자산 비디오 작업을 참조하십시오.

    다중 선택

    BYO CDN 및 원격 자산을 모두 사용하도록 Video Cloud 계정을 구성할 수 있습니다. 예를 들어, 일부 비디오 콘텐츠가 CDN에서 이미 사용 가능한 경우 기존 콘텐츠에 원격 자산을 사용하여 다시 업로드하지 않도록 하고 앞으로 새로운 업로드에 BYO CDN을 사용할 수 있습니다. 이 설정이 필요한 경우 당사에 문의하십시오. 또한 각각 다른 콘텐츠 전송 전략을 사용하는 여러 Video Cloud 계정을 만들 수도 있습니다.

    BYO CDN 또는 원격 자산을 사용하도록 콘텐츠 전송 구성

    다음은 BYO CDN 또는 원격 자산으로 설정하기 위한 주요 단계입니다.

    1. CDN을 사용하여 계정을 설정합니다 (현재 지원되는 항목을 보려면 다음 섹션 참조).
    2. 기존 비디오 콘텐츠를 마이그레이션해야 하는 경우 (브라이트코브 또는 외부 CDN에서) 서비스 팀에 문의하십시오.
    3. 설정을 완료하려면 Video Cloud 계정 관리자에게 문의하십시오.

    비디오 클라우드가 BYO CDN을 지원하는 CDN은 무엇입니까?

    비디오 클라우드는 BYOB CDN 콘텐츠 전송을 위해 다음과 같은 주요 CDN을 지원합니다.

    • Akamai
    • 클라우드프론트
    • 금식하게
    • 요크든
    • Limelight

    다른 BYO CDN이 지원될 수 있지만 기능 지원은 다를 수 있습니다. 자세한 내용은 영업 담당자또는 계정 관리자에게 문의하십시오.

    BYO CDN 지원 기능

    BYO CDN에 대해 지원되는 기능은 다음과 같습니다.

    • HTTP/HTTPS
    • HTTPS 전용
    • 서명 매니페스트 URL
    • 서명된 컨텐츠 URL (위에 나열된 CDN에만 해당)
    • CDN 지리적 제한 (CDN에서 지원하는 경우)
    • CDN IP 제한 (CDN에서 지원하는 경우)
    • 사용자 정의 TTL 설정 (CDN에서 지원하는 경우)

    비디오 클라우드 보고 및 BYO CDN 또는 원격 자산

    BYO CDN 또는 원격 에셋을 사용하는 경우 CDN과 통합되지 않으므로 다운로드된 기가바이트가 비디오 클라우드 스튜디오 분석에 표시되지 않습니다. (해당 시간은 브라이트코브 플레이어에서 보고하므로 시청 시간 (분) 을 확인할 수 있습니다.) CDN 공급자로부터 직접 이러한 보고서를 받아야 합니다. Video Cloud 보고서에서 “비동영상” 콘텐츠, 즉 Video Cloud 플레이어, 이미지 등에서 발생하는 대역폭 사용량에 대한 기가바이트 다운로드가 계속 표시될 수 있습니다.
