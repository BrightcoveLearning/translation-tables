    HLS暗号化を使って動画を保護する

    このトピックでは、Video Cloud HLS暗号化機能を使って、デスクトップデバイスおよびモバイルデバイスで再生する動画を保護する方法について説明します。ビデオクラウドHLS暗号化を使用する場合は、アカウントで有効にする必要があります。詳細については、アカウントマネージャーにお問い合わせください。アカウントで有効にすると、その後に取り込まれたすべての動画でHLSeが使用されることに注意してください。

    Video Cloud HLS暗号化について

    HTTP Live Streaming Encryption（HLSe）を使用する場合、コンテンツはAES 128標準で暗号化され、マニフェストにはHLSコンテンツの復号化を可能にするキーへのリンクが含まれます。キーを取得すると、コンテンツを簡単に復号化して再配布できるため、これだけでは完全なセキュリティは提供されません。HTTP経由での提供や、さまざまなトークン認証モデルなど、キーの保護を可能にするメカニズムはほとんどありません。HLSeは、コンテンツを取得しようとするほとんどの標準ユーザーからコンテンツを保護しますが、DRMレベルのコンテンツ保護とは見なされません。Video Cloudは、BrightcoveプレーヤーでHLS暗号化をサポートしています。Brightcoveプレーヤーサポートの詳細は「Brightcoveプレーヤーのシステム要件」を参照してください。また、チェックしてください制限以下のセクション。

    Video Cloud Enterpriseは、HLS暗号化を使用してコンテンツをどのように保護しますか？

    Apple HTTPライブストリーミング（HLS）は暗号化とは無関係に、さまざまなビットレートを使用する動画提供プロトコルです。Video Cloudは、ネットワーク帯域幅の変化やサービスの変動に応じてレンディションをインテリジェントに切り替える複数のレンディションの作成をサポートしています。HLSは基本的に、ビデオを一連の小さなファイルダウンロードに分割し、それぞれがHTTPを介してビデオの1つの短いチャンクまたはセグメントを一度にロードします。

    注：Appleは長編動画にHLSを必要とします。5 MBを超える、つまり10分以上の動画に必要となります。暗号化に関係なく、iOSデバイス上で約10分以上の動画を再生するには、HLSレンディションを作成する必要があります。

    Video Cloudは、Apple HLS向け動画レンディションの暗号化をサポートしています。これにより、パブリッシャーはHLSを通してデバイスへ配信される長編動画コンテンツを保護できます。Video Cloud HLS暗号化は標準HLSソリューションにAESを追加することによって、コンテンツを保護します。Apple HLSに暗号化を実施するとき、Video Cloudは動画を分割してできた小さなファイルのそれぞれを暗号化し、レンディション選択を処理するファイルを安全に提供します。

    サポートされる機能

    電子データの暗号化にAES仕様を利用することに加えて、Video Cloud HLS暗号化は、次の方法でコンテンツをさらに保護します。

    • 各セグメントファイルを暗号化する
    • Video Cloudによって配信されるHLSマニフェスト（.m3u8ファイル）には、各セグメントのキーへのリンクが含まれています
    • デフォルトでは、キーは10分ごとにローテーションされます。あなたは指定することができますencryption_key_rotation_periodカスタム取り込みプロファイルで。
    • 有効になっているアカウントのHLSレンディションに暗号化を追加するには動的配信、アカウントのHLSeを有効にするために、Brightcoveサポートにリクエストを送信するだけです。（HLSeが既に有効になっているアカウントの場合、アカウントで動的配信が有効になっていると、暗号化は引き続きすべてのHLSレンディションに適用されます。）
    • 従来の取り込みシステム（非動的配信）を使用してアカウントのHLSレンディションに暗号化を追加するには、encryption_methodそしてencryption_key_rotation_period取り込みプロファイルで定義された各HLSレンディションへのフィールド： 
      {
  "media_type": "video",
  "reference_id": "ts0",
  "format": "ts",
  "type": "segmented",
  "audio_codec": "aac",
  "audio_bitrate": 64,
  "video_codec": "h264",
  "encryption_method": "aes-128",
  "encryption_key_rotation_period": 10,
  "video_bitrate": 450,
  "decoder_bitrate_cap": 771,
  "decoder_buffer_size": 1028,
  "keyframe_rate": 0.5,
  "max_frame_rate": 30,
  "width": 480,
  "height": 270,
  "h264_profile": "baseline"
    注：ビデオ暗号化は、それ自体が強力なコンテンツ保護ではありません。コンテンツセキュリティがお客様の組織にとって極めて重要な場合、DRM保護の使用を推奨します。コンテンツをどこで再生しても保護できるように、HLS暗号化とDRM保護をバンドルする方法の詳細については、アカウントマネージャーにお問い合わせください。

    HLS暗号化実施後の状況

    Video Cloud HLS暗号化は安全なマルチビットレートエンコーディングを提供し、各レンディションおよび各レンディションの各セグメントは複数の手段で保護されます。動画の最初のレンディションがアップロードされ暗号化されると、HLSに暗号化された動画はデスクトップデバイスおよびモバイルデバイスでの再生に利用できるようになります。一度実装されると、その後にアップロードされた全動画はHLS暗号化を使用して保護されます。Video Cloud HLS暗号化は、デバイスでのビデオの再生に検出可能な変更を追加しません。Video Cloud HLS暗号化は、ビデオファイルのHLSレンディションにのみ影響し、MP4レンディションには影響しません。

    制約事項

    • HLS暗号化は、アカウント内のすべての非DRMHLSレンディションに適用されます。宣伝用の動画など、暗号化なしで配信したい動画がある場合、HLS暗号化を有効にしていない別のVideo Cloudアカウントに動画をアップロードします。
    • 従来の取り込みシステムを使用して取り込まれたビデオの場合、Video Cloudは、HLS暗号化の実装前にVideoCloudにアップロードされたHLSレンディションの暗号化をサポートしていません。以前にアップロードしたHLSコンテンツは暗号化されません。絶対ですビデオを再トランスコードするそれらを保護するためにHLS暗号化の前にアップロードされます。これはしますない DynamicDeliveryシステムを使用して取り込まれたビデオに適用されます。
    • Video Cloudは、MediaモジュールでHLS暗号化ビデオを識別するためのインジケーターを表示しません。
    • ユーザーがAppleデバイスでHLS暗号化ビデオを再生し、TTLの期限が切れた後にそれを再生しようとすると、再生は開始に失敗し、ユーザーにアラートメッセージを提供しません。
    • Brightcoveプレーヤーを使用する場合、HLS再生はIE8またはIE9ではサポートされません。これらのデバイスではMP4を使用します。
    • HLSeは Android SDK でサポートされています。VisualOnコンポーネントを使って、以前のバージョンでサポートされる予定です。HLSeはHTML5プレーヤーを使うと4.xでも機能します。AndroidデバイスでのHLSeサポートの詳細は「Androidサポート対象メディアフォーマット」を参照してください。
    • HTML5プレーヤーの場合、HLSeのサポートは使用するOSやデバイスによって完全に決まります。
    • Video Cloud HLS暗号化を終了することを選択した場合、新しくアップロードされたビデオは保護されません。ただし、以前に暗号化されたHLSレンディションは再生に失敗し、再生するには再トランスコーディングが必要になります。
    • 現在、HLSE コンテンツは Android、iOS、または tvOS 用 Brightcove ネイティブ SDK を使用したオフライン再生ではサポートされていません。
