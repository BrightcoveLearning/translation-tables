    メディアモジュールを使用したメディア共有

    このトピックでは、他のアカウントにビデオを共有する方法を学びます。
     

    メディア共有によって、パブリッシャーは動画を一度アップロードすると、他のアカウントに動画を共有することができます。そのアカウントは、ご自身のアカウント、企業アカウント、他のパブリッシャーのアカウントのいずれでも構いません。共有動画は、他のVideo Cloudアカウントで独自に利用できます。通常、メディア共有が使用されるのは、パブリッシャーが動画コンテンツのマスターアカウントを維持しながら、組織の他の部門または子会社と動画を共有したい場合です。

    注:
    • メディア共有は、Video CloudEnterpriseのお客様のみが利用できます。Video Cloudアカウントのアップグレードについては、ブライトコーブにお問い合わせください。
    • メディア共有は双方向にすることができます。各アカウントは、他のアカウントのマスターとアフィリエイトになることができます。

    メディア共有は、CMS API を使用してプログラムで実装することもできます。詳細については、「 CMS API を使用したメディア共有」を参照してください。

    メディア共有の用語

    このトピックでは、以下の用語を使用します。

    • マスター -元のビデオを作成したアカウント。マスターはコンテンツを所有し、コンテンツを設定、管理、アフィリエイトに提供する責任があります。
    • アフィリエイト -動画を受信しているアカウント。アフィリエイトは、マスターから共有されたコンテンツを受け入れることができます。

    メディア共有の制限

    • DRMを使用して保護されたビデオの共有はないサポートされています。DRM 対応でないアカウントから DRM 対応のアカウントへのビデオの共有はサポートされますが、共有動画は DRM 用にパッケージされません。

    • 「ライブストリームをVODに変換」オプションを使用して作成されたビデオの共有は失敗します。ライブストリームをクリッピングして作成されたビデオで機能します。

    • 共有ビデオは、元のビデオが可能な場合にのみ、オフラインで視聴できるようにダウンロードできます。このプロパティは元のビデオから継承され、共有ビデオに対してのみ変更することはできません。

    • SSAI で共有ビデオを再生する場合、SSAI マクロ置換では、子ビデオの代わりに親ビデオのメタデータが使用されます。また、親動画がとしてマークされている場合、子動画にというラベルが付いている場合でもAdvertising='Free'、SSAI Ad Supportedは広告検索をスキップします。

    メディア共有関係の管理

    アカウント間で動画を共有する前に、メディア共有設定をの一部として構成する必要があります管理者>メディア共有設定。アカウント管理者のみが[管理]メニューにアクセスできることに注意してください。他のアカウントと動画を共有する場合は、アフィリエイトを追加する必要があり、アフィリエイトアカウントはその関係に同意する必要があります。マスターアカウントとアフィリエイトアカウントは、いつでも関係を終了することができます。メディア共有関係を終了しても、アフィリエイトアカウントから共有動画が削除されることはありません。アフィリエイトのアカウントから共有動画を削除する場合は、自分の（マスター）アカウントから共有動画を削除するか、動画のプロパティを開き、[ メディア共有 ] セクションに移動して、アフィリエイトアカウントの [ 動画の削除 ] アイコンをクリックします。

    メディア共有設定を構成する方法については、「メディア共有設定の管理」を参照してください。

    動画を共有する

    デフォルトでは、Mediaモジュールにアクセスできるアカウントの全ユーザーが動画を共有できます。動画を共有するには、次の手順を行います。

    1. メディアモジュールを開き、1つ以上のビデオを選択して、[ 詳細>共有。
      注：多数の動画を共有する場合、ブライトコーブは一度に最大 50 本の動画を共有することをお勧めします。一度に多数の動画を共有すると、Studioでタイムアウトエラーが発生する場合があります。多数の動画を共有する必要がある場合は、CMS API使用すべきです。
    2. 動画を共有するアフィリエイトアカウントを選択し、[ 共有 ] をクリックします。検索フィールドを使用して、アカウント ID を検索できます。検索フィールドはあいまい検索をサポートしているため、ID の一部を入力すると一致が返されます。

    動画を再共有（以前に共有した動画を同じアカウントで共有する）ことは可能です。これにより、アフィリエイトアカウントによって行われた動画メタデータの変更は、マスター (共有) アカウントの値に置き換えられます。動画を再共有するときは注意してください。

    共有動画を確認する

    アフィリエイトアカウントには、マスターアカウントごとに承認モードを設定できます。いつ受け入れモードです自動、共有動画は、メディアモジュールのアフィリエイトアカウントの動画リストに表示されます。いつ受け入れモードですマニュアル、保留中の株式共有動画の数を表示するリンクがメディアモジュールに表示されます。

    注：承認モードの設定については、「メディア共有設定の管理」を参照してください。

    [共有保留]リンクをクリックすると、共有動画のリストが表示されます。

    共有保留リストから、以下ができます。

    • 拒否 - 共有動画を拒否し、お客様のアカウントに入れません。共有動画を拒否しても、今後マスターがアカウントに再度共有できなくなるわけではありません。
    • 同意する -あなたのアカウントに共有ビデオを受け入れます。

    クイックビューを使用すると、パネルを開いてビデオのプロパティを表示できます。共有動画を承諾すると、その動画はMediaモジュールの[共有保留]リストに表示されなくなり、[すべての動画]ビューに表示されるようになります。

    Mediaモジュールにおける共有動画

    共有動画は、動画のサムネイルの上隅に共有アイコンが表示され、その動画がアカウント内またはアカウントから共有されたことを示します。

    このアカウントが共有先である動画

    共有アイコン（）をポイントすると、共有の詳細が表示されます。

    このアカウントが共有元である動画

    共有アイコン（）をポイントすると、共有の詳細が表示されます。

    マスターアカウントは、を使用してアフィリエイトアカウントに共有されたビデオのステータスを確認できますメディア共有ビデオプロパティページのセクション。

    エラーはここにも表示されることがあります。たとえば、 [カスタム項目の照合を強制する ] がオンの場合、アフィリエイトアカウントにはマスターアカウントと同じカスタム項目が必要です。カスタムフィールドがないと、共有が失敗します。

    共有動画を削除する

    共有動画がマスターアカウントから削除されると、共有した動画を削除すると、その動画を共有しているすべてのアフィリエイトアカウントから削除されることを知らせるメッセージが表示されます。

    共有ビデオのプロパティ

    動画がアフィリエイトアカウントで共有されると、次の動画プロパティが共有動画とともにコピーされます。

    • 名前
    • 短い説明と長い説明
    • 参照ID（共有先のアフィリエイトアカウントに同じ参照IDの動画がある場合には、エラーメッセージが表示され、動画共有に失敗します。参照IDの命名規則を工夫し、アフィリエイトと同じ参照IDを使用しないようにしてください）
    • 関連リンク（リンクテキストとリンクURL）
    • タグ
    • ビデオプロジェクション
    • サムネイルとポスター画像-デフォルトでは、マスタービデオのポスターとサムネイル画像はアフィリエイトアカウントで使用されます。マスターアカウントのポスターおよび/またはサムネイルに変更が加えられた場合、アフィリエイトアカウントが以前に新しいポスターまたはサムネイル画像を生成していない限り、それらの変更はアフィリエイトアカウントに反映されます。アフィリエイトアカウントがポスターやサムネイル画像に変更を加えると、マスターアカウントで行われたポスター/サムネイルへの変更はアフィリエイトアカウントには反映されません。
    • テキストトラック-アフィリエイトには、共有動画のテキストトラックプロパティを削除または編集することはできません。
    • 広告（無料/広告サポート）
    • 広告キー
    • 開始日と終了日の設定
    • 地理的制限-パブリッシャーは、マスターアカウントとアフィリエイトアカウントが地理的制限に関して同じように構成されていることを確認する必要があります。
    • 関連付けられたメタデータを持つキューポイント
    • カスタムフィールド-マスターアカウントが [カスタムフィールドマッチングを強制する ] に設定されている場合、アフィリエイトアカウントにはマスターアカウントと同じカスタムフィールドが必要です。カスタムフィールドがないと、共有が失敗します。
    注：カスタムフィールドがリストタイプの場合、カスタムフィールドには両方のアカウントで同じオプションが必要です。

    共有ビデオには、マスタービデオが共有された時点で保持していたビデオメタデータフィールドのすべての値が含まれます。ただし、その時点から、マスターコピーとアフィリエイトコピーは独立したエンティティになります。

    • メディア共有の使用時には動画ファイルはコピーされません。動画メタデータのみがコピーされます。共有動画は、ご自身のアカウントのオリジナルの動画をポイントしています。
    • マスターアカウントから動画を削除すると、その動画はすべてのアフィリエイトアカウントから削除されます。
    • アフィリエイトが動画に加えた変更は、オリジナルまたは他の共有コピーには影響しません。アフィリエイトが共有動画を削除した場合、その動画はアフィリエイトのアカウントから削除されますが、マスターアカウントと他のすべてのアフィリエイトアカウントには残ります。
    • アフィリエイトは、メディアモジュールを使用してテキストトラックを削除または更新することはできません。
    • アフィリエイトが動画を共有して受信すると、その動画はアクティブな状態になります。これは、ビデオがマスターアカウントでアクティブでない場合でも当てはまります。
    • アフィリエイトが動画を受け入れ、必須フィールドがない場合、動画は無効になります。
    • 共有動画を受信したアフィリエイトアカウントは、その動画を別のアカウントと共有することはできません。マスターアカウントのみがビデオを共有できます。
    • 各アカウントにおける共有動画の再生（動画の再生および帯域幅の使用）に対して、それぞれのアカウントへ別々に請求が行われます。
    • 共有動画のレポートは各アカウント固有で、それぞれのアカウントの個々の動画コピーの情報を集約したものではありません。

    動画を再共有（以前に共有した動画を同じアカウントで共有する）ことは可能です。これにより、アフィリエイトアカウントによって行われた動画メタデータの変更は、マスター (共有) アカウントの値に置き換えられます。動画を再共有するときは注意してください。

    共有ビデオレンディション

    ビデオはアフィリエイトアカウントに共有できます前トランスコーディングが完了しました。共有ビデオを受信するアフィリエイトアカウントは、マスターアカウントで作成されたレンディションにアクセスできます。ソースアカウントの動画が再トランスコードされた場合、またはソースファイルが変更された場合、それらの変更は共有された動画を受領したアフィリエイトアカウントでも反映されます。

    アフィリエイトアカウントは、共有ビデオのソースファイルを再トランスコードまたは置換することはできません。

    共有メディアと請求

    アカウント間で共有されるメディアが請求にどのように影響するかについて、よく質問があります。パブリッシャー間で共有されているコンテンツについて、請求がどのように処理されるかを説明するシナリオを下にいくつか示します。

    シナリオ1

    • パブリッシャーAとパブリッシャーBはブライトコーブCDN上にコンテンツを持っている
    • パブリッシャーAはパブリッシャーBと共有する
    • パブリッシャーBはストリームと帯域に対してブライトコーブに支払う

    シナリオ2

    • パブリッシャーAはBYO CDNを使用している
    • パブリッシャーBはブライトコーブCDNまたはBYO CDN上にいる（どちらかは関係なし）
    • パブリッシャーAはパブリッシャーBと共有する
    • パブリッシャーBはストリームに対してブライトコーブに支払う。パブリッシャーAは帯域幅に対してご自身のCDNに支払う

    シナリオ3

    • パブリッシャーAはブライトコーブCDN上にコンテンツを持っている
    • パブリッシャーBはBYO CDNを使用している
    • パブリッシャーAはパブリッシャーBと共有する
    • パブリッシャーBはストリームと帯域幅に対してブライトコーブに支払う。この場合、請求パラメーターが渡されると仮定している。請求パラメーターを適切に渡さないカスタム動画プレーヤーが作成されたために請求パラメーターが渡されないという極端的な例が今までにありました。

    シナリオ4-コンテンツはOriginからフェッチする必要があります

    アカウントIDを渡します すべてのリクエストで 各アカウントが独自のオリジンアクセスに対して請求されるようにします。だからもし共有者アカウントはCDNキャッシュを頻繁に見逃し、コンテンツの元に戻らなければなりません。共有者そのために請求されます。の場合ShareeアカウントはCDNキャッシュを頻繁に見逃し、コンテンツの元に戻らなければなりません。 Shareeアカウントはそのために請求されます。CDN構成シナリオに関係なく、Originアクセスの料金は別途異なります。

    シナリオ5-画像

    ダイナミックデリバリーのデフォルトでは、マスターアカウント（画像の作成者）は、共有アセットに関連付けられたすべての画像トラフィックに対して請求されます。アフィリエイトアカウントが共有コンテンツの新しい画像をアップロードまたはキャプチャすると、画像トラフィックが関連付けられ、アフィリエイトアカウントに請求されます。
