    Uploadモジュールを使用して動画をアップロードする

    このトピックでは、Video Cloudアカウントへの動画のアップロードに使用されるUploadモジュールの概要を示します。
     

    アップロードモジュールは、動的な取り込みプロセスを使用して動画コンテンツを取得およびトランスコードします。アップロードするファイルを選択すると、Video Cloud はプルベースのプロセスを使用してストレージロケーションからコンテンツをプルし、Zencoder トランスコーディングサービスを使用してコンテンツをトランスコードします。

    Uploadモジュールにアクセスするには、ナビゲーションヘッダーの[Upload]をクリックします。

    サポートされているビデオ形式

    ビデオフォーマットとコーデック、およびVideo Cloudでサポートされているソースビデオフォーマットについては、を参照してください。サポートされているビデオコーデックとコンテナ。

    トランスコードプロファイルを選択する

    動画をアップロードする前に、取り込みプロファイルを選択してください。取り込みプロファイルには、ビデオがトランスコードされるときに使用されるトランスコード設定が含まれています。使用可能な取り込みプロファイルのリストは、VideoCloudアカウントの構成方法によって異なります。次のスクリーンショットでは、両方CONTEXT AWARE ENCODINGそしてDYNAMIC DELIVERYプロファイルが利用可能です。

    注：動的配信の詳細については、を参照してください。動的配信の概要。
    注：高効率ビデオコーディング（HEVC）プロファイルは、有料のアドオンとしても利用できます。HEVCの詳細については、を参照してください。 HEVCビデオ。

    作成されるビデオレンディションは、選択された取り込みプロファイルに基づいています。Brightcoveが提供する取り込みプロファイルの詳細については、を参照してください。動的配信とコンテキストアウェアエンコーディングの標準的な取り込みプロファイル

    ポスター画像とサムネイル画像の画像サイズは、取り込みプロファイルで指定された画像サイズに基づいています。

    注：追加の取り込みプロファイルの作成、またはアカウントのデフォルトプロファイルの設定については、を参照してください。取り込みプロファイルの管理。

    動画をアップロードする

    注：動画をアップロードする前に、適切な取り込みプロファイルが選択されていることを確認してください。

    動画は、次のいずれかの方法でアップロードできます。

    • ドラッグアンドドロップ -ファイルをファイルシステムから[モジュールのアップロード]ページにドラッグします。ナビゲーションヘッダーの下のページ全体がドロップゾーンになります。ファイルが削除されると、アップロードプロセスが開始されます。
    • ファイル選択ダイアログ -[ ビデオを参照 ] をクリックしてファイル選択ダイアログを開き、アップロードするファイルを参照して選択します。
    • リモートURL -クリックURLを入力ソースファイルの場所を指定します。ソースファイルは、パブリックDropboxフォルダーまたはAmazonS3の保存場所にある場合があります。クリックビデオの処理終わった時に。
    注
    • ファイルサイズの制限はありません。ブライトコーブは問題なく100GBのアップロードでテストしました。
    • 大きなファイルをアップロードする場合、Brightcoveはキューイングをアップロードに設定一度に1つのファイルをアップロードします。
    • 大きなファイルをアップロードするときにStudioのタイムアウトエラーが発生した場合は、ファイルをS3またはその他のリモートストレージの場所にアップロードして、StudioのURLアップロードオプションを使用してみてください。
    • YouTubeのURLは、ビデオファイルへの直接リンクを生成しないため、使用できません。
    • Googleドライブからファイルをアップロードする方法については、を参照してください。リモートURLを使用したビデオのアップロード。
    • Dropboxからアップロードするときに、URLが「？dl = 0」で終わる場合は、ファイルをアップロードする前に「？dl = 1」に変更する必要がある場合があります。
    • ビデオファイル名（拡張子を含む）は、120 シングルバイト（60 全バイト）文字を超えないようにしてください。その場合は、ビデオは正常に取り込まれますが、後で再変換することはできません。

    ファイルを選択すると、ファイルのアップロードが開始されます。アップロードをキャンセルするには、[ アップロードをキャンセル ] をクリックします。[ 閉じる ] をクリックして、ページからファイルエントリを削除します。アップロードが処理状態にある間は、ブラウザウィンドウを閉じたり、ブラウザウィンドウから離したりしないでください。そうすると、アップロードが不完全になる可能性があります。

    アップロードが完了すると、「処理完了」ステータスが表示されます。これは、処理が完了し、アップロードモジュールから安全に移動できることを示します。

    クリック編集取り込みプロファイルとビデオプロパティのサブセットを表示します。ビデオトランスコーディングが開始されていない場合は、取り込みプロファイルを変更できます。次のビデオプロパティを編集できます。

    • 状態 -ビデオステータス
      • アクティブ -ビデオを再生できます
      • 非活性 -ビデオを再生できません

      デフォルトでは、ビデオがアップロードされると、必須フィールドが欠落していない限り、ステータスはアクティブに設定されます。必須フィールドがない場合、メッセージが表示され、必須フィールドが赤色で強調表示されます。必須フィールドが欠落している動画は引き続きアップロードできますが、すべての必須フィールドに値が設定されるまで非アクティブのままになります。必須フィールドの使用に関する詳細は、「動画の必須フィールドを設定する」を参照してください。

      注：アカウントレベルの設定があります、デフォルトは非アクティブ、アップロードされたすべての動画を非アクティブに設定するために使用できます。詳細は「アカウント設定を構成する」を参照してください。
    • 名前 -ビデオのタイトル
    • 簡単な説明 - 動画の簡単な説明
    • タグ - 動画と関連付けるタグ
    • 参照ID - 動画参照ID
    • ビデオプロジェクション - 選択する360度ビデオが360度ビデオであることを示します。360 度ビデオの操作の詳細については、「 360° ビデオの操作」を参照してください。
    • オフライン再生を有効にする -オンにすると、デバイス SDK を使用して作成したアプリケーションを使用するときに、ビデオをローカルに保存し、オフラインで表示できるようになります。このオプションは、アカウントがオフライン再生が有効になっている場合にのみ表示されます。使用する場合にのみこのオプションを設定することをお勧めします高度なアップロード（ユーザーが手動でアップロード処理を開始する必要がある場合）。動画のアップロード中にこれを設定すると、問題が発生する可能性があります。
    • フォルダへ移動 -ビデオを割り当てるフォルダを選択します。フォルダの操作の詳細については、「フォルダを使用してビデオを整理する」を参照してください。
    • カスタムフィールド - 必須のマークが付いたカスタムフィールドがすべて表示されます。

    動画プロパティに関する詳細は「Mediaモジュールを使用して動画のプロパティ値を編集する」を参照してください。

    クリック保存する変更を保存して、トランスコーディングプロセスを開始します。

    高度なアップロードを使用する

    いつ高度なアップロードアカウント設定が有効になっている場合、ビデオプロパティパネルにも表示されます選択するポスターとサムネイル画像をビデオと一緒にアップロードできるボタン。クリック選択するポスターとサムネイル画像を選択し、をクリックします処理するトランスコードを開始します。

    ポスター画像とサムネイル画像の画像サイズは、取り込みプロファイルで指定された画像サイズに基づくことに注意してください。構成については、高度なアップロードアカウント設定、を参照してくださいアカウント設定の管理。

    デフォルトのフォルダとタグを設定する

    動画をアップロードするときに、アップロードする動画のフォルダとタグを指定できます。フォルダドロップダウンリストには、メディアモジュールで作成されたすべてのフォルダのリストが表示されます。クリック+フォルダを作成新しいフォルダを作成し、ビデオを新しいフォルダに割り当てます。新しいフォルダを作成する場合：

    • 名前は100文字に制限されています
    • フォルダ名はアカウント内で一意である必要があります
    • 不正な文字:/

    フォルダを選択すると、アップロードされたすべての動画がそのフォルダに追加されます。メディアモジュールを使用してビデオを作成してフォルダに割り当てる方法については、を参照してください。フォルダを使用してビデオを整理する。

    タグは、アップロード時に動画に割り当てることもできます。ビデオに割り当てるタグのコンマ区切りリストを入力します。

    フォルダを選択するかタグを追加したら、をクリックしますすべてに適用変更を保存します。フォルダとタグは、アップロード中、アップロード中、またはアップロードが終了し、アップロードモジュールにビデオが表示されているすべてのビデオに適用されることに注意してください（退出させるボタンはクリックされていません）。

    モバイルデバイスから動画をアップロードする

    Mediaモジュールは完全にHTML5で記述されており、タブレットやモバイルデバイスでも使用できます。これにより、モバイルデバイスでビデオを撮影し、VideoCloudアカウントにアップロードできます。モバイルデバイスからコンテンツをアップロードするには、アップロードモジュールを開き、取り込みプロファイルを選択します。クリックするとビデオを閲覧する、デバイス上の写真アプリケーションが開きます。アップロードするビデオを選択し、[完了]をクリックします。モバイルデバイスからのビデオのアップロードが開始され、進行状況がデバイスに表示されます。

    特殊文字

    ビデオクラウドがビデオをフェッチするためのURLを提供していて、URLの一部として資格情報を送信している場合は、次のことを行う必要があります。パーセントエンコード特定の予約済み特殊文字：

    文字 パーセントエンコード
    ] %5B
    [ %5D
    ? %3F
    / %2F
    < %3C
    ~ %7E
    # %23
    ` %6D
    ! %21
    @ %40
    $ %24
    % %25
    ^ %5E
    & %26
    * %2A
    ( %28
    ) %29
    + %2B
    = %3D
    } %7D
    | %7C
    : %3A
    「 %22
    ; %3B
    ' %27
    , %2C
    > %3E
    { %7B
    スペース %20

    例は次のとおりです。

    
    /* unencoded */
    ftp://user@example.com:pass!word@example.com/path/to/input.mp3

    /* encoded */
    ftp://user%40example.com:pass%21word@example.com/path/to/input.mp3
