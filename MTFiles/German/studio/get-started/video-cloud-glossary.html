    Video Cloud-Glossar

    Ein Leitfaden zu wichtiger Terminologie für Video Cloud und Internetvideo.

    | -A- | -B- | -C- | -D- | -E- | -F- | -G- | -H- | -ICH- | -J- | -K- | -L- | -M- | -N- | -O- | -P- | -Q- | -R- | -S- | -T- | -U- | -V- | -W- | -X- | -Y- | -Z- |

    A

    Aktive Medien
    Die Anzahl der aktiven Videos im Konto. Beachten Sie, dass diese Daten nicht in Echtzeit sind - sie werden einmal täglich aus Ihrer Videothek synchronisiert. Beachten Sie auch, dass diese Daten möglicherweise nicht korrekt sind, wenn Sie sich „Echtzeit“ -Analysedaten ansehen (weniger als 3 Tage alt).
    Analytics-API
    REST-basierte Anwendungsprogrammierschnittstelle, mit der Entwickler Analytics-Metriken für Video Cloud-Videos direkt im JSON-Format abrufen können. Der Analytics API stellt die Daten für die Analytics-Benutzeroberfläche in Video Cloud Studio bereit.
    Zugriffstoken
    Ein temporäres OAuth2-Token, das den Zugriff auf die REST-APIs wie die CMS-API, Analytics-API und Dynamic Ingest-API ermöglicht.
    Partnerunternehmen
    Webpublisher, die Video Cloud-Player syndizieren.
    Anzeigenanfrage
    Eine Anforderung, die von einem Video Cloud-Player an einen Anzeigenserver gesendet wird; auch als Aufruf bezeichnet. Die Anzeigenanforderung ist ein HTTP-Aufruf und enthält Schlüssel-/Wertpaare, die Informationen übergeben, die die Ausrichtung des Anzeigenservers auf eine entsprechende Anzeige für den Kontext vereinfachen. Nicht jede Anzeigenanforderung gibt eine Anzeige zurück.
    Analytics-API
    Das Analytics API Ermöglicht das programmgesteuerte Abrufen von Analysedaten für Video Cloud-Konten und -Videos.
    Apple HLS
    Apple HTTP Live Streaming, ein von Apple implementiertes Protokoll zur Bereitstellung von Video on Demand. Apple HLS unterteilt das Video in eine Reihe kleiner Dateidownloads, wobei jeder Download mithilfe von HTTP einen kleinen Abschnitt des Videos lädt. Apple HTTP Live Streaming hat nichts mit Live-Streaming-Video zu tun.
    Seitenverhältnis
    Das Verhältnis von Breite und Höhe für den Frame eines Videos. 4:3 für Standardsysteme und 16:9 für HDTV.

    B

    Bandbreite
    Der Bereich von Signalfrequenzen, die ein Audio- oder Video-Gerät codiert oder decodiert. Da Video eine höhere Frequenz als Audio verwendet ist eine größere Bandbreite erforderlich.
    Blog
    Eine webbasierte Publikation, die aus periodisch veröffentlichen Beiträgen besteht (häufig in umgekehrt chronologischer Reihenfolge).
    Brightcove Player
    Der Brightcove Player ist ein hochleistungsfähiger, plattformübergreifender „HTML5-First“-Video-Player, der schnell geladen wird, HLS-Video mit adaptiver Bitrate in hoher Qualität auf Desktops und Mobilgeräten abspielt und in Ihre bevorzugten Werbe- und Analytics-Systeme integriert werden kann.
    Brightcove-Player-API
    Die Brightcove-Player-API dient zum programmgesteuerten Festlegen von Aussehen und Verhalten des Brightcove-Players.
    Breitband-Video-Kanal
    Eine Zusammenstellung verwandter Videos, die über das Internet verteilt werden.
    Bumper
    Ein kurzer Video-Clip, der vor einem langen Video oder beim Laden eines Players angezeigt wird.

    C

    CMS-API
    Das CMS API Bietet nicht zwischengespeicherten Lese- und Schreibzugriff auf die Videometadaten.
    Codec
    Der zum Aufzeichnen von analogem Video oder Audio im Digitalformat verwendete Algorithmus. Vergleichen Sie Container.
    Container
    Ein Video-Container ist ein Format zur Definition der Speicherung der Video-Metadaten mit den Video- und Audio-Daten. Es definiert weder die Codierung noch die Komprimierung der Daten. Beispielsweise kann MP4 ein Container für H.264-Dateien sein.
    Content-Delivery-Netzwerk (CDN)
    Ein Service, der vernetzte Server hostet, die Inhalt auf transparente Weise für Endbenutzer bereitstellen (insbesondere umfangreichen Medien-Content). Zwei bekannte CDNs sind Akamai und Limelight.
    Content-Targeting
    Beim Content-Targeting wird basierend auf den über einen Besucher bekannten Informationen unterschiedlicher Content angezeigt.
    CSS
    Cascading Style Sheets sind ein standardmäßiger Mechanismus zur Steuerung des Designs einer HTML-Seite. CSS werden für das Design von Brightcove-Playern verwendet.
    Cue-Points
    Unsichtbare Markierungen, die an bestimmten Stellen in ein Video eingefügt werden und externe Ereignisse auslösen können, wie zum Beispiel Mid-Roll-Anzeigenanforderungen oder Untertitel. Sie können Cue-Points mit dem Media-Modul festlegen.

    D

    DASH (MPEG-DASH)
    Dynamic Adaptive Streaming über HTTP (auch MPEG-DASH genannt), ist Streaming mit adaptiver Bitrate, mit dem qualitativ hochwertiges Streaming von Medien-Content von herkömmlichen HTTP-Webservern über das Internet ermöglicht wird. MPEG-DASH gleicht der Apple-Lösung HTTP Live Streaming (HLS) und unterteilt den Content in kleine HTTP-Dateisegmente.
    Domänenbeschränkung
    Eine grundlegende Form des Inhaltsschutzes, der dafür sorgt, dass ein Player nur in festgelegten Domänen funktioniert.
    DRM
    Digital Rights Management ist eine Methode zum Verschlüsseln von Mediendateien, die so verpackt werden, dass sie nur mit einem Richtlinienschlüssel dechiffrierbar sind.
    Dynamische Übermittlung
    Siehe Streaming mit variablen Bitraten.
    Dynamic Ingest-API
    Mit der Dynamic Ingest-API (DI) wird Video Cloud angewiesen, ein Video-Objekt abzurufen und Wiedergabeversionen zu erstellen. Das Dynamic Ingest API kann auch verwendet werden, um Miniatur- und Posterbilder oder WebVTT-Dateien für Untertitel, Kapitel oder Untertitel aufzunehmen.

    E

    Codierung
    Konvertiert Video- und Audio-Daten in einen Stream, der in einem Player wiedergegeben werden kann.
    Verschlüsselung
    Der Prozess des Decodierens von Videos auf eine Weise, dass sie nur von Berechtigten abspielbar sind.

    F

    Bildrate
    Die Anzahl der Bilder pro Sekunde, die bei der Wiedergabe eines Video-Clips angezeigt werden.

    G

    Geofilterung
    Fügt Videos geografische Einschränkungen hinzu, z. B. zur Verwaltung von Eigentumsrechten.

    H

    H.264 Format
    Auch bekannt als MPEG-4 AVC (Advanced Video Coding). Ein Dateiformat zur Bereitstellung von Video über das Internet. Für die Bereitstellung von Online-Video wird das H.264-Format empfohlen. H.264-Dateien verwenden in der Regel den MP4-Container und verfügen über eine MP4-Dateierweiterung. Dieses Format ermöglicht eine deutlich höhere Komprimierung als Vorgängerformate.
    HLS
    HTTP Live Streaming – dieses von Apple entwickelte Video-Format stellt ein Video in kurzen Segmenten bereit, die auf der Clientseite gestreamt werden.
    HTML
    (HyperText Markup Language) Eine Auszeichnungssprache zur Erstellung von Webseiten mit Hypertext und weiteren Informationen für das Anzeigen in einem Webbrowser.
    HTML5
    HTML5 ist eine im Internet verwendete Kerntechnologie-Auszeichnungssprache zum Strukturieren und Bereitstellen von Content im World Wide Web. HTML5 ist die fünfte Version des HTML-Standards vom World Wide Web Consortium (W3C).

    I

    Ingest-Profil
    Ingest-Profile enthalten die Transcodierungseinstellungen, die beim Hochladen von Videos mit dem Upload-Modul oder der Dynamic Ingest-API verwendet werden.
    Ingest Profiles-API
    Die Ingest Profiles-API ermöglicht das Erstellen, Abrufen, Aktualisieren und Löschen von Ingest-Profilen für Ihr Video Cloud-Konto. Ingest-Profile definieren die beim Hinzufügen eines Videos zu Video Cloud erstellten Wiedergabeversionen.

    J

    JavaScript
    Eine Skriptsprache, die am besten für ihre Verwendung auf Websites bekannt ist. Sie wird jedoch auch verwendet, um den Skriptszugriff auf Objekte zu ermöglichen, die in andere Anwendungen eingebettet sind.
    JPEG (Joint Photographic Experts Group)
    Eine gängige Standardmethode zur verlustbehafteten Komprimierung von digitalen Bildern. JPEG ist das standardmäßige Bildformat, das im Brightcove-Service für Miniaturansichten, Video-Standbilder und Hintergrundbilder verwendet wird.

    K

    Schlüssel- / Wertepaare
    Wenn von einem Player Anzeigenanforderungen gesendet werden, wird mindestens ein Schlüssel-/Wertpaar eingeschlossen (z. B. Kategorie=Sport), das die Anzeigenserverinformationen zum Anzeigenkontext enthält und die Anzeigenausrichtung verbessert.

    L

    Letterbox
    Eine Methode zur Darstellung von Breitbild-Video auf einem Bildschirm mit einem anderen Seitenverhältnis, bei der dem Bild oben und unten meist schwarze Ränder hinzugefügt werden.
    Logo
    Eine individuelle Grafik, die eine Marke symbolisch repräsentiert.
    Logo-Overlay
    Ein kleines, transparentes Bild, das im Fenster für die Video-Wiedergabe angezeigt wird. Manchmal wird es auch als Cornerlogo oder Wasserzeichen bezeichnet.
    Schleife
    Eine Schleife beschreibt die erneute Wiedergabe der Videos in einer Wiedergabeliste am Anfang der Liste, nachdem die Wiedergabe der Liste beendet wurde.
    Live-Streaming-Video
    Ein Protokoll für das Live-Streaming von Videos bei einer Veranstaltung; zum direkten Veröffentlichen von aufgezeichneten und codierten Streams; im Gegensatz zur Bereitstellung codierter Dateien für das Streamen bei beispielsweise VOD.

    M

    Medienbibliothek
    Die Sammlung der Videos, Wiedergabelisten und anderen Dateien in einem Video Cloud-Konto.
    Metadaten
    Der Ausdruck Metadaten bezieht sich auf die Beschreibungen, Tags, das Genre und andere verwandte Informationen, die in der Definition eines Videos bereitgestellt werden.
    MPAA-Filmbewertungssystem
    Ein von der Motion Picture Association of America eingerichtetes System zur Bewertung eines Films auf Grundlage seines Inhalts. Das MPAA-Bewertungssystem (G, PG, R usw.) ist das bekannteste System zur Klassifizierung potenziell anstößigen Inhalts. Allerdings wird es normalerweise nur für die Filmbranche verwendet, da die MPAA Markenrechte für jede individuelle Bewertung besitzt.
    Streaming mit variablen Bitraten
    Eine Funktion zur Unterstützung mehrerer Versionen oder Wiedergaben eines einzelnen Videos, die die Bereitstellung in der für die Browserfenstergröße und Bandbreite des Benutzers am besten geeigneten Wiedergabeform ermöglicht. Früher wurde dies auch als „dynamische Übermittlung“ bezeichnet.
    Muxing und Demuxing
    Schreiben und Lesen von Audio- und Videostreams in und aus unterschiedlichen Containerformaten.

    N

    Native Player-SDKs
    Die Brightcove Native Player-SDKs umfassen die Tools, die zum Erstellen videofähiger Apps für iOS und Android benötigt werden.

    O

    OAuth
    OAuth ist ein offener Standard für die Autorisierung. OAuth bietet Clientanwendungen den „sicheren delegierten Zugriff“ auf Serverressourcen im Auftrag eines Ressourceneigentümers.
    OAuth-API
    Die OAuth-API ist eine RESTful-API, die den Erhalt von Autorisierungstoken für den Zugriff auf die anderen REST-APIs von Brightcove ermöglich.

    P

    Playback
    Das Wiedergeben von Video-Medien in einem Video Cloud-Player.
    Wiedergabe-API
    Die Wiedergabe-API kann zur Laufzeit für den Zugriff auf Video- und Wiedergabelistendaten für Mobilgeräte-Apps oder Video-Portale genutzt werden.
    Player-Verwaltungs-API

    Mit der Player-Verwaltungs-API können Sie:

    • Erstellen eines Players
    • Die Einstellungen eines Players anzeigen
    • Ändern Sie die Einstellungen eines Players
    • Alle Player zu einem Publisher auflisten
    • Einen Player veröffentlichen, um diesen Verbrauchern bereitzustellen
    Player-Veröffentlichungscode
    Der Code "Snippet" zum Einbetten eines Brightcove-Players in eine Webseite.
    Wiedergabeliste
    Eine Zusammenstellung mehrerer Videos, die in einer bestimmten Reihenfolge gruppiert sind und in einem Video Cloud-Player wiedergegeben werden sollen.
    Wiedergabelisten-API
    Mit der Wiedergabelisten-API wird eine Wiedergabeliste mittels JavaScript programmgesteuert.
    Richtlinien-API
    Mit der Richtlinien-API werden Richtlinienschlüssel erstellt oder abgerufen, die für den Zugriff auf die Wiedergabe-API verwendbar sind. Richtlinienschlüssel werden automatisch für Brightcove-Player generiert.
    Plakatbild
    Ein statisches Bild, das vor oder nach dem Abspielen des Videos im Wiedergabebereich eines Video-Players angezeigt wird.
    px
    Abkürzung für Pixel, die in CSS verwendet werden.

    Q

     
     

    R

    Wiedergabeversion
    Bei der Verwendung von Streaming mit variablen Bitraten können Sie mehrere Versionen des gleichen Videos mit unterschiedlicher Bitrate und unterschiedlichen Abmessungen verwenden. Jede dieser Versionen wird als Wiedergabeversion bezeichnet. Das Streaming mit variablen Bitraten von Video Cloud, das entsprechend den Playerabmessungen und der Verbindungsgeschwindigkeit des Benutzers angepasst wird, bietet die beste Wiedergabe.
    Reference ID
    Eine eindeutige ID, die ein Publisher einem Video oder einer Wiedergabeliste zuweist. Diese Referenz-ID (refid) unterscheidet sich von den Video- und Wiedergabelisten-IDs, bei denen es sich um eindeutige, von Video Cloud generierte IDs handelt. Die Referenz-ID ermöglicht die Koordination zwischen Video Cloud und anderen Diensten, wie beispielsweise Content-Management-Systemen.
    RSS-Feeds
    Really Simple Syndication ist eine weit verbreitete Technologie, um Benutzer über Content-Aktualisierungen für eine Website, einen Blog oder einen Internet-TV-Kanal zu informieren.

    S

    Suchmaschine
    Ein Programm, z. B. Google, zum Durchsuchen des Webs nach Informationen.
    Website des sozialen Netzwerks
    Eine Website, z. B. Facebook oder Twitter, die Mitgliedern die Möglichkeit zur Freigabe von Mitteilungen, Medien und anderen Informationen für andere Benutzer bietet.
    Streaming
    Ein Codierformat mit niedriger Bitrate für die Verwendung in Netzwerken und im Internet. Beim Streaming wird das Video in kleinen Segmenten in einer codierten Bitrate bereitgestellt, die der Verbindungsgeschwindigkeit des Benutzers entspricht. Beim Streaming kann der Remotebenutzer das Audio oder Video mit minimalen Unterbrechungen abspielen, ohne zuerst die gesamte Video-Datei herunterzuladen.

    T

    Tags
    Individuelle Keywords oder Sätze zur Strukturierung von Content. Brightcove unterstützt die Verwendung von Tags als Möglichkeit für Publisher zur Strukturierung und Klassifizierung von Videos und zum Erstellen von Wiedergabelisten.
    Miniaturbild
    Ein kleines, einem Video zugeordnetes Bild, das zum Anzeigen des Videos in Playern, auf Websites und in Video-Suchmaschinen verwendet wird.
    Token-basierte Authentifizierung
    Systeme mit tokengestützter Authentifizierung ermöglichen Benutzern die Eingabe ihres Benutzernamens und Kennwortes, um ein Token abzurufen, mit dem sie ohne Benutzername und Kennwort den Zugriff auf eine bestimmte Ressource erhalten. Nach dem Abrufen des Tokens kann der Benutzer das Token, das eine bestimmte Zeit lang den Zugriff auf eine bestimmte Ressource ermöglicht, an die Ressource übermitteln.
    TV Parental Guidelines
    Ein von der Federal Communications Commission (FCC) der USA entworfenes System zur Klassifizierung von TV-Sendungen auf Grundlage der Altersangemessenheit des Inhalts.

    U

    URL
    Eine URL, kurz für Uniform Resource Locator, ist eine Möglichkeit zur Ermittlung einer Ressource und ihres Standorts im Internet (z. B. http://www.brightcove.com).

    V

    Variable Bitrate
    Ein Komprimierungsschema, bei dem jede Einheit des Eingangsmaterials auf unterschiedliche Größen komprimiert werden kann. Für MPEG-2-Videos bedeutet dies beispielsweise, dass „einfachere“ Sequenzen (dh ohne Bewegung) auf sehr kleine Größen komprimiert werden können, während „harte“ Sequenzen (mit vielen Bewegungs- und Szenenschnitten) auf viel größere komprimiert werden können Größen.
    Video

    Ein einzelner Video-Titel zur Wiedergabe in einem Video Cloud-Player. Ein Video basiert auf einer Video-Datei, die zu Beginn eines der vielen unterstützten Video-Dateiformate aufweisen kann und anschließend in eine FLV- oder H.264-Datei konvertiert wird, sofern sie nicht bereits eines dieser Formate aufweist. Zudem kann ein Video Metadaten wie die folgenden beinhalten:

    • Name
    • Kurzbeschreibung
    • Ausführliche Beschreibung
    • Tags
    • Video-Standbild
    • Posterbild
    • Bumper
    • Logo-Overlay
    Video Cloud Studio

    Eine vielseitige Internetanwendung für den Zugriff auf den Video Cloud-Service und dessen Steuerung. Studio umfasst eine Reihe von Modulen zum Ausführen verschiedener Aufgaben:

    • Hochladen - Laden Sie Videos in Ihr Video Cloud-Konto hoch
    • Medien - Legen Sie Videoeigenschaften fest, organisieren Sie Videos mithilfe von Ordnern und veröffentlichen Sie Videos
    • Spieler - Erstellen und formatieren Sie Player und generieren Sie Veröffentlichungscode für Player
    • Analytik - Anzeigen von Analysen zur Video- und Player-Leistung
